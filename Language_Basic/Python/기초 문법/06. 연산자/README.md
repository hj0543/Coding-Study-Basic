# 🧮 Programming Operators

프로그래밍에서 자주 사용하는 연산자(Operator)에 대한 정리입니다.

## 1. 산술 연산자 (Arithmetic Operators)
기본적인 수학 계산을 수행합니다.

| 연산자 | 설명 | 예시 (`a=10, b=3`) | 비고 |
|:---:|:---|:---|:---|
| `+` | 덧셈 | `a + b` → `13` | |
| `-` | 뺄셈 | `a - b` → `7` | |
| `*` | 곱셈 | `a * b` → `30` | |
| `/` | 나눗셈 | `a / b` → `3.33..` | |
| `%` | 나머지 | `a % b` → `1` | 홀/짝수 판별 등에 사용 |
| `//` | 몫 | `a // b` → `3` | **Python** 전용 |
| `**` | 거듭제곱 | `a ** b` → `1000` | **Python** 전용 |

<br>

## 2. 비교 연산자 (Comparison Operators)
두 값을 비교하여 참(`True`) 또는 거짓(`False`)을 반환합니다.

| 연산자 | 설명 |
|:---:|:---|
| `==` | 값이 같다 |
| `!=` | 값이 다르다 |
| `>` / `<` | 크다 / 작다 |
| `>=` / `<=` | 크거나 같다 / 작거나 같다 |

#### ⚖️ `is` vs `==` 차이점 (Python)

Python에서 두 객체를 비교할 때 사용하는 두 연산자의 결정적인 차이입니다.

| 연산자 | 비교 대상 | 의미 | 판단 기준 (Python 내부) |
|:---:|:---:|:---|:---|
| **`==`** | **값 (Value)** | 내용이 같은가? | `a.__eq__(b)` |
| **`is`** | **참조 (Reference)** | 같은 객체(메모리)인가? | `id(a) == id(b)` |

### 📝 예시 코드
```python
a = [1, 2, 3]
b = [1, 2, 3] # a와 값은 같지만 다른 객체
c = a         # a와 완전히 같은 객체를 참조

# 1. 값 비교 (Equality)
a == b  # True (내용이 같음)

# 2. 식별 비교 (Identity)
a is b  # False (메모리 주소가 다름)
a is c  # True (메모리 주소가 같음)
```
### ⚠️ 주의: `1` vs `True` / `2` vs `2.0` 비교 예시
기본은 '==' / None, True, False는 'is'
Python의 자료형 특성으로 인해 발생하는 `is`와 `==`의 결과 차이입니다.

| 코드 | 결과 | 설명 |
|:---|:---:|:---|
| `1 == True` | **True** ✅ | `True`는 내부적으로 정수 `1`로 취급됨 (Value 일치) |
| `1 is True` | **False** ❌ | `int` 객체와 `bool` 객체는 다름 (Reference 불일치) |
| `2 == 2.0` | **True** ✅ | 정수와 실수의 크기가 같음 (Value 일치) |
| `2 is 2.0` | **False** ❌ | `int`와 `float`는 저장 방식이 다름 (Reference 불일치) |

> **요약:** 값의 크기만 중요하다면 `==`를, 자료형(객체 타입)까지 엄격하게 구분해야 한다면 `is`보다는 `type()` 확인이나 `is`를 주의해서 사용해야 합니다.
<br>

## 3. 논리 연산자 (Logical Operators)
조건문의 결합을 위해 사용합니다. 언어별로 표기법이 다를 수 있습니다.

| 설명 | 기호 (Java/C/JS) | 기호 (Python) | 의미 |
|:---:|:---:|:---:|:---|
| **AND** | `&&` | `and` | 두 조건이 **모두 참**일 때만 참 |
| **OR** | `\|\|` | `or` | 두 조건 중 **하나라도 참**이면 참 |
| **NOT** | `!` | `not` | 참/거짓 반전 |

<br>

## 4. 대입 연산자 (Assignment Operators)
변수에 값을 저장하거나 연산 후 저장합니다.

* **기본:** `=` (대입)
* **복합:** `+=`, `-=`, `*=`, `/=`, `%=` (연산 후 대입)
  * *예시:* `x += 3` 은 `x = x + 3`과 동일

<br>

## 5. 비트 연산자 (Bitwise Operators)
데이터를 비트(0, 1) 단위로 제어합니다.

| 연산자 | 설명 | 비고 |
|:---:|:---|:---|
| `&` | AND | 둘 다 1일 때 1 |
| `\|` | OR | 하나라도 1이면 1 |
| `^` | XOR | 서로 다르면 1 |
| `~` | NOT | 비트 반전 (0↔1) |
| `<<` | Left Shift | 비트를 왼쪽으로 이동 (*2) |
| `>>` | Right Shift | 비트를 오른쪽으로 이동 (/2) |

<br>

## ❗ 연산자 우선순위 (Operator Precedence)
하나의 식에 여러 연산자가 있을 때 계산되는 순서입니다.

1. **`()`** : 괄호 (최우선)
2. **`**`** : 거듭제곱
3. **`*`, `/`, `%`** : 곱셈, 나눗셈
4. **`+`, `-`** : 덧셈, 뺄셈
5. **`==`, `>`** : 비교 연산
6. **`&&`, `||`** : 논리 연산
7. **`=`** : 대입 연산 (가장 나중)

> **Tip:** 우선순위가 헷갈린다면 항상 **괄호 `()`**를 사용하여 명시하는 것이 좋습니다

## ⚡ 단축 평가 (Short-Circuit Evaluation)

논리 연산 시 결과가 확정되면 나머지 연산 과정을 생략하고, **결과를 결정지은 값 자체를 반환**하는 방식입니다.

### 1. 동작 규칙

| 연산자 | 첫 번째 값(Left) | 평가 과정 | 결과값 |
|:---:|:---:|:---|:---:|
| **`A and B`** | **False** | `B` 확인 안 함 (Short-circuit) | `A` (값) |
| | **True** | `B`까지 확인해야 함 | `B` (값) |
| **`A or B`** | **True** | `B` 확인 안 함 (Short-circuit) | `A` (값) |
| | **False** | `B`까지 확인해야 함 | `B` (값) |

### 2. 활용 예시 (Python)

#### ✅ 안전한 코드 작성 (Guard Clause)
앞의 조건이 거짓이면 뒤의 위험한 코드(에러 발생 가능)를 실행하지 않습니다.
```python
# user가 None이면 뒤의 user.name에 접근하지 않음 (AttributeError 방지)
if user is not None and user.name == "Admin":
    print("관리자 접속")
```
