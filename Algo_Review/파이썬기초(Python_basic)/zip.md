파이썬의 **`zip()`** 내장 함수는 코딩 테스트에서 코드를 극단적으로 줄여주는 마법 같은 도구다.

이름 그대로 옷의 **'지퍼(Zipper)'**를 올리는 것처럼, 길이가 같은 여러 개의 리스트(또는 튜플, 문자열 등 반복 가능한 객체)를 **같은 인덱스끼리 짝지어서 튜플 형태로 묶어주는 역할**을 한다. `for`문 여러 개를 써야 할 일을 단 한 줄로 끝내버릴 수 있어서, 파이썬의 '비밀 무기'라고 불리기도 한다.

### 💡 핵심 개념 및 동작 과정

1. 2개 이상의 리스트를 `zip()` 안에 쉼표로 구분해서 넣는다.
2. 각 리스트의 0번째 원소끼리, 1번째 원소끼리, 2번째 원소끼리... 묶어서 튜플로 반환한다.
3. **만약 리스트들의 길이가 다르다면?** 가장 길이가 짧은 리스트에 맞춰서 묶고, 남는 데이터는 깔끔하게 버린다. (에러가 발생하지 않는다!)
4. 묶인 데이터를 다시 풀고 싶을 때는 `zip(*묶인_데이터)` 형태(언패킹)로 다시 분리할 수도 있다.

### 💻 zip 활용법 기본 코드 (Python)

`zip()`이 코딩 테스트에서 어떻게 쓰이는지, 가장 대표적인 4가지 활용법이다.

#### 1. 여러 리스트를 동시에 순회하기

인덱스(`i`)로 접근할 필요 없이 변수 두 개로 바로 값을 뽑아 쓸 수 있다.

```python
names = ['Alice', 'Bob', 'Charlie']
scores = [85, 92, 78]

# 기존 방식: for i in range(len(names)): print(names[i], scores[i])
# zip 활용:
for name, score in zip(names, scores):
    print(f"{name}의 점수는 {score}점")

```

#### 2. 두 리스트를 딕셔너리(Hash)로 순식간에 합치기

Key 리스트와 Value 리스트가 따로 있다면 `zip` 한 방에 끝낼 수 있다.

```python
keys = ['A', 'B', 'C']
values = [1, 2, 3]

# zip으로 묶고 dict로 변환
my_dict = dict(zip(keys, values))
print(my_dict) # 결과: {'A': 1, 'B': 2, 'C': 3}

```

#### 3. (🔥핵심) 2차원 배열의 행과 열 뒤집기 (전치 행렬, Transpose)

2차원 격자(Grid)를 다루는 시뮬레이션 문제에서, **세로열을 가로행으로 바꾸거나 배열을 90도 회전시킬 때** 이 코드는 그야말로 치트키다.

```python
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# *matrix로 리스트의 껍데기를 벗기고(언패킹), zip으로 세로끼리 묶어버림
transposed = list(zip(*matrix))

for row in transposed:
    print(row)
# 출력 결과:
# (1, 4, 7)
# (2, 5, 8)
# (3, 6, 9)

```

> 이 코드 한 줄이면 `for`문을 이중으로 돌려서 $x, y$ 좌표를 뒤집는 수고를 완벽하게 덜어낼 수 있다!

#### 4. 리스트 안의 원소들끼리 연산하기

```python
list1 = [10, 20, 30]
list2 = [1, 2, 3]

# 두 리스트의 같은 위치 원소끼리 더하기
sum_list = [a + b for a, b in zip(list1, list2)]
print(sum_list) # 결과: [11, 22, 33]

```

---

### ⏱️ 시간 복잡도

`zip()` 함수의 시간 복잡도는 묶어주는 데이터의 최소 길이를 $N$이라고 할 때 **$O(N)$**이다. 내부적으로 C언어로 고도로 최적화되어 있어서, 파이썬에서 인덱스(`arr[i]`)로 직접 접근하며 `for`문을 도는 것보다 `zip`을 쓰는 것이 실행 속도도 미세하게 더 빠르고 메모리도 적게 사용한다.

## 2차원 배열 치트키

코딩 테스트에서 2차원 배열(격자)을 다루는 시뮬레이션 문제(카카오나 삼성 등)를 풀 때 절대 모르면 안 되는 궁극의 치트키 공식이다.

`for`문을 이중으로 돌리면서 $x, y$ 인덱스를 복잡하게 계산할 필요 없이, 파이썬의 `zip`과 `*`(언패킹)을 쓰면 딱 한 줄로 90도 회전이 끝나버린다.

### 💡 시계 방향 90도 회전 황금 공식

```python
rotated = list(zip(*matrix[::-1]))

```

단 한 줄이다. 이 공식이 어떻게 마법처럼 90도 회전을 만들어내는지 3단계로 쪼개서 보여주겠다.

then zip unpacking]

1. **1단계: 배열 위아래 뒤집기 (`matrix[::-1]`)**
먼저 배열의 행(Row) 순서를 맨 아래부터 맨 위로 뒤집는다.
* 원본: `[[1, 2, 3], [4, 5, 6], [7, 8, 9]]`
* 결과: `[[7, 8, 9], [4, 5, 6], [1, 2, 3]]`


2. **2단계: 언패킹 (`*`)**
껍데기(가장 바깥 리스트)를 벗겨서 3개의 독립된 리스트로 던져준다.
* 결과: `[7, 8, 9]`, `[4, 5, 6]`, `[1, 2, 3]`


3. **3단계: 같은 인덱스끼리 묶기 (`zip`)**
던져진 3개의 리스트에서 0번째 끼리, 1번째 끼리, 2번째 끼리 세로로 묶어버린다.
* 0번째 원소들: `(7, 4, 1)`
* 1번째 원소들: `(8, 5, 2)`
* 2번째 원소들: `(9, 6, 3)`



---

### 💻 완벽한 실전 코드 (Python)

`zip`으로 묶인 결과는 내부가 '튜플' 형태가 되기 때문에, 만약 회전시킨 배열의 값을 수정해야 한다면 안쪽 튜플을 다시 리스트로 바꿔주는 리스트 컴프리헨션을 살짝 곁들이면 완벽하다!

```python
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# 시계 방향 90도 회전 공식! (내부를 list로 변환까지 완벽하게)
rotated_90 = [list(row) for row in zip(*matrix[::-1])]

for row in rotated_90:
    print(row)

# 출력 결과:
# [7, 4, 1]
# [8, 5, 2]
# [9, 6, 3]

```

---

### 🔥 꿀팁 추가 (반시계 방향 90도 회전):

시계 방향이 배열을 먼저 뒤집고(`[::-1]`) 언패킹해서 묶었다면, 반시계 방향은 언패킹해서 묶은(`zip`) 다음 그 전체 결과를 뒤집어버리면 된다!

* **반시계 방향 공식:** `list(zip(*matrix))[::-1]`

이 공식은 정말 구구단처럼 달달 외워두면 격자 문제에서 남들 10분 동안 인덱스 계산할 때 1초 만에 다음 스텝으로 넘어갈 수 있다!
