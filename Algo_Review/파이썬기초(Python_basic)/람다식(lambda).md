람다는 파이썬 코딩 테스트에서 절대 빠질 수 없는 **'가성비 끝판왕'** 문법이다.

정식으로 `def`를 써서 함수를 정의하기엔 너무 작고 일회성인 연산을 딱 한 줄로 처리하고 버릴 때 사용하는 **'이름 없는 익명 함수'**다. 특히 리스트를 정렬할 때 **'다중 조건'**을 걸어주는 용도로 코딩 테스트에서 숨 쉬듯이 사용된다.

### 💡 람다 함수 작성법

기본적인 구조는 아주 단순하다. `def`나 `return` 같은 키워드 없이 한 줄로 끝난다.

* **형식:** `lambda 매개변수1, 매개변수2: 실행할 표현식(반환값)`

---

### 💻 람다 기본 코드 (Python)

일반 함수와 람다 함수가 어떻게 다른지, 그리고 주로 어떤 내장 함수(`map`, `filter`)와 찰떡궁합으로 쓰이는지 한눈에 볼 수 있는 코드다.

```python
# 1. 일반 함수 vs 람다 함수
def add(a, b):
    return a + b

add_lambda = lambda a, b: a + b

print(add(3, 5))         # 결과: 8
print(add_lambda(3, 5))  # 결과: 8

# 2. map()과 함께 사용하기 (리스트의 모든 원소에 일괄 적용)
numbers = [1, 2, 3, 4, 5]
# numbers의 각 원소(x)를 제곱(x**2)해서 리스트로 만듦
squared = list(map(lambda x: x**2, numbers))
print(squared) # 결과: [1, 4, 9, 16, 25]

# 3. filter()와 함께 사용하기 (조건에 맞는 원소만 걸러냄)
# 짝수(x % 2 == 0)만 남김
evens = list(filter(lambda x: x % 2 == 0, numbers))
print(evens) # 결과: [2, 4]
```

### 🔥 코딩 테스트 실전 압축: 정렬(`sort`)에서의 람다

람다가 코딩 테스트에서 진짜 빛을 발하는 순간은 **복잡한 조건으로 정렬(`sort`, `sorted`)을 해야 할 때**다. `key` 속성에 람다 함수를 넘겨주면, 내가 원하는 기준대로 배열을 자유자재로 요리할 수 있다.

#### 1. 다중 조건 정렬 (우선순위 부여)

튜플을 반환하도록 하면, 앞에 있는 조건부터 순서대로 정렬해 준다.

```python
students = [
    ('Alice', 85, 100), # (이름, 영어 점수, 수학 점수)
    ('Bob', 90, 90),
    ('Charlie', 85, 95)
]

# 정렬 조건: 
# 1순위. 영어 점수 오름차순
# 2순위. 영어 점수가 같다면 수학 점수 오름차순
students.sort(key=lambda x: (x[1], x[2]))
print(students) 
# 결과: [('Alice', 85, 100), ('Charlie', 85, 95), ('Bob', 90, 90)]
```

#### 2. 오름차순과 내림차순 섞기 (마이너스 부호 활용)

숫자 데이터의 경우, 람다 표현식에 **마이너스(`-`)**를 붙이면 해당 기준만 **내림차순(역순)**으로 정렬한다! (문자열에는 마이너스를 붙일 수 없다)

```python
# 정렬 조건:
# 1순위. 영어 점수 내림차순 (높은 순)
# 2순위. 영어가 같다면 수학 점수 오름차순 (낮은 순)
# 3순위. 둘 다 같다면 이름의 사전 순
students.sort(key=lambda x: (-x[1], x[2], x[0]))
```

#### 3. 문자열 길이를 기준으로 정렬

```python
words = ["banana", "apple", "cat", "dog", "elephant"]

# 1순위. 단어 길이 순 (짧은 것부터)
# 2순위. 길이가 같다면 알파벳 사전 순
words.sort(key=lambda x: (len(x), x))
print(words)
# 결과: ['cat', 'dog', 'apple', 'banana', 'elephant']
```

### ⏱️ 시간 복잡도

람다 함수 자체를 실행하는 시간은 보통 **$O(1)$**이다.
하지만 정렬(`sort`)의 `key`로 사용할 때는 파이썬의 정렬 알고리즘(Tim Sort)이 돌아가므로 전체 시간 복잡도는 **$O(N \log N)$**이 된다. 데이터가 10만 개 주어져도 람다를 곁들인 `sort()` 한 번이면 순식간에 정렬이 끝난다는 뜻이다.
