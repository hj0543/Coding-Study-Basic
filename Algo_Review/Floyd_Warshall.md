플로이드 워셜(Floyd-Warshall) 알고리즘은 **'모든 지점에서 다른 모든 지점까지의 최단 경로'**를 모두 구해야 하는 경우에 사용하는 알고리즘이다.

다익스트라 알고리즘이 하나의 출발점에서 다른 모든 노드로의 최단 경로를 구하는 것과 대조적이다. 매 단계마다 '거쳐 가는 노드'를 기준으로 알고리즘을 수행하며, 2차원 리스트(행렬)에 최단 거리 정보를 저장한다는 특징이 있다. 또한, 다익스트라와 달리 음수 가중치를 갖는 간선이 있어도 정상적으로 동작하지만, 사이클의 합이 음수인 '음수 사이클'이 존재한다면 사용할 수 없다.

### 💡 알고리즘 동작 과정

1. 2차원 리스트(배열)를 생성하고, 모든 노드에서 다른 모든 노드로 가는 최단 거리를 무한대(Infinity)로 초기화한다.
2. 자기 자신에서 자기 자신으로 가는 비용(대각선 위치)은 0으로 초기화한다.
3. 주어진 각 간선에 대한 정보를 2차원 리스트에 반영한다. (초기 상태)
4. 1번 노드부터 차례대로 $k$번 노드를 **'거쳐 가는 노드'**로 설정한다.
5. 현재 노드 $k$를 제외한 다른 모든 노드 쌍 $(a, b)$에 대해, $a$에서 $b$로 직접 가는 비용과 $k$를 거쳐서 가는 비용($a \rightarrow k \rightarrow b$)을 비교하여 더 작은 값으로 2차원 리스트를 갱신한다.
6. 모든 노드를 거쳐 가는 경우를 다 확인할 때까지 4~5번 과정을 반복한다.

> **참고:** 플로이드 워셜 알고리즘의 핵심은 다이나믹 프로그래밍(DP)의 성격을 띠는 다음 점화식이다.
> 
> $$D_{ab} = \min(D_{ab}, D_{ak} + D_{kb})$$
> 
>

### 💻 플로이드 워셜 알고리즘 기본 코드 (Python)

2차원 리스트와 3중 반복문만으로 구현할 수 있어 코드가 매우 직관적이고 짧다.

```python
INF = int(1e9) # 무한을 의미하는 값으로 10억을 설정

# 노드의 개수 및 간선의 개수
n = 4
m = 7

# 2차원 리스트(그래프 표현)를 만들고, 모든 값을 무한으로 초기화
graph = [[INF] * (n + 1) for _ in range(n + 1)]

# 자기 자신에서 자기 자신으로 가는 비용은 0으로 초기화
for a in range(1, n + 1):
    for b in range(1, n + 1):
        if a == b:
            graph[a][b] = 0

# 각 간선에 대한 정보를 입력받아, 그 값으로 초기화 (예시 생략)
# for _ in range(m):
#     a, b, c = map(int, input().split())
#     graph[a][b] = c # a에서 b로 가는 비용은 c라고 설정

# 점화식에 따라 플로이드 워셜 알고리즘 수행
for k in range(1, n + 1):
    for a in range(1, n + 1):
        for b in range(1, n + 1):
            graph[a][b] = min(graph[a][b], graph[a][k] + graph[k][b])

# 수행된 결과를 출력
for a in range(1, n + 1):
    for b in range(1, n + 1):
        # 도달할 수 없는 경우, 무한(INFINITY)이라고 출력
        if graph[a][b] == INF:
            print("INFINITY", end=" ")
        # 도달할 수 있는 경우 거리를 출력
        else:
            print(graph[a][b], end=" ")
    print()

```

### ⏱️ 시간 복잡도

플로이드 워셜 알고리즘의 시간 복잡도는 **$O(V^3)$** 이다. ($V$: 노드의 개수)
노드의 개수가 $V$개일 때, $V$번의 단계를 거치며 각 단계마다 $O(V^2)$의 연산(2중 반복문)을 수행하기 때문이다. 따라서 시간 복잡도가 매우 큰 편이므로, 문제에서 주어지는 **노드의 개수가 대략 500개 이하일 때** 주로 사용해야 시간 초과를 피할 수 있다.

### 📝 예시 문제: 플로이드 (백준 11404번)

$n$개의 도시가 있다. 그리고 한 도시에서 출발하여 다른 도시에 도착하는 $m$개의 버스가 있다. 각 버스는 한 번 사용할 때 필요한 비용이 있다. 모든 도시의 쌍 $(A, B)$에 대해서 도시 $A$에서 $B$로 가는데 필요한 비용의 최솟값을 구하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에 도시의 개수 $n$ ($1 \le n \le 100$)이 주어지고 둘째 줄에는 버스의 개수 $m$ ($1 \le m \le 100,000$)이 주어진다. 셋째 줄부터 $m+2$줄까지 다음과 같은 버스의 정보가 주어진다. 먼저 시작 도시 $a$, 도착 도시 $b$, 한 번 타는데 필요한 비용 $c$가 주어진다. 시작 도시와 도착 도시가 같은 경우는 없다. (비용은 100,000 이하의 자연수)
* **출력:** $n$개의 줄을 출력해야 한다. $i$번째 줄에 출력하는 $j$번째 숫자는 도시 $i$에서 $j$로 가는데 필요한 최소 비용이다. 만약, $i$에서 $j$로 갈 수 없는 경우에는 그 자리에 0을 출력한다.

---

### 💡 풀이 접근법

'모든 지점에서 다른 모든 지점까지의 최단 경로'를 요구하고 있으며, 정점 $n$의 크기가 100 이하로 매우 작기 때문에 **플로이드 워셜 알고리즘**을 사용하기 완벽한 조건이다.

* **주의할 점:** 시작 도시와 도착 도시를 연결하는 노선은 **하나가 아닐 수 있다.** 따라서 입력을 받을 때 기존에 기록된 비용과 비교하여 더 작은 값으로 갱신해 주어야 한다.
* 출력 시 경로가 존재하지 않아 값이 여전히 `INF`인 경우는 0으로 변환하여 출력하는 조건도 놓치지 말아야 한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline
INF = int(1e9)

n = int(input()) # 도시의 개수
m = int(input()) # 버스의 개수

# 2차원 리스트(그래프) 무한대로 초기화
graph = [[INF] * (n + 1) for _ in range(n + 1)]

# 1. 자기 자신으로 가는 비용은 0으로 초기화
for a in range(1, n + 1):
    graph[a][a] = 0

# 2. 간선 정보 입력받기
for _ in range(m):
    a, b, c = map(int, input().split())
    # 동일한 노선이 여러 개일 수 있으므로, 가장 짧은 간선 정보만 저장
    if c < graph[a][b]:
        graph[a][b] = c

# 3. 플로이드 워셜 알고리즘 수행 (3중 반복문)
for k in range(1, n + 1):         # 거쳐가는 노드
    for a in range(1, n + 1):     # 출발 노드
        for b in range(1, n + 1): # 도착 노드
            # a에서 b로 가는 최소 비용 갱신
            graph[a][b] = min(graph[a][b], graph[a][k] + graph[k][b])

# 4. 결과 출력
for a in range(1, n + 1):
    for b in range(1, n + 1):
        if graph[a][b] == INF:
            print(0, end=" ") # 갈 수 없는 경우 0 출력
        else:
            print(graph[a][b], end=" ")
    print() # 줄 바꿈

```
