LCS(Longest Common Subsequence, 최장 공통 부분 수열)는 두 수열(또는 문자열)이 주어졌을 때, **두 수열 모두의 부분 수열이 되는 수열 중 가장 긴 것**을 찾는 알고리즘이다.

LIS와 헷갈릴 수 있는데, LIS는 한 수열 내에서의 증가 조건을 따지는 것이고, LCS는 **두 문자열 사이의 공통된 형태**를 찾는 것이 핵심이다. 이때 부분 수열은 원소들이 연속할 필요는 없지만, 순서는 유지되어야 한다.

### 💡 알고리즘 동작 과정

LCS는 대표적인 **다이나믹 프로그래밍(DP)** 문제다. 두 문자열을 행과 열로 하는 2차원 테이블을 만들어 값을 채워 나간다.

1. 두 문자열의 길이를 각각 $N, M$이라 할 때, $(N+1) \times (M+1)$ 크기의 2차원 배열 `dp`를 생성하고 0으로 초기화한다.
2. 두 문자열을 한 글자씩 비교하며 테이블을 채운다. (인덱스 $i, j$)
3. **두 문자가 같다면 (`str1[i] == str2[j]`):** 대각선 왼쪽 위 값에 1을 더한 값을 저장한다.
* 점화식: `dp[i][j] = dp[i-1][j-1] + 1`


4. **두 문자가 다르다면 (`str1[i] != str2[j]`):** 왼쪽(`dp[i][j-1]`)과 위쪽(`dp[i-1][j]`) 값 중 더 큰 값을 가져온다.
* 점화식: `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`


5. 테이블의 마지막 값(`dp[N][M]`)이 최장 공통 부분 수열의 길이가 된다.

---

### 💻 LCS 기본 코드 (Python)

가장 직관적인 2차원 DP 방식의 코드다.

```python
import sys

# 두 문자열 입력
str1 = sys.stdin.readline().strip()
str2 = sys.stdin.readline().strip()

n = len(str1)
m = len(str2)

# DP 테이블 초기화
dp = [[0] * (m + 1) for _ in range(n + 1)]

# 테이블 채우기
for i in range(1, n + 1):
    for j in range(1, m + 1):
        # 두 문자가 같다면 대각선 위에서 +1
        if str1[i - 1] == str2[j - 1]:
            dp[i][j] = dp[i - 1][j - 1] + 1
        # 다르다면 위와 왼쪽 중 큰 값 선택
        else:
            dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

# 결과 출력 (최장 길이)
print(dp[n][m])

```

### ⏱️ 시간 복잡도

LCS 알고리즘의 시간 복잡도는 **$O(N \times M)$** 이다. ($N, M$: 각 문자열의 길이)
두 문자열을 각각 한 번씩 전체 순회하며 테이블을 채우기 때문에 문자열의 길이가 각각 1,000~2,000 정도일 때까지 무난하게 해결할 수 있다.

### 📝 예시 문제: LCS (백준 9251번)

LCS(Longest Common Subsequence, 최장 공통 부분 수열) 문제는 두 수열이 주어졌을 때, 모두의 부분 수열이 되는 수열 중 가장 긴 것을 찾는 문제이다. 예를 들어, `ACAYKP`와 `CAPCAK`의 LCS는 `ACAK`가 되어 길이는 4이다.

* **입력:** 첫째 줄과 둘째 줄에 두 문자열이 주어진다. 문자열은 알파벳 대문자로만 이루어져 있으며, 최대 1,000글자이다.
* **출력:** 첫째 줄에 입력으로 주어진 두 문자열의 LCS의 길이를 출력한다.

---

### 💡 풀이 접근법

문제 제목 그대로 LCS의 길이를 구하는 가장 표준적인 문제다.

* 문자열의 길이가 최대 1,000이므로 2차원 DP 테이블을 만들어도 $1,000,000$칸 정도의 메모리만 사용하므로 충분히 통과 가능하다.
* **주의:** `dp` 테이블의 0행과 0열은 '비어있는 문자열'과의 비교를 의미하므로 0으로 비워두고, 인덱스 계산 시 `str1[i-1]`과 같이 접근하는 점에 유의한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

# 문자열 입력
s1 = input().strip()
s2 = input().strip()

# n: 행의 길이, m: 열의 길이
n, m = len(s1), len(s2)
dp = [[0] * (m + 1) for _ in range(n + 1)]

# 2차원 DP 진행
for i in range(1, n + 1):
    for j in range(1, m + 1):
        if s1[i-1] == s2[j-1]:
            # 문자가 같으면 대각선 왼쪽 위 값 + 1
            dp[i][j] = dp[i-1][j-1] + 1
        else:
            # 문자가 다르면 위쪽과 왼쪽 중 최댓값
            dp[i][j] = max(dp[i-1][j], dp[i][j-1])

# 최종 LCS 길이 출력
print(dp[n][m])

```

---

LCS는 길이를 구하는 것 외에도 **"실제 공통 부분 수열이 무엇인지"** 역추적(Backtracking)해서 출력하라는 변형 문제도 자주 나온다. (이때는 테이블 끝에서부터 거꾸로 올라가며 값을 찾으면 된다.)

이제 드디어 DP의 꽃이라고 불리는 **'배낭 문제(Knapsack Problem)'**에 대해 정리해 줄까? 아니면 문자열 문제의 연장선으로 문자열 사이의 편집 거리를 구하는 **'편집 거리(Edit Distance)'** 알고리즘을 알아볼까?
