이분 매칭(Bipartite Matching)은 두 개의 나누어진 그룹(예: 사람과 일, 소와 축사)이 있을 때, 간선들이 서로 겹치지 않게 각 그룹의 원소들을 1:1로 짝지어 줄 수 있는 **최대 매칭(짝짓기)의 수**를 구하는 알고리즘이다.

본래는 네트워크 유량(Network Flow) 알고리즘의 일종이지만, 모든 간선의 용량이 1인 특수한 경우이기 때문에 복잡한 유량 알고리즘 대신 **깊이 우선 탐색(DFS)**을 활용하여 훨씬 직관적이고 짧은 코드로 구현하는 것이 일반적이다.

### 💡 알고리즘 동작 과정

A 그룹(주체)과 B 그룹(대상)이 있다고 가정할 때, 매칭 과정은 일종의 **'양보와 밀어내기'** 방식으로 진행된다.

1. A 그룹의 원소들을 순서대로 순회하며 DFS 탐색을 시작한다.
2. 현재 A 원소가 원하는 B 그룹의 원소(도착지)를 확인한다.
3. **만약 대상 B가 비어 있다면(아무와도 매칭되지 않았다면):** 즉시 매칭을 성사시키고 탐색을 종료한다.
4. **만약 대상 B가 이미 다른 A 원소(기존 주인)와 매칭되어 있다면:** 기존 주인이 B를 양보하고 다른 대상과 매칭될 수 있는지 다시 DFS를 재귀적으로 호출하여 물어본다.
5. 기존 주인이 다른 곳으로 이동할 수 있다면 현재 A 원소가 B를 차지하고, 이동할 수 없다면 다른 B를 찾아본다.
6. 한 번의 DFS 탐색 사이클 내에서, 이미 방문한(확인한) B 그룹 원소는 다시 확인하지 않도록 방문 처리(`visited`)를 해둔다.
7. A 그룹의 모든 원소에 대해 위 과정을 반복하여 매칭이 성공한 횟수를 센다.

---

### 💻 이분 매칭 기본 코드 (Python)

DFS와 매칭 정보를 담을 리스트만 있으면 구현할 수 있어 코드가 생각보다 매우 간결하다.

```python
# A 그룹의 크기 n, B 그룹의 크기 m
n, m = 3, 3

# A 그룹의 각 노드가 연결될 수 있는 B 그룹의 노드 정보 (인접 리스트)
# 예: 1번 A는 1, 2번 B를 원함 / 2번 A는 1번 B만 원함 / 3번 A는 2번 B만 원함
graph = [
    [],
    [1, 2],
    [1],
    [2]
]

# B 그룹의 각 노드가 현재 어떤 A 그룹 노드와 매칭되어 있는지 저장하는 배열
# 0이면 아직 아무와도 매칭되지 않은 상태를 의미함
selected = [0] * (m + 1)

# DFS를 이용한 매칭 함수
def dfs(a):
    # a가 연결될 수 있는 B 그룹의 노드들을 차례대로 확인
    for b in graph[a]:
        # 이미 이번 매칭 턴에서 확인한 B 노드라면 패스 (무한 루프 방지)
        if visited[b]:
            continue
        visited[b] = True # 방문 처리
        
        # 1) B 노드가 비어 있거나
        # 2) B 노드를 이미 점유하고 있는 기존 A 노드가 다른 B 노드로 양보할 수 있다면
        if selected[b] == 0 or dfs(selected[b]):
            selected[b] = a # a와 b를 매칭시킴
            return True     # 매칭 성공
            
    return False # 모든 시도에도 매칭 실패

match_count = 0

# A 그룹의 1번 노드부터 n번 노드까지 차례대로 매칭 시도
for i in range(1, n + 1):
    # 매번 새로운 A 노드에 대해 매칭을 시도할 때마다 visited 배열을 초기화해야 함
    visited = [False] * (m + 1)
    
    # 매칭에 성공했다면 카운트 증가
    if dfs(i):
        match_count += 1

print(match_count) # 결과: 2 (모두가 원하는 곳에 갈 수는 없음)

```

### ⏱️ 시간 복잡도

DFS 기반 이분 매칭의 시간 복잡도는 **$O(V \times E)$** 이다. ($V$: A 그룹 정점의 개수, $E$: 간선의 개수)
A 그룹의 각 정점($V$)에 대해 DFS를 수행하며, DFS 한 번에 최악의 경우 모든 간선($E$)을 확인하게 되기 때문이다. 정점과 간선의 개수가 수천 개 수준일 때까지 무난하게 사용할 수 있다.

### 📝 예시 문제: 축사 배정 (백준 2188번)

농부 존은 소 $N$마리를 키우고 있다. 존에게는 소를 키울 수 있는 축사가 $M$개가 있다. 소들은 자신이 희망하는 축사에서만 들어가서 쉬고 싶어 한다. 존은 최대한 많은 소가 축사에 들어갈 수 있도록 배정해 주고 싶다. 소가 희망하는 축사의 정보가 주어질 때, 축사에 들어갈 수 있는 소의 최댓값을 구하는 프로그램을 작성하시오. (하나의 축사에는 한 마리의 소만 들어갈 수 있다.)

* **입력:** 첫째 줄에 소의 수 $N$과 축사의 수 $M$이 주어진다. ($1 \le N, M \le 200$) 둘째 줄부터 $N$개의 줄에는 각 소가 들어가기 원하는 축사에 대한 정보가 주어진다. 첫 번째 정수 $S_i$는 소가 원하는 축사의 수이고, 이후 $S_i$개의 정수는 축사의 번호다.
* **출력:** 첫째 줄에 축사에 들어갈 수 있는 소의 최댓값을 출력한다.

---

### 💡 풀이 접근법

'소(A 그룹)'와 '축사(B 그룹)'를 1:1로 짝지어 주면서 그 최대 개수를 구하는, 알고리즘 분류 자체가 **이분 매칭**인 가장 완벽한 기본 문제다.

* A 그룹 정점 $N$개, B 그룹 정점 $M$개로 두고, 소가 희망하는 축사 번호를 인접 리스트(`graph`)에 담는다.
* 앞서 배운 이분 매칭 DFS 코드를 소 1번부터 $N$번까지 그대로 돌려주면 끝난다.

---

### 💻 풀이 코드 (Python)

```python
import sys
# 재귀 깊이 제한 해제 (이분 매칭 특성상 재귀가 깊어질 수 있음)
sys.setrecursionlimit(int(1e5))
input = sys.stdin.readline

n, m = map(int, input().split())

# 소가 원하는 축사 정보를 담을 그래프
graph = [[] for _ in range(n + 1)]

# 축사의 점유 상태를 나타내는 배열 (0이면 비어있음)
selected = [0] * (m + 1)

# 소(A)의 선호도 입력받기
for i in range(1, n + 1):
    data = list(map(int, input().split()))
    # 첫 번째 원소는 선호하는 축사의 개수이므로 제외하고 그래프에 추가
    graph[i] = data[1:]

def dfs(cow):
    # 해당 소가 원하는 축사들을 하나씩 확인
    for barn in graph[cow]:
        if visited[barn]:
            continue
        visited[barn] = True
        
        # 축사가 비어있거나, 기존에 차지한 소가 다른 곳으로 갈 수 있다면
        if selected[barn] == 0 or dfs(selected[barn]):
            selected[barn] = cow # 현재 소가 이 축사를 차지함
            return True
            
    return False

result = 0

# 1번 소부터 N번 소까지 차례대로 축사 배정 시도
for i in range(1, n + 1):
    # 각 소마다 방문 기록 초기화
    visited = [False] * (m + 1)
    if dfs(i):
        result += 1

print(result)

```
