**set(집합)**은 파이썬 코딩 테스트에서 **'중복 제거'**와 **'초고속 탐색'**이 필요할 때 0.1초의 망설임도 없이 꺼내 들어야 하는 마법의 자료구조다.

앞서 다뤘던 딕셔너리(`dict`)와 마찬가지로 내부적으로 해시(Hash) 테이블로 구현되어 있다. 딕셔너리가 '키(Key): 값(Value)'의 쌍을 저장한다면, 집합은 오직 '키(Key)'만 저장하는 딕셔너리라고 생각하면 이해하기 쉽다!

### 💡 핵심 특징 2가지

1. **중복을 허용하지 않는다:** 리스트에 똑같은 숫자가 100만 개 있어도, `set`으로 변환하는 순간 단 1개만 남고 다 사라진다.
2. **순서가 없다 (Unordered):** 들어간 순서를 기억하지 않기 때문에 리스트처럼 인덱스(`s[0]`)로 접근할 수 없다.

### 💻 집합(Set) 기본 코드 (Python)

집합은 데이터를 넣고 빼는 기본 연산뿐만 아니라, 수학 시간에 배웠던 **집합 연산(교집합, 합집합 등)**을 연산자 하나로 기가 막히게 지원한다!

```python
# 1. 초기화 (리스트를 집합으로 변환하여 중복 제거)
arr = [1, 2, 2, 3, 3, 3, 4]
my_set = set(arr)
print(my_set) # 결과: {1, 2, 3, 4}

# 2. 데이터 추가 및 삭제
my_set.add(5)        # 원소 1개 추가
my_set.update([6, 7]) # 여러 원소 한 번에 추가
my_set.remove(3)     # 원소 삭제 (없으면 에러 발생)
my_set.discard(10)   # 원소 삭제 (없어도 에러 발생 안 함! 안전함)

print(my_set) # 결과: {1, 2, 4, 5, 6, 7}

# 3. 강력한 집합 연산 (수학)
a = set([1, 2, 3, 4, 5])
b = set([3, 4, 5, 6, 7])

print(a | b) # 합집합: {1, 2, 3, 4, 5, 6, 7}
print(a & b) # 교집합: {3, 4, 5}
print(a - b) # 차집합: {1, 2}
print(a ^ b) # 대칭 차집합 (합집합 - 교집합): {1, 2, 6, 7}

```

---

### ⏱️ 시간 복잡도

* **탐색 (`x in set`):** 평균 **$O(1)$**
* **삽입 (`add`), 삭제 (`remove`):** 평균 **$O(1)$**

리스트(`list`)에서 특정 원소가 있는지 `if x in arr:`로 찾으면 최악의 경우 리스트 끝까지 뒤져야 해서 $O(N)$이 걸린다. 하지만 집합(`set`)에서 `if x in my_set:`을 쓰면 해시 함수를 통해 방 번호를 바로 찾아가므로 **$O(1)$** 만에 존재 여부를 알아낸다! 데이터가 10만 개 이상일 때 탐색 속도 차이는 어마어마하다.

### 📝 예시 문제: 문자열 집합 (백준 14425번)

총 $N$개의 문자열로 이루어진 집합 $S$가 주어진다. 입력으로 주어지는 $M$개의 문자열 중에서 집합 $S$에 포함되어 있는 것이 총 몇 개인지 구하는 프로그램을 작성하시오.

* **입력:** 문자열의 개수 $N$과 $M$ ($1 \le N, M \le 10,000$).
* **출력:** $M$개의 문자열 중 집합 $S$에 포함된 총 개수.

#### 💡 풀이 접근법

가장 전형적인 "이게 저 안에 있어?"를 물어보는 탐색 문제다. 만약 $N$개의 문자열을 **리스트(List)**에 저장하고 $M$개의 문자열을 하나씩 꺼내어 `in` 연산자로 비교한다면, 최대 $10,000 \times 10,000 = 1억$ 번의 비교 연산이 발생해서 시간 초과가 날 수 있다.

하지만 기준이 되는 집합 $S$를 리스트가 아닌 **set**에 담아둔다면? 해시 테이블 덕분에 탐색 시간이 $O(1)$로 줄어들어서 전체 연산 횟수가 그냥 $N + M$ (약 2만 번)으로 확 줄어버린다.

#### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

# 기준이 되는 문자열 집합 S를 set으로 입력받음
s = set()
for _ in range(n):
    s.add(input().strip())

count = 0
# M개의 문자열이 집합 S에 존재하는지 확인
for _ in range(m):
    target = input().strip()
    
    # set의 탐색 시간 복잡도는 O(1)이므로 매우 빠름!
    if target in s:
        count += 1

print(count)

```

코딩 테스트에서 "중복을 허용하지 않는 개수를 세어라" 혹은 **"수많은 데이터 중에서 특정 값이 있는지 빠르게 확인하라"**는 문구를 보면 반사적으로 `set`을 타이핑하면 된다!
