큐(Queue)는 먼저 들어온 데이터가 먼저 나가는 **선입선출(FIFO, First In First Out)** 원칙을 따르는 아주 기초적이고 중요한 선형 자료구조다.

쉽게 말해 **'식당에서 밥을 먹기 위해 서 있는 대기열(줄)'**이라고 생각하면 된다. 먼저 줄을 선 사람이 먼저 식당에 들어가고, 나중에 온 사람은 맨 뒤에 서야 하는 것과 완전히 똑같다. 앞서 살펴본 BFS(너비 우선 탐색)나, 프린터의 인쇄 대기열, 운영체제의 작업 스케줄링 등 순서대로 무언가를 처리해야 할 때 핵심적으로 사용된다.

### 💡 자료구조 동작 과정

큐의 핵심 동작은 데이터를 넣는 것과 빼는 것, 딱 두 가지로 나뉜다.

1. **인큐 (Enqueue / Push):** 큐의 **맨 뒤(Rear)**에 새로운 데이터를 추가한다.
2. **디큐 (Dequeue / Pop):** 큐의 **맨 앞(Front)**에 있는 데이터를 꺼내고(삭제하고) 반환한다.
3. **피크 (Peek / Front):** 맨 앞의 데이터를 삭제하지 않고 슬쩍 확인만 한다.

### 💻 큐 기본 코드 (Python)

파이썬에는 `queue`라는 내장 모듈(`queue.Queue`)이 있긴 하지만, 이는 멀티 스레딩 환경에서 안전하게 동작하도록(Thread-safe) 설계되어 있어 속도가 꽤 느리다.

따라서 코딩 테스트에서 큐를 구현할 때는 무조건 직전에 배웠던 **`collections.deque`**를 사용하는 것이 국룰이자 정답이다. `deque`를 큐처럼 쓰려면 **'오른쪽으로 넣고(`append`), 왼쪽으로 뺀다(`popleft`)'**는 것만 기억하면 된다.

```python
from collections import deque

# 빈 큐 생성
queue = deque()

# 1. 삽입 (Enqueue): 5 - 2 - 3 - 7 순서로 넣기
queue.append(5)
queue.append(2)
queue.append(3)
queue.append(7)
print("초기 큐 상태:", queue) # deque([5, 2, 3, 7])

# 2. 삭제 (Dequeue): 먼저 들어온 5가 먼저 나감 (FIFO)
first_item = queue.popleft() 
print("꺼낸 데이터:", first_item) # 5
print("현재 큐 상태:", queue)     # deque([2, 3, 7])

# 3. 추가 삽입 및 삭제
queue.append(1)        # 1을 맨 뒤에 추가 -> deque([2, 3, 7, 1])
queue.popleft()        # 2가 나감 -> deque([3, 7, 1])
queue.popleft()        # 3이 나감 -> deque([7, 1])

# 최종 큐 상태 출력 (먼저 들어온 순서대로 출력됨)
print("최종 큐 상태:", list(queue)) # [7, 1]

```

### ⏱️ 시간 복잡도

`deque`를 사용하여 큐를 구현했을 때의 시간 복잡도다.

* **삽입 (Enqueue):** **$O(1)$**
* **삭제 (Dequeue):** **$O(1)$**

만약 일반 리스트(`list`)를 사용해서 `pop(0)`으로 맨 앞의 데이터를 꺼내면, 뒤에 있는 모든 데이터를 한 칸씩 앞으로 당겨야 하므로 $O(N)$의 시간이 걸려 매우 비효율적이다. 다시 한번 강조하지만, 큐는 무조건 `deque`다.

### 📝 예시 문제: 카드2 (백준 2164번)

$N$장의 카드가 있다. 각각의 카드는 차례로 1부터 $N$까지의 번호가 붙어 있으며, 1번 카드가 제일 위에, $N$번 카드가 제일 아래인 상태로 순서대로 카드가 놓여 있다.
이제 다음과 같은 동작을 카드가 한 장 남을 때까지 반복하게 된다.

1. 제일 위에 있는 카드를 바닥에 버린다.
2. 그다음, 제일 위에 있는 카드를 제일 아래에 있는 카드 밑으로 옮긴다.
$N$이 주어졌을 때, 제일 마지막에 남게 되는 카드를 구하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에 정수 $N$ ($1 \le N \le 500,000$)이 주어진다.
* **출력:** 첫째 줄에 남게 되는 카드의 번호를 출력한다.

---

### 💡 풀이 접근법

문제의 동작 원리가 **'맨 앞의 데이터를 빼고, 그다음 맨 앞의 데이터를 빼서 맨 뒤로 넣는다'**는 전형적인 큐(Queue)의 FIFO 구조를 설명하고 있다.
$N$이 최대 50만 개이므로, $O(N)$ 시간 복잡도를 가지는 리스트의 `pop(0)`을 쓰면 시간 초과가 발생한다.

1. 1부터 $N$까지의 숫자를 `deque`에 차례대로 넣는다.
2. 큐의 길이가 1이 될 때까지 `while`문을 돌린다.
3. `popleft()`를 한 번 수행하여 맨 위 카드를 버린다.
4. 다시 `popleft()`로 카드를 꺼낸 뒤, 그 값을 `append()`를 사용해 맨 뒤로 보낸다.
5. 마지막으로 큐에 남은 딱 1개의 원소를 출력한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
from collections import deque
input = sys.stdin.readline

n = int(input())

# 1부터 n까지의 번호를 가진 큐(데크) 생성
# 리스트 컴프리헨션을 이용해 한 번에 덱으로 초기화
q = deque([i for i in range(1, n + 1)])

# 카드가 딱 한 장 남을 때까지 반복
while len(q) > 1:
    # 1. 제일 위에 있는(맨 앞의) 카드를 바닥에 버림
    q.popleft()
    
    # 2. 그다음 제일 위에 있는 카드를 뽑아서 제일 아래(맨 뒤)로 옮김
    top_card = q.popleft()
    q.append(top_card)

# 마지막에 남은 카드 번호 출력
print(q[0])

```
