해시(Hash)는 데이터를 빠르게 저장하고 찾기 위해 **'키(Key)'를 '해시 함수(Hash Function)'에 넣어 '해시 값(Index)'으로 변환한 뒤, 이를 배열의 인덱스로 사용하여 데이터를 저장하는 자료구조**다.

수만, 수십만 개의 데이터가 있어도 처음부터 끝까지 뒤질 필요 없이, 키값만 알면 데이터가 있는 방 번호(인덱스)를 단번에 계산해 낼 수 있다. 덕분에 탐색, 삽입, 삭제를 무려 **$O(1)$**이라는 경이로운 속도로 처리할 수 있다. 코딩 테스트에서 "이름으로 점수를 찾아라", "등장한 횟수를 세어라" 같은 문제가 나오면 가장 먼저 떠올려야 하는 치트키다.

### 💡 핵심 개념과 동작 과정

해시는 크게 3가지 요소로 이루어져 있다.

1. **키 (Key):** 고유한 값 (예: 사람 이름 "Gemini")
2. **해시 함수 (Hash Function):** 다양한 길이의 키를 입력받아, 일정한 길이의 '숫자(해시 값)'로 변환해 주는 마법의 자판기다.
3. **해시 테이블 (Hash Table):** 해시 함수가 뱉어낸 숫자를 인덱스(방 번호)로 삼아 실제 '값(Value)'을 저장하는 배열이다.

> **🚨 주의할 점: 충돌(Collision)**
> 가끔 서로 다른 키를 해시 함수에 넣었는데 **우연히 같은 해시 값(방 번호)이 나오는 경우**가 생긴다. 이를 '충돌'이라고 한다.
> 충돌을 해결하기 위해 같은 방에 리스트(Linked List)를 주렁주렁 매달아 놓는 **'체이닝(Chaining)'** 기법이나, 바로 옆의 빈방을 찾아 들어가는 **'개방 주소법(Open Addressing)'**을 사용한다.

### 💻 해시 기본 코드 (Python)

코딩 테스트 실전에서 해시 함수와 테이블을 직접 바닥부터 구현할 일은 거의 없다. **파이썬의 딕셔너리(`dict`)와 집합(`set`)이 이미 완벽하게 최적화된 해시 테이블로 구현되어 있기 때문**이다.

```python
# 1. 딕셔너리(dict)를 이용한 해시 (Key-Value 구조)
hash_table = {}

# 데이터 삽입 (O(1))
hash_table["Alice"] = 95
hash_table["Bob"] = 80
hash_table["Charlie"] = 90

# 데이터 탐색 (O(1))
print(hash_table["Alice"]) # 결과: 95

# 데이터 삭제 (O(1))
del hash_table["Bob"]

# Key 존재 여부 확인 (O(1))
if "Charlie" in hash_table:
    print("Charlie가 존재합니다.")

# 2. 집합(set)을 이용한 해시 (Key만 저장, 중복 제거용)
hash_set = set(['apple', 'banana', 'apple'])
print(hash_set) # 결과: {'banana', 'apple'}

# 데이터 삽입 및 탐색 (O(1))
hash_set.add('orange')
print('orange' in hash_set) # 결과: True

```

### ⏱️ 시간 복잡도

* **탐색 / 삽입 / 삭제:** 평균적으로 **$O(1)$**
* **최악의 경우:** **$O(N)$** (해시 충돌이 너무 많이 발생해서 한 방에 데이터가 다 몰려버린 경우다. 하지만 파이썬 내장 해시는 최적화가 잘 되어 있어 코딩 테스트에서 이런 일이 발생하는 경우는 극히 드물다.)

리스트(`list`)에서 `in` 연산자를 쓰면 $O(N)$이 걸리지만, 딕셔너리(`dict`)나 셋(`set`)에서 `in` 연산자를 쓰면 $O(1)$ 만에 찾아낸다는 점이 결정적인 차이다.

### 📝 예시 문제: 나는야 포켓몬 마스터 이다솜 (백준 1620번)

도감에 수록되어 있는 포켓몬의 이름과 번호가 주어졌을 때, 이름을 보면 번호를 말하고 번호를 보면 이름을 말하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에는 도감에 수록되어 있는 포켓몬의 개수 $N$과 문제 개수 $M$이 주어진다. ($1 \le N, M \le 100,000$) 둘째 줄부터 $N$개의 줄에 1번부터 $N$번까지의 포켓몬 이름이 들어온다. 그 다음 $M$개의 줄에 문제가 들어온다.
* **출력:** 숫자가 들어오면 포켓몬 이름을, 문자가 들어오면 포켓몬 번호를 출력한다.

### 💡 풀이 접근법

포켓몬과 문제가 각각 10만 개씩이다. 문제가 주어질 때마다 리스트를 뒤지면 $100,000 \times 100,000 = 100억$ 번의 연산이 필요해 시간 초과가 난다.

1. **숫자(번호)로 이름 찾기:** 리스트(배열)의 인덱스를 활용하면 $O(1)$이다.
2. **이름(문자열)으로 번호 찾기:** 딕셔너리를 써서 `{"피카츄": 25}` 형태로 저장해 두면 $O(1)$ 만에 찾을 수 있다.

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

# 번호로 이름을 찾기 위한 리스트 (1번 인덱스부터 사용)
pokemon_list = [""]
# 이름으로 번호를 찾기 위한 해시(딕셔너리)
pokemon_dict = {}

# 1. 도감 채우기
for i in range(1, n + 1):
    name = input().strip()
    pokemon_list.append(name)
    pokemon_dict[name] = i  # 해시에 키(이름) : 값(번호) 저장

# 2. 문제 풀기
for _ in range(m):
    query = input().strip()
    
    # 입력이 숫자인 경우
    if query.isdigit():
        print(pokemon_list[int(query)])
    # 입력이 문자인 경우
    else:
        print(pokemon_dict[query])

```

해시는 코딩 테스트에서 "빠른 탐색"이 필요할 때 리스트를 대체하는 가장 훌륭한 무기다.
