**`collections.deque`**(데크 또는 덱, Double-Ended Queue)는 양방향에서 데이터를 넣고 뺄 수 있는 파이썬의 내장 자료구조다.

파이썬 코딩 테스트에서 **BFS(너비 우선 탐색)**를 구현하거나 큐(Queue) 자료구조가 필요할 때 선택이 아닌 **'필수'**로 사용해야 하는 핵심 모듈이다. "큐가 필요하면 무조건 데크를 임포트한다"라고 기계적으로 암기해도 좋을 정도다.

### 💡 왜 일반 리스트(`list`) 대신 `deque`를 써야 할까?

파이썬의 기본 리스트(`list`)로도 `pop(0)`을 쓰면 맨 앞의 데이터를 뽑아내며 큐처럼 흉내 낼 수는 있다. 하지만 리스트는 맨 앞의 데이터를 제거하면 뒤에 있는 모든 데이터를 한 칸씩 앞으로 당겨와야 하므로 **O(N)**의 시간이 걸린다.

반면 `deque`는 양쪽 끝이 뚫려 있는 구조로 설계되어 있어, 맨 앞이나 맨 뒤의 데이터를 넣고 뺄 때 단 **O(1)**밖에 걸리지 않는다. 데이터가 10만 개 단위로 넘어가는 코딩 테스트에서 리스트로 `pop(0)`을 남발하면 100% 시간 초과(TLE)의 쓴맛을 보게 된다.

---

### 💻 deque 기본 코드 (Python)

사용법은 일반 리스트와 거의 비슷하지만, '왼쪽(앞)'에서 삽입하고 삭제하는 전용 메서드가 추가되어 있다.

```python
from collections import deque

# 1. 빈 데크 생성 (리스트를 넣어 초기화할 수도 있음)
q = deque([1, 2, 3])

# 2. 데이터 삽입
q.append(4)      # 오른쪽(맨 뒤)에 4 삽입 -> [1, 2, 3, 4]
q.appendleft(0)  # 왼쪽(맨 앞)에 0 삽입 -> [0, 1, 2, 3, 4]

# 3. 데이터 삭제 및 반환
right_val = q.pop()      # 오른쪽(맨 뒤) 데이터 뽑기 -> 4 반환, [0, 1, 2, 3] 남음
left_val = q.popleft()   # 왼쪽(맨 앞) 데이터 뽑기 -> 0 반환, [1, 2, 3] 남음

print(q) # 결과: deque([1, 2, 3])

```

> **꿀팁:** 데크도 리스트처럼 인덱싱(`q[0]`, `q[-1]`)이 가능하긴 하지만, 중간에 있는 데이터를 읽거나 수정하는 작업은 리스트보다 느리다. 따라서 **'양 끝에서 데이터를 넣고 뺄 때'**만 사용하는 것이 가장 좋다.

---

### ⏱️ 시간 복잡도 비교

| 연산 | `list` (리스트) | `deque` (데크) |
| --- | --- | --- |
| **맨 뒤 삽입 (`append`)** | **O(1)** | **O(1)** |
| **맨 뒤 삭제 (`pop`)** | **O(1)** | **O(1)** |
| **맨 앞 삽입 (`insert(0, x)` / `appendleft`)** | **O(N)** | **O(1)** |
| **맨 앞 삭제 (`pop(0)` / `popleft`)** | **O(N)** | **O(1)** |

### 📝 예시 문제: 미로 탐색 (백준 2178번)

N×M 크기의 배열로 표현되는 미로가 있다. 1은 이동할 수 있는 칸을 나타내고, 0은 이동할 수 없는 칸을 나타낸다. 이러한 미로가 주어졌을 때, (1, 1)에서 출발하여 (N, M)의 위치로 이동할 때 지나야 하는 최소의 칸 수를 구하는 프로그램을 작성하시오.
(칸을 셀 때에는 시작 위치와 도착 위치도 포함한다.)

* **입력:** 첫째 줄에 두 정수 N, M (2 ≤ N, M ≤ 100)이 주어진다. 다음 N개의 줄에는 M개의 정수로 미로가 주어진다.
* **출력:** 첫째 줄에 지나야 하는 최소의 칸 수를 출력한다. 항상 도착위치로 이동할 수 있는 경우만 입력으로 주어진다.

---

### 💡 풀이 접근법

가까운 곳부터 레벨(단계)별로 탐색해 나가는 **BFS(너비 우선 탐색)**의 가장 완벽한 기본 문제다. "최단 거리"나 "최소 횟수"를 묻는 문제에서 모든 간선의 비용이 동일하다면(이 문제에선 한 칸 이동 = 1) 다익스트라 대신 BFS를 쓰는 것이 정배다.

1. `deque`를 불러와 시작 좌표 (0, 0)을 큐에 넣는다.
2. 큐에서 좌표를 하나씩 꺼내 상, 하, 좌, 우 4방향을 확인한다.
3. 이동할 수 있는 길(1)이라면, **현재 칸의 값에 1을 더한 값을 다음 칸에 덮어씌워** 거리를 기록하고 큐에 넣는다.
4. 도착 지점 (N-1, M-1)에 기록된 값을 출력하면 끝이다.

---

### 💻 풀이 코드 (Python)

```python
import sys
from collections import deque
input = sys.stdin.readline

n, m = map(int, input().split())
# 띄어쓰기 없이 붙어서 들어오는 입력 처리
maze = [list(map(int, input().strip())) for _ in range(n)]

# 상, 하, 좌, 우 이동을 위한 방향 벡터
dx = [-1, 1, 0, 0]
dy = [0, 0, -1, 1]

def bfs(x, y):
    q = deque()
    q.append((x, y))
    
    # 큐가 빌 때까지 반복
    while q:
        x, y = q.popleft() # 큐의 맨 앞에서 좌표를 꺼냄 (O(1) 소요)
        
        # 현재 위치에서 4가지 방향으로 위치 확인
        for i in range(4):
            nx = x + dx[i]
            ny = y + dy[i]
            
            # 미로 찾기 공간을 벗어난 경우 무시
            if nx < 0 or nx >= n or ny < 0 or ny >= m:
                continue
            # 벽인 경우 무시
            if maze[nx][ny] == 0:
                continue
                
            # 해당 노드를 처음 방문하는 경우에만 최단 거리 기록
            if maze[nx][ny] == 1:
                maze[nx][ny] = maze[x][y] + 1
                q.append((nx, ny)) # 다음 탐색을 위해 큐에 삽입
                
    # 가장 오른쪽 아래까지의 최단 거리 반환
    return maze[n - 1][m - 1]

# 시작 좌표 (0, 0)부터 BFS 시작
print(bfs(0, 0))

```
