
## 🔍 백트래킹(Backtracking)이란?

백트래킹은 해를 찾는 과정에서 **현재의 경로가 해가 될 것 같지 않으면, 더 이상 깊게 들어가지 않고 이전 단계로 돌아가서(Backtrack)** 다른 경로를 탐색하는 알고리즘이다. 
보통 **DFS(깊이 우선 탐색)**를 기반으로 구현한다.

### 💡 핵심 개념: 가지치기 (Pruning)

백트래킹의 가장 큰 특징은 **'가지치기'**다. 모든 경우의 수를 다 확인하는 '브루트 포스(Brute Force)'와 달리, 조건에 맞지 않는 경로는 중간에 차단하여 탐색 효율을 극적으로 높인다.

1. **상태 공간 트리**를 탐색하며 노드의 유망성(Promising)을 점검한다.
2. 해당 노드가 유망하지 않다면(조건에 맞지 않는다면), 그 노드의 부모로 돌아가서 다음 자식 노드를 탐색한다.

### 💻 백트래킹 기본 구조 (Python Template)

대부분의 백트래킹 문제는 아래와 같은 **재귀(Recursion)** 구조를 가진다.

```python
def backtracking(depth):
    # 1. 종료 조건: 원하는 해를 찾았거나 탐색의 끝에 도달했을 때
    if depth == 목표_깊이:
        # 결과 처리 (출력 혹은 리스트 저장)
        return

    # 2. 후보군 탐색
    for candidate in 후보_리스트:
        # 3. 유망성 검사 (Constraint 만족 여부)
        if 유망한가(candidate):
            방문_표시(candidate)   # 상태 변화 (결정)
            backtracking(depth + 1) # 다음 단계로 진행 (재귀)
            방문_해제(candidate)   # 상태 복구 (Backtrack) - 핵심!

```

### 📝 예시 문제: N과 M (1) (백준 15649번)

1부터 N까지 자연수 중에서 중복 없이 M개를 고른 수열을 모두 구하는 문제다.

* **입력:** 자연수 N과 M ($1 \le M \le N \le 8$)
* **출력:** 한 줄에 하나씩 문제의 조건에 맞는 수열 출력

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

def backtracking(depth):
    # 목표한 길이(m)만큼 숫자를 뽑았다면 결과 출력 (종료 조건)
    if depth == m:
        print(*result)
        return

    # 1부터 n까지의 숫자를 하나씩 확인
    for i in range(1, n + 1):
        # 유망성 검사: 아직 사용하지 않은 숫자라면
        if not visited[i]:
            visited[i] = True  # 방문 표시 (숫자 사용)
            result.append(i)   # 결과 리스트에 추가

            backtracking(depth + 1)   # 다음 숫자 뽑으러 가기 (재귀 호출)

            # --- 이 시점이 '되돌아온(Backtrack)' 시점 ---
            result.pop()       # 마지막에 넣었던 숫자 제거
            visited[i] = False # 방문 표시 해제 (다른 경로에서 쓸 수 있게 함)

# n: 전체 숫자 범위, m: 뽑을 개수
n, m = map(int, input().split())

visited = [False] * (n + 1) # 숫자 사용 여부 체크
result = []                 # 현재까지 만든 수열

backtracking(0)

```

### ⏱️ 시간 복잡도

백트래킹은 기본적으로 모든 경우의 수를 탐색하므로 시간 복잡도가 꽤 높다.

* **중복 없는 순열:** $O(N!)$
* **중복 가능한 순열:** $O(N^M)$
* **N-Queen 문제:** $O(N!)$

따라서 문제에서 주어진 **N의 범위가 작을 때(보통 20 이하)** 백트래킹을 먼저 의심해보는 것이 좋다.

---

### ✅ 백트래킹 vs DFS

많은 분들이 DFS와 백트래킹을 혼동하곤 힌다.

| 구분 | DFS (깊이 우선 탐색) | 백트래킹 (Backtracking) |
| --- | --- | --- |
| **목적** | 모든 노드를 방문하는 것 | 해를 찾는 과정에서 불필요한 경로를 차단하는 것 |
| **특징** | 그래프/트리의 완전 탐색 | 조건문(`if`)을 통한 **가지치기** 포함 |
| **효율성** | 상대적으로 비효율적일 수 있음 | 가지치기를 통해 탐색 시간을 단축함 |
