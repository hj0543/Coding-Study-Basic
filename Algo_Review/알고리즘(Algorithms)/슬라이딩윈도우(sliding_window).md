슬라이딩 윈도우(Sliding Window) 알고리즘은 마치 창문을 옆으로 밀면서 바깥 풍경을 보듯이, 1차원 배열이나 리스트에서 **'고정된 크기의 구간(윈도우)'**을 이동시키면서 원하는 결과를 도출하는 탐색 기법이다.

투포인터와 매우 유사하게 연속된 구간을 다루지만, 투포인터가 조건에 따라 구간의 길이를 늘리거나 줄이는 반면, 슬라이딩 윈도우는 **구간의 넓이(크기)가 항상 일정하게 고정되어 있다는 점**이 가장 큰 차이다. 매번 구간 안의 모든 원소를 다시 계산하지 않고, 창문이 이동할 때 '새로 들어오는 값은 포함하고, 빠져나가는 값은 제외하는' 방식으로 불필요한 중복 연산을 없애는 것이 핵심이다.

### 💡 알고리즘 동작 과정

크기가 $K$인 구간의 합을 구한다고 가정했을 때의 기본 동작 과정이다.

1. 배열의 맨 앞에서부터 크기가 $K$인 첫 번째 구간(초기 윈도우)의 합을 미리 계산하여 저장한다.
2. 이 값을 정답을 담을 변수(예: 최댓값 또는 최솟값)에 초기값으로 기록한다.
3. 창문을 오른쪽으로 딱 한 칸 이동시킨다.
4. 이때 구간 안의 모든 값을 다시 더하는 것이 아니라, 기존 구간 합에서 **창문 밖으로 벗어난 맨 앞의 값은 빼고, 창문 안으로 새롭게 들어온 맨 뒤의 값은 더해준다.**
5. 계산된 새로운 구간 합을 기존 정답과 비교하여 갱신한다.
6. 배열의 끝에 도달할 때까지 3~5번 과정을 반복한다.

---

### 💻 슬라이딩 윈도우 기본 코드 (Python)

크기가 $K$인 연속된 부분 배열의 합 중 '최댓값'을 구하는 가장 스탠다드한 뼈대 코드다.

```python
# 전체 배열과 윈도우의 크기 K 설정
arr = [1, 2, 3, 4, 5, 6, 7]
k = 3
n = len(arr)

# 1. 초기 윈도우(첫 번째 구간)의 합을 계산
window_sum = sum(arr[:k])
max_sum = window_sum # 현재까지의 최댓값으로 설정

# 2. 윈도우를 한 칸씩 오른쪽으로 이동시키며 탐색
# 인덱스 i는 새롭게 윈도우에 들어오는 원소의 위치를 의미함
for i in range(k, n):
    # 기존 합 + 새로 들어온 값(arr[i]) - 빠져나간 값(arr[i-k])
    window_sum += arr[i] - arr[i - k]
    
    # 최댓값 갱신
    max_sum = max(max_sum, window_sum)

print(max_sum) # 결과: 18 (5+6+7)

```

### ⏱️ 시간 복잡도

슬라이딩 윈도우 알고리즘의 시간 복잡도는 **$O(N)$** 이다. ($N$: 데이터의 개수)
만약 슬라이딩 윈도우를 쓰지 않고 매번 $K$개의 원소를 다시 더했다면 $O(N \times K)$의 시간이 걸렸겠지만, 윈도우를 이동할 때마다 단 두 번의 연산(덧셈 1번, 뺄셈 1번)만 수행하므로 데이터가 아무리 많아도 배열을 딱 한 번만 순회하면 끝난다.

### 📝 예시 문제: 수열 (백준 2559번)

매일 아침 9시에 학교에서 측정한 온도가 어떤 정수의 수열로 주어졌을 때, 연속적인 며칠 동안의 온도의 합이 가장 큰 값을 알아보고자 한다.
예를 들어, 온도가 3, -2, -4, -9, 0, 3, 7, 13, 8, -3 이고, 연속적인 이틀 동안의 온도의 합을 구한다면 합이 가장 큰 값은 21(13+8)이 된다.

* **입력:** 첫째 줄에는 두 개의 정수 $N$과 $K$가 한 개의 공백을 사이에 두고 주어진다. 첫 번째 정수 $N$은 온도를 측정한 전체 날짜의 수이다. ($2 \le N \le 100,000$) 두 번째 정수 $K$는 합을 구하기 위한 연속적인 날짜의 수이다. ($1 \le K \le N$) 둘째 줄에는 매일 측정한 온도를 나타내는 $N$개의 정수가 빈칸을 사이에 두고 주어진다.
* **출력:** 첫째 줄에는 입력되는 온도의 수열에서 연속적인 $K$일 동안의 온도의 합이 최대가 되는 값을 출력한다.

---

### 💡 풀이 접근법

문제에서 '연속적인 $K$일 동안의 온도 합'을 요구하고 있다. 구간의 크기($K$)가 고정되어 있으므로 전형적인 **슬라이딩 윈도우** 문제다.

전체 날짜 $N$이 최대 10만 개이므로, 매번 $K$일을 일일이 더하면 시간 초과가 날 수 있다. 앞서 배운 대로 첫 $K$일의 온도 합을 구해두고, 창문을 하루씩 밀면서 이전 날짜의 온도는 빼고 새로운 날짜의 온도는 더해가는 방식으로 $O(N)$ 만에 해결하면 된다. (참고로 온도가 음수일 수도 있으므로, 파이썬에서 최댓값을 저장할 변수의 초기값을 무작정 0으로 두면 안 되고 첫 번째 구간의 합 자체로 설정하는 것이 안전하다.)

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

# n: 전체 날짜 수, k: 연속적인 날짜 수(윈도우 크기)
n, k = map(int, input().split())
temps = list(map(int, input().split()))

# 1. 초기 윈도우 세팅 (첫 k일 동안의 온도 합)
window_sum = sum(temps[:k])
max_sum = window_sum # 첫 윈도우 합을 최댓값으로 초기화

# 2. 슬라이딩 윈도우 진행
# k번째 인덱스부터 끝까지 순회
for i in range(k, n):
    # 새로운 온도 추가하고, 구간에서 벗어난 가장 오래된 온도 빼기
    window_sum += temps[i] - temps[i - k]
    
    # 최댓값 갱신
    if window_sum > max_sum:
        max_sum = window_sum

# 결과 출력
print(max_sum)

```
