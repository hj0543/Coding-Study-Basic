깊이 우선 탐색(DFS, Depth-First Search)은 그래프 탐색에서 **한 우물만 미친 듯이 끝까지 파고드는** 알고리즘이다.

갈림길이 나오면 일단 한쪽 길을 선택해 막다른 길이 나올 때까지 전진하고, 더 이상 갈 곳이 없으면 가장 최근의 갈림길로 되돌아와서(Backtracking) 다른 길을 다시 끝까지 탐색한다. 미로를 찾을 때 한 손을 벽에 짚고 모든 길을 다 걸어보는 것과 똑같은 원리다.

### 💡 핵심 자료구조와 동작 원리

DFS를 구현할 때는 **스택(Stack)** 자료구조를 쓰거나, 컴퓨터의 메모리 스택을 그대로 활용하는 **재귀 함수(Recursion)**를 사용한다. 코딩 테스트에서는 코드가 훨씬 짧고 직관적인 재귀 함수 방식을 99% 사용한다.

1. 탐색 시작 노드를 방문 처리한다.
2. 현재 노드와 연결된 인접 노드 중, 아직 방문하지 않은 노드를 하나 골라 그 노드를 시작점으로 삼아 **자기 자신(DFS 함수)을 다시 호출**한다.
3. 연결된 모든 노드를 방문했다면 이전 노드로 돌아간다.

### 💻 그래프 입력부터 DFS까지 완벽 템플릿 (Python)

코딩 테스트에서 주어지는 간선 정보를 **'인접 리스트'** 형태로 입력받고, 이를 재귀로 탐색하는 가장 완벽하고 스탠다드한 코드다.

```python
import sys
# 🔥 파이썬에서 DFS를 쓸 때 절대 빼먹으면 안 되는 필수 코드 (재귀 깊이 해제)
sys.setrecursionlimit(10**6)
input = sys.stdin.readline

# 1. 정점(v)과 간선(e)의 개수 입력
v, e = map(int, input().split())

# 2. 그래프 초기화 (1번 노드부터 인덱스를 맞추기 위해 v + 1 크기로 생성)
graph = [[] for _ in range(v + 1)]

# 방문 처리를 위한 리스트 초기화
visited = [False] * (v + 1)

# 3. 간선 정보 입력받기 (양방향 그래프 기준)
for _ in range(e):
    a, b = map(int, input().split())
    # a와 b가 연결되어 있으므로 양쪽에 모두 추가
    graph[a].append(b)
    graph[b].append(a) # 단방향 그래프라면 이 줄은 생략

# (선택) 문제에서 "방문할 수 있는 정점이 여러 개인 경우 정점 번호가 작은 것을 먼저 방문" 
# 같은 조건이 있다면 각 리스트를 정렬해 주어야 함
for i in range(1, v + 1):
    graph[i].sort()

# --- DFS 재귀 함수 구현 ---
def dfs(node):
    # 현재 노드를 방문 처리하고 출력
    visited[node] = True
    print(node, end=' ')
    
    # 현재 노드와 연결된 다른 인접 노드들을 확인
    for nxt in graph[node]:
        # 아직 방문하지 않은 노드라면, 그 노드를 기준으로 깊게 파고듦
        if not visited[nxt]:
            dfs(nxt)

# 탐색 시작 (예: 1번 노드부터 탐색)
dfs(1)

```

### ⏱️ 시간 복잡도와 사용처

DFS의 시간 복잡도는 인접 리스트를 사용할 경우 간선과 정점을 모두 한 번씩만 확인하므로 **$O(V + E)$** 이다. ($V$: 정점의 개수, $E$: 간선의 개수)

**🔥 코딩 테스트 실전 팁:**

* **"최단 거리"**를 묻는다면 무조건 **BFS**를 쓴다. DFS는 먼저 찾은 경로가 최단 경로라는 보장이 없기 때문이다.
* **"경로의 특징"**을 저장해야 할 때 (예: A에서 B로 가는 길에 특정 색깔의 노드를 밟으면 안 된다) DFS를 쓴다.
* 모든 경우의 수를 하나씩 다 만들어보면서 조건에 안 맞으면 돌아와야 하는 **백트래킹(Backtracking)** 문제를 풀 때는 무조건 DFS 형태의 재귀를 사용한다.
