너비 우선 탐색(BFS, Breadth-First Search)은 시작 노드에서 출발해 **가장 가까운(인접한) 노드들부터 먼저 얕고 넓게 탐색**하는 알고리즘이다.

잔잔한 호수에 돌을 던졌을 때 동심원 모양으로 물결이 퍼져나가는 모습을 상상하면 정확하다. 거리가 1인 곳을 모두 방문한 뒤, 거리가 2인 곳을 모두 방문하고, 그다음 거리가 3인 곳을 방문하는 식으로 뻗어나간다.

### 💡 핵심 자료구조와 동작 원리

BFS를 코딩 테스트에서 구현할 때는 **반드시 큐(Queue)**, 즉 파이썬의 **`collections.deque`**를 사용한다. 먼저 발견한 노드를 먼저 방문해야 하는 '선입선출(FIFO)'의 원리를 따르기 때문이다.

1. 탐색 시작 노드를 큐에 삽입하고 방문 처리를 한다.
2. 큐에서 노드를 꺼내(`popleft`), 해당 노드와 연결되어 있으면서 아직 방문하지 않은 모든 인접 노드를 큐에 삽입하고 방문 처리를 한다.
3. 큐에 더 이상 노드가 남아있지 않을 때까지 2번 과정을 반복한다.

### 💻 BFS 기본 템플릿 코드 (Python)

어떤 그래프 탐색 문제가 나와도 뼈대가 되는 가장 스탠다드한 인접 리스트 방식의 BFS 코드다.

```python
from collections import deque

def bfs(graph, start, visited):
    # 1. 시작 노드를 큐에 넣고 방문 처리
    queue = deque([start])
    visited[start] = True
    
    # 3. 큐가 빌 때까지 반복
    while queue:
        # 큐의 맨 앞(왼쪽)에서 노드를 하나 꺼냄
        v = queue.popleft()
        print(v, end=' ')
        
        # 2. 꺼낸 노드와 인접한 다른 노드들을 확인
        for i in graph[v]:
            if not visited[i]:
                # 방문하지 않은 노드라면 큐에 넣고 방문 처리
                queue.append(i)
                visited[i] = True

# 각 노드가 연결된 정보를 2차원 리스트로 표현 (인덱스 0은 비워둠)
graph = [
    [],
    [2, 3, 8], # 1번 노드와 인접한 노드들
    [1, 7],    # 2번 노드와 인접한 노드들
    [1, 4, 5],
    [3, 5],
    [3, 4],
    [7],
    [2, 6, 8],
    [1, 7]
]

# 방문 여부를 기록할 리스트 (노드 개수 8개 + 인덱스 0을 위해 9개 생성)
visited = [False] * 9

# 1번 노드부터 탐색 시작
print("BFS 탐색 순서: ", end='')
bfs(graph, 1, visited)
# 결과: 1 2 3 8 7 4 5 6

```

### ⏱️ 시간 복잡도와 사용처

BFS의 시간 복잡도는 인접 리스트를 사용할 경우 **$O(V + E)$** 이다. ($V$: 정점의 개수, $E$: 간선의 개수) 모든 노드와 간선을 한 번씩만 확인하므로 데이터가 많아도 매우 빠르게 동작한다.

**🔥 코딩 테스트 실전 팁: BFS는 '최단 거리'를 보장한다.**
모든 간선의 비용(가중치)이 동일할 때, "A에서 B까지 가는 최소 이동 횟수를 구하시오" 또는 "미로를 빠져나가는 가장 빠른 길을 찾으시오"라는 문구를 보면 1초의 망설임도 없이 BFS를 꺼내들어야 한다.
