2차원 누적 합(2D Prefix Sum) 알고리즘은 1차원 배열에서의 누적 합 개념을 2차원 평면(행렬, 격자)으로 확장한 기법이다.

2차원 배열(Grid)이 주어졌을 때, 특정 직사각형 영역 안의 모든 데이터 합을 구해야 하는 상황에서 사용한다. 매번 2중 반복문을 돌며 값을 더하면 시간 초과가 발생하기 쉬운데, 2차원 누적 합 배열을 미리 한 번만 만들어두면 아무리 넓은 영역의 합이라도 덧셈과 뺄셈 몇 번만으로 **O(1)** 만에 구해낼 수 있다.

### 💡 알고리즘 동작 과정

이 알고리즘의 핵심은 **포함-배제의 원리(Inclusion-Exclusion Principle)**를 활용하여 겹치는 부분을 더하거나 빼주는 것이다.

1. 원본 2차원 배열의 크기보다 가로, 세로가 1칸씩 더 큰 누적 합 배열(`prefix`)을 생성하고 0으로 초기화한다. (계산의 편의를 위해 1-based index를 사용한다.)
2. 원본 배열을 순회하며 `prefix` 배열을 채운다.
현재 위치 `(i, j)`까지의 누적 합은 다음과 같이 구한다.
* `현재 칸의 값 + 위쪽 영역의 누적 합 + 왼쪽 영역의 누적 합 - 왼쪽 위 대각선 영역의 누적 합(중복으로 더해졌으므로)`
* **점화식:** `prefix[i][j] = arr[i-1][j-1] + prefix[i-1][j] + prefix[i][j-1] - prefix[i-1][j-1]`


3. `(x1, y1)`부터 `(x2, y2)`까지의 직사각형 영역의 합을 구하고 싶다면 다음과 같이 계산한다.
* `(x2, y2)까지의 전체 누적 합 - 위쪽 불필요한 영역의 누적 합 - 왼쪽 불필요한 영역의 누적 합 + 왼쪽 위 대각선 영역의 누적 합(중복으로 빼졌으므로)`
* **계산식:** `prefix[x2][y2] - prefix[x1-1][y2] - prefix[x2][y1-1] + prefix[x1-1][y1-1]`



---

### 💻 2차원 누적 합 기본 코드 (Python)

```python
# 3x3 원본 2차원 배열
arr = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
n = 3 # 행의 개수
m = 3 # 열의 개수

# 1. 2차원 누적 합 배열 초기화 (크기를 n+1, m+1로 설정)
prefix = [[0] * (m + 1) for _ in range(n + 1)]

# 2. 누적 합 배열 만들기 (전처리)
for i in range(1, n + 1):
    for j in range(1, m + 1):
        # 원본 배열은 0-based index이므로 arr[i-1][j-1] 참조
        prefix[i][j] = arr[i-1][j-1] + prefix[i-1][j] + prefix[i][j-1] - prefix[i-1][j-1]

# 3. 특정 직사각형 영역의 합 구하기
# 예: (2, 2)부터 (3, 3)까지의 영역 합 구하기 (1-based index 기준)
x1, y1 = 2, 2
x2, y2 = 3, 3

# 포함-배제의 원리를 적용하여 구간 합 계산
result = prefix[x2][y2] - prefix[x1-1][y2] - prefix[x2][y1-1] + prefix[x1-1][y1-1]

print(result) # 결과: 28 (5 + 6 + 8 + 9)

```

### ⏱️ 시간 복잡도

* **전처리 (2차원 누적 합 배열 생성):** **O(N × M)** ($N$: 행의 개수, $M$: 열의 개수)
* **구간 합 쿼리 (합 구하기):** **O(1)**

구간 합을 묻는 질의가 $K$번 주어질 때, 매번 2중 반복문으로 구하면 O(N × M × K)가 걸리지만, 2차원 누적 합을 사용하면 **O(N × M + K)** 만에 모든 처리가 끝난다.

### 📝 예시 문제: 구간 합 구하기 5 (백준 11660번)

N×N개의 수가 N×N 크기의 표에 채워져 있다. (x1, y1)부터 (x2, y2)까지 합을 구하는 프로그램을 작성하시오. (x, y)는 x행 y열을 의미한다.

* **입력:** 첫째 줄에 표의 크기 N과 합을 구해야 하는 횟수 M이 주어진다. (1 $\le$ N $\le$ 1024, 1 $\le$ M $\le$ 100,000) 둘째 줄부터 N개의 줄에는 표에 채워져 있는 수가 1행부터 차례대로 주어진다. 다음 M개의 줄에는 네 개의 정수 x1, y1, x2, y2 가 주어지며, (x1, y1)부터 (x2, y2)의 합을 구해 출력해야 한다.
* **출력:** 총 M줄에 걸쳐 (x1, y1)부터 (x2, y2)까지 합을 구해 출력한다.

---

### 💡 풀이 접근법

가장 대표적인 2차원 누적 합 문제다. N이 최대 1024이므로 표의 전체 데이터는 약 100만 개이고, 질의 횟수 M이 10만 개이므로 브루트 포스(Brute Force) 방식으로는 절대 풀 수 없다.

앞서 설명한 2차원 누적 합 배열 `prefix`를 생성하는 점화식과, 특정 영역의 합을 구하는 계산식을 그대로 적용하면 Python으로도 시간 초과 없이 깔끔하게 통과할 수 있다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

# 표의 크기 n, 질의 횟수 m
n, m = map(int, input().split())

# 원본 2차원 배열 입력받기
arr = [list(map(int, input().split())) for _ in range(n)]

# 누적 합 배열 초기화
prefix = [[0] * (n + 1) for _ in range(n + 1)]

# 1. 2차원 누적 합 계산
for i in range(1, n + 1):
    for j in range(1, n + 1):
        prefix[i][j] = arr[i-1][j-1] + prefix[i-1][j] + prefix[i][j-1] - prefix[i-1][j-1]

# 2. m개의 구간 합 질의 처리
for _ in range(m):
    x1, y1, x2, y2 = map(int, input().split())
    
    # 겹치는 부분은 빼주고, 두 번 빠진 부분은 다시 더해줌
    result = prefix[x2][y2] - prefix[x1-1][y2] - prefix[x2][y1-1] + prefix[x1-1][y1-1]
    
    print(result)
