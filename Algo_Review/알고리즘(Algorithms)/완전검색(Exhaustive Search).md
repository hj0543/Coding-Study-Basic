완전 탐색(Brute Force, 브루트 포스)은 이름 그대로 무식하게(**Brute**) 힘으로(**Force**) 밀어붙여서 **'발생할 수 있는 모든 경우의 수를 전부 다 확인'**하여 정답을 찾아내는 알고리즘 기법이다.

머리를 써서 최적의 지름길을 찾는 다익스트라나 분할 정복과 달리, "모든 길을 다 걸어보면 그중에 정답이 무조건 있다"는 아주 단순하고 무식한 마인드로 접근한다. 100%의 정확도를 보장하지만, 데이터의 개수가 조금만 커져도 시간이 기하급수적으로 늘어나기 때문에 **주어지는 데이터의 크기($N$)가 매우 작을 때만** 사용해야 한다.

### 💡 알고리즘 동작 과정

완전 탐색은 문제의 특성에 따라 경우의 수를 만들어내는 방법이 달라진다. 대표적으로 4가지 방식을 가장 많이 사용한다.

1. **단순 반복문 (for / while):** 가장 기초적인 방법. $N$개의 데이터 중 2~3개를 뽑는 경우처럼 깊이가 얕을 때 다중 반복문을 사용한다.
2. **순열과 조합 (`itertools`):** 앞서 배운 파이썬의 `itertools` 모듈을 사용하여 서로 다른 $N$개에서 $R$개를 뽑아 나열하는 모든 경우의 수를 만든다.
3. **재귀 함수 (Recursion):** 쪼갤 수 있는 부분 문제들을 재귀적으로 호출하여 모든 경우를 탐색한다.
4. **비트마스킹 (Bitmasking):** 데이터가 '포함되거나 / 포함되지 않거나'의 2가지 상태를 가질 때, 이를 2진수 비트 연산으로 치환하여 모든 부분 집합을 빠르게 탐색한다.

코딩 테스트에서 문제를 마주했을 때, **전체 경우의 수를 대략 계산해 보고 그 값이 1,000만~2,000만 번 이하**라면 주저 없이 완전 탐색으로 밀어붙이는 것이 가장 빠르고 확실한 전략이다.

### 💻 완전 탐색 기본 코드 (Python)

가장 직관적이고 많이 쓰이는 '다중 반복문'을 이용해 배열에서 3개의 숫자를 뽑아 더하는 모든 경우를 확인하는 뼈대 코드다.

```python
data = [1, 2, 3, 4, 5]
n = len(data)

target_sum = 8
found = False

# 3개의 숫자를 뽑는 모든 경우의 수 탐색 (3중 for문)
for i in range(n):
    for j in range(i + 1, n):
        for k in range(j + 1, n):
            # 3개 숫자의 합이 목표값과 같은지 확인
            current_sum = data[i] + data[j] + data[k]
            
            if current_sum == target_sum:
                print(f"찾았습니다! {data[i]} + {data[j]} + {data[k]} = {target_sum}")
                found = True

if not found:
    print("조건을 만족하는 경우의 수가 없습니다.")

```

### ⏱️ 시간 복잡도

완전 탐색의 시간 복잡도는 경우의 수를 어떻게 생성하느냐에 따라 천차만별이다.

* 다중 루프: $O(N^2), O(N^3)$
* 비트마스킹 (부분 집합): $O(2^N)$
* 순열: $O(N!)$

경우의 수가 팩토리얼($N!$)이나 지수($2^N$) 단위로 커지는 경우, $N$이 15~20만 넘어가도 우주가 멸망할 때까지 정답을 찾지 못할 수 있으니 데이터 제한을 반드시 확인해야 한다.

### 📝 예시 문제: 블랙잭 (백준 2798번)

카지노에서 즐겨 하는 게임인 블랙잭의 규칙을 변형했다. N장의 카드가 주어졌을 때, 플레이어는 N장의 카드 중에서 3장의 카드를 고르며, 그 합은 M을 넘지 않으면서 M과 최대한 가깝게 만들어야 한다.
N장의 카드에 써져 있는 숫자가 주어졌을 때, M을 넘지 않으면서 M에 최대한 가까운 카드 3장의 합을 구해 출력하시오.

* **입력:** 첫째 줄에 카드의 개수 N ($3 \le N \le 100$)과 M ($10 \le M \le 300,000$)이 주어진다. 둘째 줄에는 카드에 적혀 있는 수가 주어진다.
* **출력:** 첫째 줄에 M을 넘지 않으면서 M에 최대한 가까운 카드 3장의 합을 출력한다.

---

### 💡 풀이 접근법

가장 유명한 완전 탐색 입문 문제다. 카드의 개수 $N$이 최대 100장이므로, 100장 중 3장을 무작위로 뽑는 경우의 수는 조합 기호로 ${}_{100}\mathrm{C}_{3}$ 이다. 이를 계산하면 $\frac{100 \times 99 \times 98}{3 \times 2 \times 1} = 161,700$ 가지밖에 되지 않는다.

파이썬의 연산 속도(초당 약 2,000만 번 이상)를 고려하면 16만 번은 코를 파면서도 1초 안에 거뜬히 해결할 수 있는 횟수다. 따라서 복잡한 알고리즘을 짤 필요 없이 **3중 for문**으로 모든 경우의 수를 순회하며 $M$보다 작거나 같은 최댓값을 갱신해 나가면 된다. (물론 앞서 배운 `itertools.combinations`를 써도 완벽하다.)

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

# 카드의 개수 n, 목표 합 m
n, m = map(int, input().split())
cards = list(map(int, input().split()))

best_sum = 0

# 1. 3중 for문을 이용한 무식한 탐색 (Brute Force)
for i in range(n):
    for j in range(i + 1, n):
        for k in range(j + 1, n):
            current_sum = cards[i] + cards[j] + cards[k]
            
            # 현재 합이 m을 넘지 않으면서, 기존의 최적합보다 크다면 갱신
            if current_sum <= m and current_sum > best_sum:
                best_sum = current_sum

# 최종적으로 가장 m에 가까운 합 출력
print(best_sum)

```

> **꿀팁:** 파이썬의 치트키인 `itertools`를 활용하면 코드가 훨씬 짧아진다.
> ```python
> from itertools import combinations
> best_sum = max([sum(c) for c in combinations(cards, 3) if sum(c) <= m])
> print(best_sum)
> 
> ```

완전 탐색은 알고리즘의 가장 밑바탕이 되는 근본 중의 근본이다. 하지만 데이터가 커지면 무용지물이 된다는 치명적인 단점이 있다.

이 단점을 극복하기 위해, "이 길은 가봤자 어차피 정답이 나올 리가 없어!"라고 판단되는 순간 탐색을 즉시 멈추고(Pruning, 가지치기) 뒤로 돌아가 다른 길을 찾는 스마트한 완전 탐색 기법인 **'백트래킹(Backtracking)'**을 사용한다.
