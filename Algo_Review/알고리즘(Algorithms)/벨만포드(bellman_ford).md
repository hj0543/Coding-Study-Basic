벨만-포드(Bellman-Ford) 알고리즘은 다익스트라와 마찬가지로 **특정한 하나의 출발 노드에서 다른 모든 노드로 가는 최단 경로**를 구하는 알고리즘이다.

다익스트라 알고리즘이 매번 '가장 비용이 적은 노드'를 선택하는 그리디 방식이라 음수 간선을 처리하지 못하는 반면, 벨만-포드 알고리즘은 매 단계마다 **'모든 간선'을 전부 확인**하기 때문에 **음수 가중치(비용이 마이너스인 간선)**가 포함된 그래프에서도 완벽하게 최단 거리를 구해낸다. 더 나아가 뺑뺑이를 돌수록 비용이 무한히 작아지는 **'음수 사이클(Negative Cycle)'**의 존재 여부까지 판별할 수 있는 강력한 무기다.

### 💡 알고리즘 동작 과정

1. 출발 노드를 설정한다.
2. 최단 거리 테이블을 모두 무한대(Infinity)로 초기화하고, 출발 노드 자신에 대한 최단 거리는 0으로 설정한다.
3. 전체 간선 $E$개를 하나씩 모두 확인하며, 각 간선을 거쳐 다른 노드로 가는 비용을 계산해 최단 거리 테이블을 갱신한다.
4. 3번의 과정을 **전체 노드 개수 - 1**번 반복한다. (최단 경로는 최대 $V-1$개의 간선으로 이루어지기 때문이다.)
5. **음수 사이클 확인:** 3번의 과정을 한 번 더 수행한다. 만약 이때도 최단 거리 테이블이 갱신된다면, 이는 무한히 거리가 줄어드는 음수 사이클이 존재한다는 뜻이다.

### 💻 벨만-포드 알고리즘 기본 코드 (Python)

모든 노드와 간선을 반복문을 통해 직접 확인하는 직관적인 코드다.

```python
import sys
INF = int(1e9) # 무한을 의미하는 값으로 10억을 설정

# 노드의 개수 v, 간선의 개수 e
v = 3
e = 4
edges = [] # 모든 간선에 대한 정보를 담는 리스트
distance = [INF] * (v + 1) # 최단 거리 테이블 초기화

# 간선 정보 입력 (예시 생략)
# for _ in range(e):
#     a, b, c = map(int, input().split())
#     edges.append((a, b, c))

def bellman_ford(start):
    # 시작 노드 초기화
    distance[start] = 0
    
    # 전체 v번의 라운드(round)를 반복
    for i in range(v):
        # 매 반복마다 모든 간선을 확인
        for j in range(e):
            cur_node = edges[j][0]
            next_node = edges[j][1]
            cost = edges[j][2]
            
            # 현재 간선을 거쳐서 다른 노드로 이동하는 거리가 더 짧은 경우
            if distance[cur_node] != INF and distance[cur_node] + cost < distance[next_node]:
                distance[next_node] = distance[cur_node] + cost
                
                # 만약 v번째 라운드에서도 값이 갱신된다면 음수 사이클이 존재하는 것
                if i == v - 1:
                    return True # 음수 사이클 존재
                    
    return False # 정상 종료 (음수 사이클 없음)

```

### ⏱️ 시간 복잡도

벨만-포드 알고리즘의 시간 복잡도는 **$O(V \times E)$** 이다. ($V$: 노드의 개수, $E$: 간선의 개수)
다익스트라 알고리즘($O(E \log V)$)에 비하면 모든 간선을 매번 다 뒤져봐야 하므로 속도가 꽤 느린 편이다. 따라서 문제에 **음수 간선이 존재할 때만** 제한적으로 사용하는 것이 현명하다.

### 📝 예시 문제: 타임머신 (백준 11657번)

N개의 도시가 있다. 그리고 한 도시에서 출발하여 다른 도시에 도착하는 버스가 M개 있다. 각 버스는 A, B, C로 나타낼 수 있는데, A는 시작도시, B는 도착도시, C는 버스를 타고 이동하는데 걸리는 시간이다. 시간 C가 양수가 아닌 경우가 있다. C = 0인 경우는 순간 이동을 하는 경우, C < 0인 경우는 타임머신으로 시간을 되돌아가는 경우이다.
1번 도시에서 출발해서 나머지 도시로 가는 가장 빠른 시간을 구하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에 도시의 개수 N ($1 \le N \le 500$), 버스 노선의 개수 M ($1 \le M \le 6,000$)이 주어진다. 둘째 줄부터 M개의 줄에는 버스 노선의 정보 A, B, C가 주어진다. ($-10,000 \le C \le 10,000$)
* **출력:** 만약 1번 도시에서 출발해 어떤 도시로 가는 과정에서 시간을 무한히 오래 전으로 되돌릴 수 있다면 첫째 줄에 -1을 출력한다. 그렇지 않다면 N-1개 줄에 걸쳐 각 줄에 2번 도시, 3번 도시, $\dots$, N번 도시로 가는 가장 빠른 시간을 순서대로 출력한다. (도달할 수 없다면 -1 출력)

### 💡 풀이 접근법

문제 설명에 대놓고 '시간(비용)이 마이너스가 될 수 있다', '시간을 무한히 오래 전으로 되돌릴 수 있는지(음수 사이클) 판별하라'고 적혀 있는 **벨만-포드 알고리즘의 가장 완벽한 교보재 문제**다.

* 1번 도시를 시작점으로 하여 벨만-포드 함수를 실행한다.
* 함수가 `True`(음수 사이클 존재)를 반환하면 무조건 -1을 출력하고 종료한다.
* 정상적으로 종료되었다면, 2번 노드부터 차례대로 최단 거리를 출력하되, 값이 여전히 `INF`라면 도달할 수 없는 곳이므로 -1을 출력한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline
INF = int(1e9)

n, m = map(int, input().split())
edges = []
distance = [INF] * (n + 1)

# 간선 정보 입력
for _ in range(m):
    a, b, c = map(int, input().split())
    edges.append((a, b, c))

def bellman_ford(start):
    distance[start] = 0
    
    # n번의 라운드 반복 (n-1번 최단거리 갱신 + 1번 음수 사이클 확인)
    for i in range(n):
        for j in range(m):
            cur, nxt, cost = edges[j]
            
            # 현재 노드를 거쳐서 다음 노드로 가는 게 더 빠르다면 갱신
            if distance[cur] != INF and distance[cur] + cost < distance[nxt]:
                distance[nxt] = distance[cur] + cost
                
                # n번째 라운드에서도 갱신된다면 음수 사이클 존재
                if i == n - 1:
                    return True
    return False

# 1번 도시에서 출발
negative_cycle = bellman_ford(1)

# 1. 음수 사이클이 존재하는 경우
if negative_cycle:
    print("-1")
# 2. 정상적으로 최단 거리를 구한 경우
else:
    # 2번 도시부터 n번 도시까지 출력
    for i in range(2, n + 1):
        if distance[i] == INF:
            print("-1") # 도달 불가능
        else:
            print(distance[i])

```
