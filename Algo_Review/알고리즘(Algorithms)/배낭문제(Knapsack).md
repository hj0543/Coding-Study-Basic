배낭 문제(Knapsack Problem)는 다이나믹 프로그래밍(DP)의 가장 전형적이면서도 중요한 문제다.

제한된 무게를 견딜 수 있는 배낭이 있고, 각기 다른 무게와 가치를 가진 물건들이 있을 때, **배낭의 무게 제한을 넘지 않으면서 담을 수 있는 물건들의 가치 합을 최대화**하는 것이 목표다. 물건을 쪼갤 수 없는 경우를 **0/1 배낭 문제(0/1 Knapsack Problem)**라고 하며, 주로 DP를 이용해 해결한다.

### 💡 알고리즘 동작 과정

1. 배낭의 최대 무게 $K$와 물건의 개수 $N$을 기준으로 $(N+1) \times (K+1)$ 크기의 2차원 `dp` 테이블을 만든다.
2. `dp[i][j]`는 **"첫 번째부터 $i$번째 물건까지 고려했을 때, 무게 한도가 $j$인 배낭에 담을 수 있는 최대 가치"**를 의미한다.
3. 물건을 하나씩 확인하며 테이블을 채운다.
* **현재 물건의 무게가 현재 배낭 한도 $j$보다 크다면:** 이 물건은 담을 수 없으므로, 이전 물건까지 고려했을 때의 가치를 그대로 가져온다.
* `dp[i][j] = dp[i-1][j]`


* **현재 물건을 담을 수 있다면:** 다음 두 가지 중 더 큰 값을 선택한다.
1. 물건을 담지 않는 경우: `dp[i-1][j]`
2. 물건을 담는 경우: `현재 물건 가치 + (현재 무게 - 물건 무게) 한도에서의 이전 최대 가치`


* `dp[i][j] = max(dp[i-1][j], 가치 + dp[i-1][j - 무게])`




4. 테이블의 가장 오른쪽 아래 값인 `dp[N][K]`가 정답이 된다.

---

### 💻 배낭 문제 기본 코드 (Python)

2차원 배열을 이용한 가장 표준적인 구현 방식이다.

```python
import sys

n, k = map(int, sys.stdin.readline().split()) # 물건 개수, 가방 한도
items = [[0, 0]] # 인덱스를 맞추기 위해 빈 데이터 삽입
for _ in range(n):
    items.append(list(map(int, sys.stdin.readline().split()))) # [무게, 가치]

# dp[물건번호][무게한도]
dp = [[0] * (k + 1) for _ in range(n + 1)]

for i in range(1, n + 1):
    w, v = items[i] # 현재 물건의 무게와 가치
    for j in range(1, k + 1):
        if j < w:
            # 현재 배낭 한도가 물건 무게보다 작으면 못 담음
            dp[i][j] = dp[i - 1][j]
        else:
            # 담지 않는 경우 vs 담는 경우 중 최댓값 선택
            dp[i][j] = max(dp[i - 1][j], v + dp[i - 1][j - w])

print(dp[n][k])

```

> **꿀팁:** 사실 2차원 배열 대신 1차원 배열만 써서 메모리를 아낄 수도 있다. 이때는 무게(`j`)를 뒤에서부터(`K`에서 `w`까지) 거꾸로 계산해야 이전 물건의 데이터가 덮어씌워지지 않는다.

---

### ⏱️ 시간 복잡도

배낭 문제의 시간 복잡도는 **$O(N \times K)$** 이다. ($N$: 물건의 개수, $K$: 배낭의 최대 무게)
물건마다 배낭의 모든 무게를 확인하므로, $N$과 $K$의 범위가 적절할 때(보통 $N \le 100, K \le 100,000$ 정도) 사용할 수 있다.

### 📝 예시 문제: 평범한 배낭 (백준 12865번)

이 문제는 아주 전형적인 0/1 배낭 문제다. $N$개의 물건이 있고 각각 무게 $W$와 가치 $V$를 가진다. 최대 $K$만큼의 무게를 넣을 수 있는 배낭에 넣을 수 있는 물건들의 가치의 최댓값을 구하라.

* **입력:** 물건의 개수 $N(1 \le N \le 100)$, 배낭의 무게 한도 $K(1 \le K \le 100,000)$.
* **출력:** 배낭에 넣을 수 있는 물건들의 가치 합의 최댓값.

---

### 💡 풀이 접근법

물건을 쪼개서 넣을 수 없으므로 0/1 배낭 문제 풀이법을 그대로 적용한다.

* $N$이 100이고 $K$가 10만이므로 전체 연산 횟수는 1,000만 번 정도다. 파이썬의 초당 연산 횟수 내에 충분히 들어오므로 DP 테이블을 생성하여 해결할 수 있다.
* 물건을 한 번씩만 사용할 수 있으므로 위에서 설명한 `max(dp[i-1][j], v + dp[i-1][j-w])` 로직을 사용한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
# 1차원 배열로 최적화된 풀이
dp = [0] * (k + 1)

for _ in range(n):
    w, v = map(int, input().split())
    # 1차원 배열 사용 시 무게를 거꾸로 순회해야 함
    for j in range(k, w - 1, -1):
        dp[j] = max(dp[j], dp[j - w] + v)

print(dp[k])

```
