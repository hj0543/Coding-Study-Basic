트리 DP(Tree DP)는 이름 그대로 **'트리(Tree) 자료구조'** 위에서 **'다이나믹 프로그래밍(DP)'**을 수행하는 매우 우아한 알고리즘 기법이다.

일반적인 1차원 배열이나 2차원 격자에서 점화식을 세우는 것을 넘어, 트리의 **부모-자식 관계**를 활용하여 부분 문제(Subproblem)의 해를 구하고 이를 이용해 전체 트리의 해답을 도출한다. 부모 노드의 상태가 자식 노드들의 상태에 의해 결정되는 구조일 때 완벽하게 들어맞는다.

### 💡 알고리즘 동작 과정

트리 DP의 핵심은 **'깊이 우선 탐색(DFS)'을 이용한 후위 순회(Post-order Traversal)**에 있다. 자식 노드의 값을 알아야 부모 노드의 값을 계산할 수 있기 때문에, 트리의 맨 밑바닥(단말 노드, Leaf)까지 먼저 내려간 다음 정답을 가지고 위로 올라오면서(Bottom-up) DP 테이블을 갱신한다.

1. 1차원 또는 2차원 형태의 `dp` 배열을 생성한다.
* 상태가 여러 개라면 2차원으로 만든다. (예: `dp[현재 노드][현재 노드의 상태]`)


2. 임의의 루트 노드(보통 1번)에서부터 DFS 탐색을 시작한다.
3. DFS를 통해 현재 노드와 연결된 자식 노드들로 재귀적으로 파고든다. (이때 무한 루프를 방지하기 위해 '방문 처리'를 하거나 '부모 노드가 아닌 경우'에만 파고들게 한다.)
4. **단말 노드(Leaf)에 도달하면:** 기저 조건(Base Case)에 해당하는 값을 `dp` 배열에 기록하고 반환한다.
5. **자식 노드들의 탐색이 끝났다면 (DFS 복귀 시점):** 자식들의 `dp` 값을 취합하여 현재 부모 노드의 `dp` 점화식을 계산하고 갱신한다.
6. 루트 노드의 탐색이 완전히 종료되면, 루트 노드의 `dp` 값이 전체 트리의 정답이 된다.

### 💻 트리 DP 기본 코드 (Python)

부모가 자식들의 값을 합산하는 가장 기초적인 형태의 뼈대 코드다.

```python
import sys
sys.setrecursionlimit(10**6)

# 노드 개수 n
n = 5
graph = [[] for _ in range(n + 1)]
visited = [False] * (n + 1)

# dp[i] : i번 노드를 루트로 하는 서브트리의 크기 (또는 특정 가치)
dp = [0] * (n + 1)

# 트리 연결 정보 입력 (예시 생략)
# graph[u].append(v)
# graph[v].append(u)

def dfs(now):
    visited[now] = True
    dp[now] = 1 # 자기 자신도 서브트리에 포함되므로 기본값 1
    
    # 연결된 노드들(자식 노드들) 탐색
    for nxt in graph[now]:
        # 아직 방문하지 않은 노드라면 자식 노드임
        if not visited[nxt]:
            dfs(nxt) # 자식 노드의 끝까지 먼저 파고듦 (Bottom-up 준비)
            
            # 자식 노드의 탐색이 끝난 후, 자식의 결과를 부모에게 더해줌
            dp[now] += dp[nxt]

# 1번 노드를 루트로 삼고 탐색 시작
dfs(1)

# 결과: 1번 노드(루트)의 dp 값이 전체 트리의 크기가 됨
print(dp[1])

```

### ⏱️ 시간 복잡도

트리 DP의 시간 복잡도는 **$O(V)$** 이다. ($V$: 노드의 개수)
트리의 특성상 간선의 개수 $E$는 항상 $V - 1$개이며, DFS를 통해 모든 노드를 정확히 한 번씩만 방문하고 DP 테이블을 갱신하기 때문에 선형 시간 안에 문제를 해결할 수 있다.

### 📝 예시 문제: 사회망 서비스(SNS) (백준 2533번)

친구 관계 트리가 주어졌을 때, 모든 사람이 새로운 아이디어를 수용하기 위해 필요한 최소 '얼리 아답터(Early Adopter)'의 수를 구하는 프로그램을 작성하시오.

* **규칙:** 얼리 아답터가 아닌 사람들은 자신의 모든 친구들이 얼리 아답터일 때만 이 아이디어를 받아들인다.
* **입력:** 첫 번째 줄에는 친구 관계 트리의 정점 개수 $N$ ($2 \le N \le 1,000,000$)이 주어진다. 다음 $N-1$개의 줄에는 각 줄마다 친구 관계 트리의 에지 $(u, v)$를 나타내는 두 정수 $u$와 $v$가 주어진다.
* **출력:** 주어진 친구 관계 그래프에서 아이디어를 전파하는 데 필요한 얼리 아답터의 최소 수를 하나의 정수로 출력한다.

---

### 💡 풀이 접근법

트리 DP의 상태(State)를 2차원 배열로 나누어 푸는 아주 클래식한 명작 문제다.

모든 사람이 아이디어를 받아들이려면 부모-자식 간에 다음과 같은 규칙이 성립해야 한다.

1. **현재 노드가 '얼리 아답터가 아니라면':** 연결된 모든 자식 노드는 **무조건 얼리 아답터여야만** 한다.
2. **현재 노드가 '얼리 아답터라면':** 자식 노드는 얼리 아답터이든 아니든 상관없다. 따라서 **최소 인원수를 위해 둘 중 더 작은 값을 선택**하면 된다.

이 논리를 바탕으로 DP 테이블을 `dp[노드번호][얼리 아답터 여부]`로 구성한다.

* `dp[now][0]`: 현재 노드가 얼리 아답터가 아닐 때의 최소 얼리 아답터 수
* `dp[now][1]`: 현재 노드가 얼리 아답터일 때의 최소 얼리 아답터 수 (자기 자신이 포함되므로 +1의 비용이 든다)

---

### 💻 풀이 코드 (Python)

```python
import sys
sys.setrecursionlimit(10**6)
input = sys.stdin.readline

n = int(input())
graph = [[] for _ in range(n + 1)]

# 트리 간선 정보 입력
for _ in range(n - 1):
    u, v = map(int, input().split())
    graph[u].append(v)
    graph[v].append(u)

# dp[i][0] : i번 노드가 얼리 아답터가 아닐 때의 최소 인원
# dp[i][1] : i번 노드가 얼리 아답터일 때의 최소 인원
dp = [[0, 0] for _ in range(n + 1)]
visited = [False] * (n + 1)

def dfs(now):
    visited[now] = True
    dp[now][0] = 0 # 본인이 얼리 아답터가 아니므로 현재로썬 0명
    dp[now][1] = 1 # 본인이 얼리 아답터이므로 일단 1명 카운트
    
    for nxt in graph[now]:
        if not visited[nxt]:
            dfs(nxt) # 자식 노드 끝까지 탐색 (후위 순회)
            
            # 자식 노드 탐색 완료 후 (Bottom-up)
            # 1. 내가 얼리 아답터가 아니면, 자식은 무조건 얼리 아답터여야 함
            dp[now][0] += dp[nxt][1]
            
            # 2. 내가 얼리 아답터면, 자식은 어느 쪽이든 상관없으므로 인원이 더 적은 쪽을 택함
            dp[now][1] += min(dp[nxt][0], dp[nxt][1])

# 임의의 루트(1번 노드)에서 탐색 시작
dfs(1)

# 루트 노드가 얼리 아답터인 경우와 아닌 경우 중 최솟값이 전체 정답
print(min(dp[1][0], dp[1][1]))

```
