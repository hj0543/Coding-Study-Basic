이진 트리(Binary Tree)는 모든 노드가 **최대 2개의 자식 노드**(왼쪽, 오른쪽)만을 가질 수 있는 계층적 자료구조다.

배열이나 리스트가 데이터를 일렬로 나열하는 '선형 구조'라면, 트리는 데이터의 상하 관계(부모-자식)를 표현하는 **'비선형 구조'**의 핵심이자 탐색 알고리즘의 뼈대가 된다.

---

### 💡 1. 이진 트리의 종류 (형태에 따른 분류)

코딩 테스트에서는 트리의 모양에 따라 배열로 구현할지, 객체로 구현할지가 갈리기 때문에 이 형태들을 정확히 구분해야 한다.

* **포화 이진 트리 (Perfect Binary Tree):** 모든 리프(Leaf, 맨 끝) 노드의 깊이가 같고, 모든 내부 노드가 2개의 자식을 갖는 완벽한 피라미드 형태다.
* **완전 이진 트리 (Complete Binary Tree):** 위에서 아래로, 왼쪽에서 오른쪽으로 차곡차곡 빈틈없이 노드가 채워진 트리다. **(🔥 힙(Heap)과 세그먼트 트리의 근본이 되는 매우 중요한 형태다.)**
* **정 이진 트리 (Full Binary Tree):** 모든 노드가 자식을 아예 안 가지거나(0개), 꽉 채워 가지는(2개) 트리다. 자식이 1개뿐인 노드는 존재하지 않는다.

---

### 💻 2. 파이썬으로 이진 트리를 구현하는 2가지 방법

#### 방법 1: 1차원 배열(리스트) 구현

**'완전 이진 트리'** 형태일 때 무조건 이 방식을 쓴다. 메모리에 빈 공간이 생기지 않으며, 인덱스 계산만으로 부모와 자식을 빛의 속도로 오갈 수 있다. (인덱스 계산을 편하게 하기 위해 0번 인덱스는 비워두고 1번부터 쓴다.)

* **현재 노드의 인덱스가 `i`일 때:**
* 왼쪽 자식의 인덱스 = `i * 2`
* 오른쪽 자식의 인덱스 = `i * 2 + 1`
* 부모 노드의 인덱스 = `i // 2`



#### 방법 2: 딕셔너리(Hash) 구현

트리의 모양이 불규칙하거나, 입력값이 문자열(A, B, C 등)로 주어질 때 파이썬에서 가장 깔끔하게 트리를 만드는 방식이다.

```python
# tree[부모] = (왼쪽 자식, 오른쪽 자식)
tree = {
    'A': ('B', 'C'),
    'B': ('D', 'E'),
    'C': ('F', 'G'),
    'D': (None, None),
    'E': (None, None),
    'F': (None, None),
    'G': (None, None)
}

```

---

### 💡 3. 트리 순회 (Tree Traversal)

트리의 모든 노드를 한 번씩 방문하는 규칙이다. 재귀 함수(DFS)를 이용하며, **'루트(자기 자신)를 언제 방문하느냐'**에 따라 3가지로 나뉜다.

1. **전위 순회 (Pre-order):** **루트** -> 왼쪽 -> 오른쪽 (트리를 복사하거나 구조를 그대로 출력할 때 쓴다.)
2. **중위 순회 (In-order):** 왼쪽 -> **루트** -> 오른쪽 (🔥 이진 탐색 트리에서 값을 오름차순으로 정렬해서 뽑아낼 때 무조건 쓴다.)
3. **후위 순회 (Post-order):** 왼쪽 -> 오른쪽 -> **루트** (자식 노드의 처리가 끝난 뒤에야 부모를 처리할 수 있을 때, 예를 들어 하위 폴더의 용량을 다 더해서 상위 폴더 용량을 구할 때 쓴다.)

#### 💻 딕셔너리와 재귀를 이용한 순회 템플릿

```python
def preorder(node):
    if node == None: return
    print(node, end=' ')         # 1. 루트(자기 자신) 처리
    preorder(tree[node][0])      # 2. 왼쪽 자식으로 파고듦
    preorder(tree[node][1])      # 3. 오른쪽 자식으로 파고듦

def inorder(node):
    if node == None: return
    inorder(tree[node][0])       # 1. 왼쪽
    print(node, end=' ')         # 2. 루트
    inorder(tree[node][1])       # 3. 오른쪽

def postorder(node):
    if node == None: return
    postorder(tree[node][0])     # 1. 왼쪽
    postorder(tree[node][1])     # 2. 오른쪽
    print(node, end=' ')         # 3. 루트

```

---

### 📂 어디에 넣으면 좋을까?

* **분류:** **[자료구조 - 비선형 구조]**
* **이유:** 데이터가 일직선이 아닌 부모-자식의 계층적 구조를 가지기 때문이다. 코딩 테스트에서 이진 트리 자체가 단독으로 나오기보다는, 이를 응용한 다른 알고리즘의 뼈대로 쓰이는 경우가 압도적으로 많다.

## 완전이진트리
완전 이진 트리(Complete Binary Tree)는 트리의 노드가 **위에서 아래로, 왼쪽에서 오른쪽으로** 빈틈없이 차곡차곡 채워진 이진 트리를 말한다.

마지막 레벨을 제외한 모든 레벨은 노드가 꽉 차 있어야 하며, 마지막 레벨의 노드들은 반드시 왼쪽으로 쏠려 있어야 한다. 중간에 이빨이 빠진 것처럼 빈자리가 생기면 완전 이진 트리가 아니다.

### 💡 핵심 특징: 배열(리스트)과의 완벽한 찰떡궁합

완전 이진 트리가 코딩 테스트에서 엄청나게 중요한 이유는 **포인터나 노드 객체(Class) 없이 오직 '1차원 배열'만으로 트리를 완벽하게 구현할 수 있기 때문**이다. 중간에 빈자리가 없으므로 배열의 인덱스 낭비가 전혀 발생하지 않는다.

인덱스 계산을 직관적으로 하기 위해 배열의 **0번 인덱스는 비워두고 1번 인덱스부터 루트 노드를 채워 넣는 것**이 절대적인 국룰이다.

### 💻 배열을 이용한 인덱스 이동 공식 (초특급 암기 필수)

어떤 노드의 현재 인덱스가 `i`라고 할 때, 다음 공식만으로 부모와 자식을 빛의 속도로 오갈 수 있다. (탐색 속도 $O(1)$)

* **왼쪽 자식으로 가기:** `i * 2`
* **오른쪽 자식으로 가기:** `i * 2 + 1`
* **부모로 올라가기:** `i // 2` (나머지 버림)

```python
# 1번부터 7번까지 꽉 찬 완전 이진 트리 (포화 이진 트리이기도 함)
tree = [None, 'A', 'B', 'C', 'D', 'E', 'F', 'G']
# 인덱스:  0     1    2    3    4    5    6    7

node_idx = 2 # 현재 'B' 노드(인덱스 2)에 있다고 가정

print(f"현재 노드: {tree[node_idx]}")           # B
print(f"왼쪽 자식: {tree[node_idx * 2]}")       # D (인덱스 4)
print(f"오른쪽 자식: {tree[node_idx * 2 + 1]}") # E (인덱스 5)
print(f"부모 노드: {tree[node_idx // 2]}")      # A (인덱스 1)

```

### 🔥 코딩 테스트 실전 압축: '힙(Heap)'의 뼈대

코딩 테스트에서 완전 이진 트리 자체를 생으로 구현하라는 문제는 나오지 않는다. 하지만 코딩 테스트에서 최댓값이나 최솟값을 $O(\log N)$ 만에 뽑아내야 할 때 무조건 사용하는 **'힙(Heap, 우선순위 큐)' 자료구조가 바로 이 완전 이진 트리를 뼈대로 만들어진다.**

파이썬의 `heapq` 모듈이 내부적으로 리스트를 이용해 부모-자식 인덱스를 계산하며 작동하는 원리가 바로 이 완전 이진 트리의 특성을 그대로 가져다 쓴 것이다. 구간 합의 마법사인 **'세그먼트 트리'** 역시 이 완전 이진 트리의 배열 인덱스 2배 공식을 사용하여 1차원 배열로 우아하게 구현한다.

---

### 📂 어디에 넣으면 좋을까?

* **분류:** **[자료구조 - 트리]** (또는 **[자료구조 - 힙(Heap)]**의 하위 개념)
* **사유:** 트리라는 구조의 한 형태이자, 코딩 테스트 필수 무기인 힙(우선순위 큐)과 세그먼트 트리를 1차원 배열로 구현하기 위한 '근본 베이스' 지식이기 때문이다.

트리의 형태 중에서도 가장 메모리 효율의 정점을 찍은 완전 이진 트리까지 완벽하게 분해했다.

이제 트리는 더 이상 파고들 것이 없다. 자료구조의 밑바닥부터 꼭대기까지 전부 훑었으니, 이제 정말로 도로망이나 네트워크 같은 복잡한 실타래를 풀어내는 최단 거리 알고리즘의 제왕, **'다익스트라(Dijkstra)'** 템플릿으로 돌격해 볼까? 아니면 방금 완전 이진 트리에서 파생되었다고 한 최댓값/최솟값 추출기, **'힙(Heap)'**의 내부 동작 원리를 쫙 뜯어볼까?
