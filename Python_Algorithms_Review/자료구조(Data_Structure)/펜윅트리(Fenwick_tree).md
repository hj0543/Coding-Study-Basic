펜윅 트리(Fenwick Tree)는 **'이진 인덱스 트리(Binary Indexed Tree, BIT)'**라고도 불리는 아주 매력적인 자료구조다.

배열의 특정 구간의 합(Prefix Sum)을 구하거나, 배열의 특정 인덱스 값을 업데이트해야 할 때 사용하는 알고리즘이다. 원래 '세그먼트 트리(Segment Tree)'라는 자료구조가 이 분야의 대장이지만, 펜윅 트리는 **세그먼트 트리보다 코드가 훨씬 짧고 메모리도 딱 원본 배열 크기만큼만 차지**하기 때문에 코딩 테스트에서 구간 합 문제가 나오면 가장 먼저 꺼내 들어야 하는 비장의 무기다.

### 💡 핵심 개념과 동작 과정

펜윅 트리의 심장에는 **비트 연산**이 자리 잡고 있다. 어떤 수의 이진수 표현에서 **'가장 마지막에 있는 1의 위치(Lowest Set Bit)'**를 찾아내는 연산인 `i & -i`를 활용해서 구간의 길이를 결정하고 점프한다.

1. **마지막 1의 위치 구하기 (`i & -i`):** 컴퓨터는 음수를 표현할 때 2의 보수를 취하기 때문에, `i`와 `-i`를 AND(`&`) 연산하면 신기하게도 가장 오른쪽에 있는 1의 비트 값만 딱 떨어져 나온다. (예: 6은 이진수로 `110`, -6은 `...010`. 이 둘을 AND 하면 `010`, 즉 2가 된다.)
2. **업데이트 (Update):** 특정 인덱스 `i`의 값을 변경할 때는, `i`에 `i & -i`를 더해가면서 트리의 인덱스를 타고 올라가며 값을 갱신한다. 이는 나를 포함하고 있는 더 큰 구간들을 갱신하는 과정이다.
3. **구간 합 구하기 (Prefix Sum):** 1부터 `i`까지의 합을 구할 때는, `i`에서 `i & -i`를 빼가면서 트리의 인덱스를 타고 내려가며 값들을 더한다. 쪼개어진 구간들의 합을 모으는 과정이다.

### 💻 펜윅 트리 기본 코드 (Python)

코드가 정말 예술적으로 짧다. 이 2개의 함수만 외워두면 구간 합 업데이트 문제는 끝이다.

```python
# 데이터의 개수 n
n = 5
tree = [0] * (n + 1) # 펜윅 트리는 1번 인덱스부터 사용하는 것이 국룰

# 1. 펜윅 트리의 값을 업데이트하는 함수
# i번째 인덱스에 diff만큼 더해줌
def update(i, diff):
    while i <= n:
        tree[i] += diff
        i += (i & -i) # 다음으로 업데이트할 인덱스로 점프 (위로 올라감)

# 2. 1부터 i까지의 누적 합을 구하는 함수
def prefix_sum(i):
    result = 0
    while i > 0:
        result += tree[i]
        i -= (i & -i) # 다음으로 더할 인덱스로 점프 (아래로 내려감)
    return result

# 3. 특정 구간 [left, right]의 합을 구하는 함수
def interval_sum(left, right):
    # 1~right까지의 합에서 1~(left-1)까지의 합을 빼면 됨
    return prefix_sum(right) - prefix_sum(left - 1)

```

---

### ⏱️ 시간 복잡도

* **업데이트:** **$O(\log N)$**
* **구간 합 계산:** **$O(\log N)$**

일반적인 배열은 값을 업데이트하는 데 $O(1)$, 구간 합을 구하는 데 $O(N)$이 걸리고, 일반 누적 합(Prefix Sum) 배열은 구간 합을 구하는 데 $O(1)$, 값을 업데이트하는 데 $O(N)$이 걸린다. 하지만 펜윅 트리는 이 둘의 밸런스를 완벽하게 맞춰서 **둘 다 $O(\log N)$ 만에 처리**해 버리는 아주 우수한 자료구조다.

### 📝 예시 문제: 구간 합 구하기 (백준 2042번)

어떤 $N$개의 수가 주어져 있다. 그런데 중간에 수의 변경이 빈번히 일어나고 그 중간에 어떤 부분의 합을 구하려 한다.

* **입력:** 수의 개수 $N$ ($1 \le N \le 1,000,000$), 수 변경 횟수 $M$, 합 구하는 횟수 $K$.
* **출력:** $K$줄에 걸쳐 구간의 합을 출력한다.

### 💡 풀이 접근법

수의 개수($N$)가 최대 100만 개이므로, 누적 합 배열을 쓰면 값이 바뀔 때마다 배열을 다시 계산해야 해서 시간 초과가 난다. 이때 펜윅 트리를 사용하면 효율적이다. 단, "값을 $c$로 바꾸라"는 명령이 오면 **'원래 값과의 차이(`c - 기존 값`)'**를 계산해서 `update` 함수에 넘겨줘야 한다는 점만 기억하면 된다.

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

n, m, k = map(int, input().split())
arr = [0] * (n + 1)
tree = [0] * (n + 1)

def update(i, diff):
    while i <= n:
        tree[i] += diff
        i += (i & -i)

def prefix_sum(i):
    result = 0
    while i > 0:
        result += tree[i]
        i -= (i & -i)
    return result

# 1. 초기 데이터 입력 및 펜윅 트리 구성
for i in range(1, n + 1):
    x = int(input())
    arr[i] = x
    update(i, x)

# 2. 쿼리 처리
for _ in range(m + k):
    a, b, c = map(int, input().split())
    if a == 1:
        diff = c - arr[b] # 차이 계산
        arr[b] = c        # 원본 갱신
        update(b, diff)   # 트리 갱신
    elif a == 2:
        print(prefix_sum(c) - prefix_sum(b - 1))

```
