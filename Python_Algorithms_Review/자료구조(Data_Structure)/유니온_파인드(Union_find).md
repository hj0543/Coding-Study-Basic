유니온 파인드는 여러 노드가 있을 때 **'두 노드가 같은 집합(그래프)에 속해 있는지'** 확인하고(`Find`), **'두 집합을 하나로 합치는'**(`Union`) 데 특화된 아주 빠른 자료구조다. '경로 압축(Path Compression)'을 쓰면 사실상 상수 시간인 $O(1)$ 만에 동작하는 마법 같은 녀석이다.

기왕 유니온 파인드가 다시 나온 김에, 이 유니온 파인드를 가장 맛있게 써먹는 그래프의 핵심 알고리즘, **'크루스칼(Kruskal) 알고리즘'**을 이어서 정리하겠다.

---

### 💡 크루스칼 알고리즘 (최소 신장 트리, MST)

크루스칼 알고리즘은 주어진 그래프의 모든 노드를 **'가장 적은 비용(최소 가중치)'**으로 싹 다 연결하고 싶을 때 사용하는 알고리즘이다. 이렇게 완성된 트리를 **최소 신장 트리(Minimum Spanning Tree)**라고 부른다. (참고로 트리는 사이클이 없어야 한다!)

이때 간선을 연결하면서 **'사이클(순환)이 발생하는지 안 하는지'** 판단하는 핵심 부품으로 바로 **유니온 파인드**가 쓰인다.

#### 🛠️ 동작 과정

1. 그래프의 모든 간선 정보를 **'비용(가중치)'을 기준으로 오름차순 정렬**한다.
2. 비용이 가장 적은 간선부터 하나씩 확인한다.
3. 간선이 잇는 두 노드에 대해 유니온 파인드의 `Find` 연산을 수행하여 루트 노드를 확인한다.
* **루트 노드가 다르다면 (사이클이 안 생긴다면):** `Union` 연산으로 두 노드를 연결하고 전체 비용에 더한다.
* **루트 노드가 같다면 (이미 연결되어 있어 사이클이 생긴다면):** 무시하고 다음 간선으로 넘어간다.


4. 모든 간선에 대해 3번을 반복하면 최소 비용으로 모든 노드가 연결된다!

### 💻 크루스칼 알고리즘 기본 코드 (Python)

앞서 배웠던 유니온 파인드 함수(`find_parent`, `union_parent`)를 그대로 가져와서 쓴다는 걸 명심하자.

```python
# --- 유니온 파인드 핵심 함수 ---
def find_parent(parent, x):
    if parent[x] != x:
        parent[x] = find_parent(parent, parent[x]) # 경로 압축
    return parent[x]

def union_parent(parent, a, b):
    a = find_parent(parent, a)
    b = find_parent(parent, b)
    if a < b:
        parent[b] = a
    else:
        parent[a] = b
# -------------------------------

v, e = 7, 9 # 노드 개수, 간선 개수
parent = [0] * (v + 1) # 부모 테이블 초기화

# 자기 자신을 부모로 갖도록 초기화
for i in range(1, v + 1):
    parent[i] = i

edges = [] # 모든 간선을 담을 리스트
total_cost = 0 # 최종 최소 비용

# 간선 정보 (비용, 노드 A, 노드 B)
edges = [
    (29, 1, 2), (75, 1, 5), (35, 2, 3), (34, 2, 6), 
    (7, 3, 4), (23, 4, 6), (13, 4, 7), (53, 5, 6), (25, 6, 7)
]

# 1. 간선을 비용 순으로 오름차순 정렬
edges.sort()

# 2. 간선을 하나씩 확인하며 사이클 판별
for edge in edges:
    cost, a, b = edge
    
    # 사이클이 발생하지 않는 경우에만 집합에 포함(Union)
    if find_parent(parent, a) != find_parent(parent, b):
        union_parent(parent, a, b)
        total_cost += cost

print(total_cost) # 결과: 159

```

### ⏱️ 시간 복잡도

크루스칼 알고리즘의 시간 복잡도는 **$O(E \log E)$** 다. ($E$: 간선의 개수)
유니온 파인드 연산은 매우 빨라서 전체 복잡도에 거의 영향을 주지 않으며, 처음에 **모든 간선을 정렬(`edges.sort()`)하는 데 걸리는 시간**이 전체 알고리즘의 속도를 결정하기 때문이다.
