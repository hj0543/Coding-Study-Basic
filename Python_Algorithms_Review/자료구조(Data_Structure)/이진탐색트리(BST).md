이진 탐색 트리(BST, Binary Search Tree)는 앞서 배운 이진 트리에 **'정렬 규칙'**을 부여하여, 데이터를 찾거나 추가하는 속도를 비약적으로 끌어올린 자료구조다.

업무를 볼 때 서류를 아무렇게나 쌓아두면(일반 트리/배열) 찾을 때마다 전부 뒤져야 하지만, '내 기준 왼쪽에는 나보다 번호가 작은 서류, 오른쪽에는 큰 서류'를 배치해 두면(BST) 탐색 범위를 단숨에 절반씩 날려버릴 수 있다.

### 💡 핵심 규칙과 특징

BST가 성립하기 위해서는 다음의 엄격한 규칙을 무조건 지켜야 한다.

1. **왼쪽 서브 트리의 모든 값은 부모의 값보다 작다.**
2. **오른쪽 서브 트리의 모든 값은 부모의 값보다 크다.**
3. 왼쪽, 오른쪽 서브 트리 역시 모두 이진 탐색 트리여야 한다. (중복된 값은 허용하지 않는 것이 일반적이다.)

이 규칙 덕분에 BST를 **중위 순회(In-order: 왼쪽 -> 루트 -> 오른쪽)** 하면 데이터가 완벽하게 **오름차순으로 정렬**되어 출력된다는 아주 아름다운 특징을 가진다.

### 💻 이진 탐색 트리 템플릿 (Python)

코딩 테스트에서 특정 노드를 찾거나(Search), 새로운 값을 규칙에 맞게 찔러 넣는(Insert) 가장 기본적인 템플릿 코드다.

```python
class Node:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None

# 1. 삽입 (Insert)
def insert(node, key):
    # 트리가 비어있다면 새로운 노드를 반환
    if node is None:
        return Node(key)
    
    # 들어갈 값이 현재 노드보다 작으면 왼쪽으로 파고듦
    if key < node.key:
        node.left = insert(node.left, key)
    # 들어갈 값이 현재 노드보다 크면 오른쪽으로 파고듦
    elif key > node.key:
        node.right = insert(node.right, key)
        
    return node

# 2. 탐색 (Search)
def search(node, key):
    # 찾았거나, 바닥에 도달했는데도 없으면 종료
    if node is None or node.key == key:
        return node
    
    # 찾는 값이 현재 노드보다 작으면 왼쪽 탐색
    if node.key > key:
        return search(node.left, key)
    # 찾는 값이 현재 노드보다 크면 오른쪽 탐색
    return search(node.right, key)

# 테스트
root = None
keys = [50, 30, 20, 40, 70, 60, 80]

# 트리 구성
for key in keys:
    root = insert(root, key)

# 탐색 확인
result = search(root, 60)
if result:
    print(f"찾았다: {result.key}") # 결과: 찾았다: 60

```

### ⏱️ 시간 복잡도와 치명적인 약점

* **평균 시간 복잡도:** **$O(\log N)$**. 트리가 예쁘게 균형을 잡고 있다면, 한 번 비교할 때마다 남은 탐색 대상이 절반으로 줄어든다.
* **🔥 최악의 시간 복잡도:** **$O(N)$**. 만약 데이터를 `10, 20, 30, 40` 순서대로 오름차순으로 삽입하면 어떻게 될까? 트리가 오른쪽으로만 길게 늘어지는 **편향 트리(Skewed Tree)**가 되어버린다. 사실상 선형 리스트와 다를 바가 없어지며 탐색 속도가 박살 난다.

*(이 치명적인 약점을 극복하기 위해, 삽입/삭제를 할 때마다 트리가 스스로 회전하며 균형을 맞추는 AVL 트리나 레드-블랙 트리가 탄생했다.)*

### 🍯 코딩 테스트 실전 압축 (매우 중요)

파이썬으로 코딩 테스트를 볼 때, 문제에서 대놓고 "이진 탐색 트리를 직접 구현하시오"라고 하지 않는 이상 **위 코드를 짤 일은 0%에 수렴한다.**

왜냐하면 데이터를 빠르게 넣고 찾아야 한다면 파이썬의 **`set`이나 `dict` (해시, $O(1)$)**를 쓰면 되고, 데이터가 정렬된 상태를 유지해야 한다면 배열을 정렬한 뒤 **`bisect` 모듈(이진 탐색, $O(\log N)$)**을 쓰거나 **`heapq` (우선순위 큐, $O(\log N)$)**를 쓰는 것이 훨씬 빠르고 강력하기 때문이다. (따라서 BST는 '이론과 동작 원리'를 이해하는 목적으로만 확실히 챙겨둔다.)
