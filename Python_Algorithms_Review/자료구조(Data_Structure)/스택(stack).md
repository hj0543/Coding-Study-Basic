스택(Stack)은 나중에 들어온 데이터가 먼저 나가는 **후입선출(LIFO, Last In First Out)** 원칙을 따르는 아주 근본적이고 중요한 자료구조다.

쉽게 말해 **'뷔페 식당에 차곡차곡 쌓여 있는 접시'**나, 우리가 컴퓨터로 문서 작업을 할 때 자주 쓰는 **'실행 취소(Ctrl + Z)'** 기능을 떠올리면 완벽하다. 맨 마지막에 쌓아 올린 접시를 가장 먼저 써야 하고, 맨 마지막에 한 작업을 가장 먼저 취소하는 것과 완전히 똑같다. 방금 전까지 다뤘던 큐(Queue)와는 정반대의 성질을 가지고 있다.

### 💡 자료구조 동작 과정

스택의 상태를 변화시키는 핵심 동작은 큐와 마찬가지로 아주 단순하다.

1. **푸시 (Push):** 스택의 **맨 위(Top)**에 새로운 데이터를 차곡차곡 쌓아 올린다.
2. **팝 (Pop):** 스택의 **맨 위(Top)**에 있는(가장 마지막에 들어온) 데이터를 꺼내고(삭제하고) 반환한다.
3. **피크 (Peek / Top):** 맨 위의 데이터를 삭제하지 않고 슬쩍 확인만 한다.

---

### 💻 스택 기본 코드 (Python)

파이썬에서 스택을 구현할 때는 다른 특별한 모듈을 불러올 필요가 전혀 없다. **파이썬의 기본 리스트(`list`) 자체가 이미 완벽한 스택으로 동작하도록 설계되어 있기 때문**이다.

리스트의 `append()` 메서드가 'Push' 역할을, `pop()` 메서드가 'Pop' 역할을 100% 동일하게 수행한다.

```python
# 빈 스택(리스트) 생성
stack = []

# 1. 삽입 (Push): 5 - 2 - 3 - 7 순서로 쌓기
stack.append(5)
stack.append(2)
stack.append(3)
stack.append(7)
print("초기 스택 상태:", stack) # [5, 2, 3, 7] (오른쪽이 맨 위(Top)를 의미함)

# 2. 삭제 (Pop): 가장 나중에 들어온 7이 가장 먼저 나감 (LIFO)
top_item = stack.pop() 
print("꺼낸 데이터:", top_item) # 7
print("현재 스택 상태:", stack) # [5, 2, 3]

# 3. 추가 삽입 및 삭제
stack.append(1)        # 1을 맨 위에 쌓음 -> [5, 2, 3, 1]
stack.append(4)        # 4를 맨 위에 쌓음 -> [5, 2, 3, 1, 4]
stack.pop()            # 가장 나중에 들어온 4가 나감 -> [5, 2, 3, 1]

# 최종 스택 상태 출력 (아래쪽부터 차례대로 출력됨)
print("최종 스택 상태:", stack) # [5, 2, 3, 1]

```

### ⏱️ 시간 복잡도

파이썬 리스트를 활용한 스택 연산의 시간 복잡도는 매우 훌륭하다.

* **삽입 (Push / `append`):** **$O(1)$**
* **삭제 (Pop / `pop`):** **$O(1)$**

단, 큐에서 배웠던 것처럼 리스트에서 `pop(0)`을 써서 맨 앞의 요소를 건드리면 $O(N)$이 걸리지만, 스택처럼 **맨 뒤(Top)의 데이터만 넣고 빼는 작업은 항상 $O(1)$**의 상수가 보장된다.

### 📝 예시 문제: 제로 (백준 10773번)

나코더 기장 재민이는 동아리 회식을 준비하기 위해서 장부를 관리하는 중이다. 재현이는 재민이를 도와서 돈을 관리하는 중인데, 애석하게도 항상 정신없는 재현이는 돈을 실수로 잘못 부르는 사고를 치기 일쑤였다.
재현이는 잘못된 수를 부를 때마다 0을 외쳐서, 가장 최근에 재민이가 쓴 수를 지우게 시킨다.
재민이는 이렇게 모든 수를 받아 적은 후 그 수의 합을 알고 싶어 한다. 재민이를 도와주자!

* **입력:** 첫 번째 줄에 정수 $K$가 주어진다. ($1 \le K \le 100,000$) 이후 $K$개의 줄에 정수가 1개씩 주어진다. 정수가 0일 경우에는 가장 최근에 쓴 수를 지우고, 아닐 경우 해당 수를 쓴다. 정수가 0일 경우에 지울 수 있는 수가 있음을 보장할 수 있다.
* **출력:** 재민이가 최종적으로 적어 낸 수의 합을 출력한다.

---

### 💡 풀이 접근법

문제 설명에서 "가장 최근에 쓴 수를 지운다"는 문장이 나오는 순간, 1초의 망설임도 없이 **'스택의 Pop 연산'**을 떠올려야 한다. 전형적인 스택의 LIFO 성질을 묻는 기본 문제다.

1. 빈 스택(리스트)을 하나 만든다.
2. 입력받은 숫자가 0이 아니라면, 장부에 적어야 하므로 스택에 `append` 한다.
3. 입력받은 숫자가 0이라면, 가장 최근에 적은 숫자를 지워야 하므로 스택에서 `pop` 한다.
4. 모든 입력이 끝난 후, 스택에 남아있는 숫자들의 합을 구해서 출력한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

k = int(input())
stack = []

for _ in range(k):
    num = int(input())
    
    # 0이 입력되면 가장 최근에 쓴 수를 지움 (Pop)
    if num == 0:
        stack.pop()
    # 0이 아니라면 장부에 새로운 수를 씀 (Push)
    else:
        stack.append(num)

# 스택에 남아있는 모든 수의 합을 출력
# sum() 함수는 리스트의 모든 원소를 더해주는 파이썬 내장 함수
print(sum(stack))

```
