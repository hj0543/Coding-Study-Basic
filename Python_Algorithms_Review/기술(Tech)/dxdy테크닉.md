2차원 격자(Grid) 탐색의 꽃이라 불리는 **dx, dy 테크닉**이다. 이 기법은 상하좌우나 대각선 방향으로 이동할 때 `if`문을 덕지덕지 붙이지 않고, **방향 벡터**를 이용해 우아하고 간결하게 처리하는 방법이다.

---

## 1. 왜 dx, dy를 쓰는가?

격자 위에서 현재 위치가 `(r, c)`일 때, 상하좌우로 이동하려면 보통 이렇게 생각한다.

* 위로 이동: `(r-1, c)`
* 아래로 이동: `(r+1, c)`
* 왼쪽 이동: `(r, c-1)`
* 오른쪽 이동: `(r, c+1)`

이걸 매번 `if direction == 'U': r -= 1` 식으로 짜면 방향이 많아질수록 코드가 지저분해진다. 이때 **변화량(Delta)**만 따로 모아두면 반복문 하나로 끝낼 수 있다.

---

## 2. 기본 구현 (상하좌우 4방향)

행의 변화량을 `dr`, 열의 변화량을 `dc`라는 리스트에 순서대로 담는다. (보통 x, y보다 r, c가 행렬 인덱스와 매칭하기 편하다.)

```python
# 상(U), 하(D), 좌(L), 우(R) 순서
dr = [-1, 1, 0, 0]
dc = [0, 0, -1, 1]

r, c = 2, 2  # 현재 위치

for i in range(4):
    nr = r + dr[i]  # 다음(Next) 행
    nc = c + dc[i]  # 다음(Next) 열
    
    # [필수] 격자 범위를 벗어나는지 반드시 확인한다
    if 0 <= nr < N and 0 <= nc < M:
        # 이동 가능한 칸일 경우 로직 수행
        print(f"다음 위치: ({nr}, {nc})")

```

---

## 3. 다양한 확장 테크닉

### ① 대각선을 포함한 8방향

상하좌우에 대각선 4방향을 추가하면 된다. 시계 방향 혹은 반시계 방향으로 순서를 맞춰주는 것이 구현할 때 덜 헷갈린다.

```python
# 북, 북동, 동, 남동, 남, 남서, 서, 북서
dr = [-1, -1, 0, 1, 1, 1, 0, -1]
dc = [0, 1, 1, 1, 0, -1, -1, -1]

```

### ② 체스의 나이트(Knight) 이동

L자 형태로 움직이는 나이트의 이동 경로도 8개 리스트로 미리 정의해두면 쉽다.

```python
dr = [-2, -2, -1, -1, 1, 1, 2, 2]
dc = [-1, 1, -2, 2, -2, 2, -1, 1]

```

### ③ 방향 회전 (시뮬레이션 문제)

"오른쪽으로 90도 회전하라" 같은 명령이 들어오면, 현재 방향 인덱스 `dir_idx`를 관리하며 나머지 연산(`%`)을 활용한다.

```python
# 동(0), 남(1), 서(2), 북(3) - 시계 방향 순서
dr = [0, 1, 0, -1]
dc = [1, 0, -1, 0]

dir_idx = 0  # 처음엔 동쪽
# 오른쪽으로 90도 회전
dir_idx = (dir_idx + 1) % 4
# 왼쪽으로 90도 회전
dir_idx = (dir_idx - 1) % 4

```

---

## 4. 실전 활용 포인트

1. **경계선 체크(Boundary Check):** `0 <= nr < N and 0 <= nc < M` 조건은 dx, dy 테크닉의 단짝이다. 잊으면 바로 `IndexError`를 만난다.
2. **가독성:** `dr`, `dc` 대신 `directions = [(-1, 0), (1, 0), ...]` 처럼 튜플 리스트를 사용해 `for dr, dc in directions:`로 순회하기도 한다.
3. **최단 거리 탐색:** BFS(너비 우선 탐색)에서 큐에 다음 좌표를 넣을 때 이 테크닉이 핵심적으로 쓰인다.

---

**정리하자면**, dx, dy 테크닉은 **"좌표의 변화량을 데이터화하여 로직을 단순화하는 것"**이다. 이 방식에 익숙해지면 어떤 복잡한 이동 시뮬레이션 문제도 규칙성 있게 풀어낼 수 있다.
