**플래트닝(Flattening)**은 2차원 이상의 다차원 배열을 1차원 리스트로 쭉 펴는 기법이다. 모든 원소를 순차적으로 나열하여 일괄 처리를 하거나, 특정 알고리즘(예: 정렬, 완전 탐색)에 입력값으로 넣을 때 자주 사용한다.

---

## 1. 플래트닝의 원리

행 단위로 데이터를 읽어 내려가며 하나의 긴 줄로 이어 붙이는 작업이다. $N \times M$ 배열을 플래트닝하면 길이가 $N \times M$인 1차원 리스트가 된다.

---

## 2. 구현 방법 (4가지)

### ① 2중 리스트 컴프리헨션 (권장 방식)

가장 파이썬답고 직관적인 방법이다. 중첩 반복문의 순서(행 → 열)를 그대로 따라간다.

```python
matrix = [[1, 2, 3], [4, 5, 6]]
# 바깥쪽 for문이 행(row), 안쪽 for문이 열(item)
flattened = [item for row in matrix for item in row]
# 결과: [1, 2, 3, 4, 5, 6]

```

### ② itertools.chain 이용 (성능 최적화)

배열의 크기가 매우 클 때 메모리 효율과 속도 면에서 가장 우수하다. `chain.from_iterable()`을 사용하면 각 행을 하나로 엮어준다.

```python
from itertools import chain

matrix = [[1, 2, 3], [4, 5, 6]]
flattened = list(chain.from_iterable(matrix))

```

### ③ sum() 함수를 이용한 트릭 (가독성)

`sum()` 함수의 두 번째 인자에 빈 리스트 `[]`를 넣으면, 리스트끼리 더하기(`+`) 연산이 수행되어 하나로 합쳐진다. 코드는 가장 짧지만 리스트를 계속 새로 만들기 때문에 **성능은 가장 좋지 않다.**

```python
matrix = [[1, 2, 3], [4, 5, 6]]
flattened = sum(matrix, [])

```

### ④ NumPy를 이용한 방법

데이터 과학 분야라면 고민할 것 없이 `flatten()`이나 `ravel()`을 사용한다.

* `flatten()`: 복사본을 반환 (안전)
* `ravel()`: 원본의 뷰(View)를 반환 (메모리 효율적)

```python
import numpy as np
arr = np.array([[1, 2, 3], [4, 5, 6]])
flattened = arr.flatten()

```

---

## 3. 실전 활용 포인트

1. **전체 원소 정렬:** 2차원 배열의 모든 원소를 크기 순으로 정렬하고 싶을 때, 먼저 플래트닝을 한 뒤 `sort()`를 수행하면 편리하다.
2. **좌표 변환:** 2차원 좌표 $(r, c)$를 1차원 인덱스 $k$로 변환해야 할 때 다음 공식을 사용한다. (열의 개수가 $M$일 때)
* **2D → 1D:** $k = r \times M + c$
* **1D → 2D:** $r = k // M, \quad c = k \% M$


3. **데이터 탐색:** 2차원 배열에서 특정 조건을 만족하는 원소의 개수를 세거나 평균을 구할 때 1차원으로 펴서 계산하면 코드가 간결해진다.

## 🛠️ 방식별 특성 요약

| 방법 | 가독성 | 성능 | 비고 |
| --- | --- | --- | --- |
| **리스트 컴프리헨션** | 매우 좋음 | 좋음 | 일반적인 상황에서 가장 추천함 |
| **itertools.chain** | 보통 | **최고** | 대용량 데이터 처리 시 필수 |
| **sum(matrix, [])** | **최고** | 나쁨 | 간단한 코드 작성 시 유용(비추천) |
| **NumPy flatten** | 좋음 | 매우 좋음 | 외부 라이브러리 의존성 발생 |

---

**정리하자면**, 플래트닝은 **"다차원의 복잡함을 1차원의 단순함으로 치환하는 작업"**이다. 인덱스 계산이 꼬일 것 같다면 일단 펴서 생각하는 것도 좋은 전략이다.
