**배열 패딩(Padding)**은 기존 배열의 가장자리를 특정 값(보통 0)으로 둘러싸서 크기를 키우는 기법이다. 알고리즘 문제 풀이나 이미지 처리(CNN 등)에서 매우 빈번하게 사용된다.

---

## 1. 왜 패딩을 사용하는가?

* **인덱스 범위 체크 생략:** BFS/DFS나 시뮬레이션 문제에서 상하좌우를 탐색할 때, `0 <= nx < N` 같은 조건문을 매번 쓰기 귀찮을 때 유용하다. 테두리를 한 칸씩 더 만들어두면 인덱스 초과 에러(IndexError) 걱정 없이 탐색할 수 있다.
* **원본 크기 유지:** 합성곱(Convolution) 연산을 할 때, 패딩 없이 필터를 적용하면 결과 이미지가 작아진다. 패딩을 주면 연산 후에도 원본과 동일한 $N \times M$ 크기를 유지할 수 있다.
* **가장자리 정보 보존:** 배열의 모서리나 벽에 있는 데이터들도 중앙 데이터만큼 충분히 연산에 참여할 수 있게 해준다.

---

## 2. 파이썬 구현 방법 (2가지)

### ① 리스트 컴프리헨션을 이용한 수동 패딩

가장 원초적이지만 라이브러리를 쓸 수 없는 알고리즘 테스트 환경에서 필수적인 방법이다.

* **원리:** $(N+2) \times (M+2)$ 크기의 빈 배열을 만들고, 그 안에 원본을 복사한다.

```python
def manual_padding(matrix, p_value=0):
    n = len(matrix)
    m = len(matrix[0])
    # 1. (N+2) x (M+2) 크기의 기본 배열 생성
    padded = [[p_value] * (m + 2) for _ in range(n + 2)]
    
    # 2. 중앙에 원본 데이터 복사
    for i in range(n):
        for j in range(m):
            padded[i + 1][j + 1] = matrix[i][j]
            
    return padded

matrix = [[1, 2], [3, 4]]
# 결과: [[0, 0, 0, 0], [0, 1, 2, 0], [0, 3, 4, 0], [0, 0, 0, 0]]

```

### ② NumPy를 이용한 효율적인 패딩

데이터 분석이나 딥러닝 환경이라면 `numpy.pad()`를 쓰는 것이 압도적으로 편하고 빠르다.

```python
import numpy as np

matrix = np.array([[1, 2], [3, 4]])
# 상하좌우에 1칸씩 0으로 패딩
padded = np.pad(matrix, ((1, 1), (1, 1)), mode='constant', constant_values=0)

```

## 3. 실전 활용 팁 (알고리즘 문제)

* **벽(Wall) 세우기:** 미로 탐색 문제에서 배열의 외곽을 모두 벽(예: 1)으로 감싸버리면, 탐색 로직이 배열 밖으로 나가는 것을 자연스럽게 막을 수 있다.
* **다양한 패딩 모드:**
* **Constant:** 특정 값(0 등)으로 채우기
* **Reflect:** 가장자리 값을 거울처럼 반사해서 채우기 (이미지 처리에서 주로 사용)
* **Edge:** 가장자리에 있는 값을 그대로 연장해서 채우기
