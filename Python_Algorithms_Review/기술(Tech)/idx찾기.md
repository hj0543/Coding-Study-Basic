파이썬에서 특정 데이터의 '위치'를 찾을 때 사용하는 **탐색 메서드 4대장(`find`, `index`, `rfind`, `rindex`)**이다.

코딩 테스트에서 문자열이나 리스트를 파싱(Parsing)할 때 숨 쉬듯 쓰이는 기본기지만, **"없을 때 어떻게 반응하는가?"**와 **"어느 자료형에 쓸 수 있는가?"**를 정확히 모르면 치명적인 런타임 에러(런타임 에러 = 0점)를 맞게 되는 무서운 녀석들이기도 하다.

---

### 💡 1. `find()` vs `index()` : 치명적인 차이점 하나

두 함수 모두 **"왼쪽부터 찾아서 처음 발견된 곳의 인덱스를 반환"**한다는 점은 완벽하게 똑같다. 하지만 **찾는 값이 없을 때의 태도**가 완전히 다르다.

* **`find(x)`:** 값이 없으면 쿨하게 **`-1`**을 반환한다. (안전함)
* **🔥 `index(x)`:** 값이 없으면 **`ValueError`를 던지며 프로그램을 터뜨려버린다.** (위험함)

```python
text = "hello world"

# 1. 있는 값을 찾을 때 (둘 다 똑같음)
print(text.find('o'))  # 결과: 4
print(text.index('o')) # 결과: 4

# 2. 없는 값을 찾을 때 (극명한 차이)
print(text.find('z'))  # 결과: -1 (프로그램 정상 진행)
# print(text.index('z')) # 🚨 ValueError 발생! 프로그램 강제 종료!

```

> **코테 실전 꿀팁:** 값이 확실히 있을 때만 `index()`를 쓰고, 있을지 없을지 모른다면 무조건 `find()`를 쓰거나 `if x in text:`로 먼저 검사해야 한다.

---

### 💡 2. `rfind()` vs `rindex()` : 뒤에서부터 찾기 (Right)

앞에 붙은 `r`은 **Right(오른쪽)**를 의미한다. 문자열의 맨 뒤(오른쪽)부터 역방향으로 스캔하여 **가장 마지막에 등장하는 위치**를 찾고 싶을 때 사용한다. (반환되는 인덱스 번호 자체는 원래의 정방향 인덱스 기준이다.)

이 녀석들 역시 찾는 값이 없을 때 `rfind`는 `-1`을, `rindex`는 에러를 뱉는다.

```python
text = "banana"
# 인덱스: 012345

print(text.find('a'))  # 결과: 1 (왼쪽에서 첫 번째 'a')
print(text.rfind('a')) # 결과: 5 (오른쪽에서 첫 번째, 즉 전체에서 마지막 'a')

```

---

### 🔥 3. 코딩 테스트 절대 주의사항: "지원하는 자료형이 다르다!"

이 4가지 함수를 쓸 때 가장 많이 하는 실수가 바로 이것이다.

* **`find()`, `rfind()`, `rindex()`:** 이 세 녀석은 **오직 문자열(`str`)에서만** 쓸 수 있다! 리스트나 튜플에서 쓰면 `AttributeError`가 발생한다.
* **`index()`:** 이 녀석은 **문자열(`str`), 리스트(`list`), 튜플(`tuple`) 모두**에서 쓸 수 있는 만능이다.

```python
arr = [10, 20, 30, 20]

# 리스트에서는 index()만 허용된다.
print(arr.index(20)) # 결과: 1

# 리스트에서 뒤에서부터 찾고 싶다고 rfind()를 쓰면?
# print(arr.rfind(20)) # 🚨 AttributeError 발생! 리스트엔 그런 거 없다!

```

> *(리스트에서 특정 값의 마지막 위치를 찾고 싶다면, 리스트를 뒤집은 뒤(`arr[::-1]`) `index`를 찾거나 `for`문을 거꾸로 돌려야 한다.)*
