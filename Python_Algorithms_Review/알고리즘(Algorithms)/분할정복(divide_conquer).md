단순히 문제를 쪼개는 것을 넘어서, 분할 정복의 진짜 강력함은 **'불필요한 탐색을 통째로 스킵(Skip)'**하거나 **'중복되는 연산을 반으로 뚝 잘라버리는'** 데 있다. 이 원리를 제대로 이해하면 $O(N)$이 걸릴 연산을 $O(\log N)$으로 마법처럼 줄일 수 있다.

### 💡 알고리즘 설계의 3원칙과 핵심 동작

분할 정복 알고리즘을 설계할 때는 보통 **재귀 함수(Recursion)**를 사용하며, 다음 3가지 단계를 명확히 구분해서 코드를 짜야 한다.

1. **기저 조건 (Base Case):** 더 이상 문제를 쪼갤 수 없을 때(원소가 1개 남았거나, 크기가 1이 되었을 때) 정답을 반환하고 재귀를 종료하는 조건이다. **이게 없으면 무한 루프에 빠진다.**
2. **분할 (Divide):** 현재 직면한 문제를 절반(또는 4등분 등)으로 쪼갠다. 이때 쪼개진 하위 문제들은 원래 문제와 성격이 완전히 똑같아야 한다.
3. **정복 및 결합 (Conquer & Combine):** 쪼개진 하위 문제들 중 **'내가 찾아야 할 정답이 있는 곳'**만 선택해서 탐색하거나(정복), 하위 문제들의 결과를 합쳐서 현재 단계의 정답을 만들어 반환한다(결합).

### 💻 분할 정복 심화 코드: $O(N)$을 $O(\log N)$으로 줄이기

단순 곱셈과 분할 정복 곱셈의 속도 차이를 보여주는 아주 직관적인 예시다. $A$를 $N$번 곱할 때, $A \times A \times A \dots$ 처럼 하나씩 곱하면 $N$번의 연산이 필요하다.
하지만 분할 정복을 사용하면 $A^8 = A^4 \times A^4$이고, $A^4 = A^2 \times A^2$라는 점을 이용해 연산 횟수를 기하급수적으로 줄일 수 있다.

```python
# 일반적인 O(N) 곱셈
def normal_power(a, n):
    result = 1
    for _ in range(n):
        result *= a
    return result

# 분할 정복을 이용한 O(log N) 곱셈
def divide_power(a, n):
    # 1. 기저 조건: 0승은 무조건 1
    if n == 0:
        return 1
    # 1승은 자기 자신
    if n == 1:
        return a
        
    # 2. 분할: 지수를 절반으로 쪼개서 재귀 호출
    half = divide_power(a, n // 2)
    
    # 3. 정복 및 결합: 쪼갠 결과를 합침
    if n % 2 == 0:
        return half * half       # 짝수면 그대로 두 번 곱함 (예: a^4 = a^2 * a^2)
    else:
        return half * half * a   # 홀수면 a를 한 번 더 곱함 (예: a^5 = a^2 * a^2 * a)

print(divide_power(2, 10)) # 10번 곱할 것을 단 3~4번의 연산으로 끝냄

```

### ⏱️ 시간 복잡도

문제를 매번 절반으로 쪼개고, 그중 하나만 탐색하거나 결과를 재사용한다면 시간 복잡도는 **$O(\log N)$**이 된다. (예: 이분 탐색, 거듭제곱)
문제를 절반으로 쪼갠 뒤, 쪼개진 모든 조각을 다 탐색하고 병합하는 과정을 거친다면 **$O(N \log N)$**이 된다. (예: 병합 정렬)

### 📝 예시 문제: Z (백준 1074번)

크기가 $2^N \times 2^N$인 2차원 배열을 Z모양으로 탐색하려고 한다. 예를 들어, $2 \times 2$ 배열을 왼쪽 위칸, 오른쪽 위칸, 왼쪽 아래칸, 오른쪽 아래칸 순서대로 방문하면 Z모양이다.
$N > 1$인 경우, 배열을 크기가 $2^{N-1} \times 2^{N-1}$로 4등분 한 후에 재귀적으로 순서대로 방문한다.
$N$이 주어졌을 때, $r$행 $c$열을 몇 번째로 방문하는지 출력하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에 정수 $N, r, c$가 주어진다. ($1 \le N \le 15, 0 \le r, c < 2^N$)
* **출력:** $r$행 $c$열을 몇 번째로 방문했는지 출력한다. (방문 순서는 0번부터 시작한다.)

---

### 💡 풀이 접근법 (자세히)

이 문제는 분할 정복의 **'탐색 스킵(Skip)'** 능력을 보여주는 끝판왕이다.
$N=15$라면 배열의 크기는 $32768 \times 32768$이고, 총 칸 수는 약 10억 개다. 이걸 정말 1번부터 10억 번까지 하나하나 방문하면서 `(r, c)`를 찾으면 100% 시간 초과가 발생한다.

따라서 전체 공간을 4등분(사분면)으로 쪼갠 뒤, **"내가 찾는 `(r, c)`가 이 4개의 구역 중 어디에 속하는가?"**만 판단해야 한다.

1. 찾고자 하는 칸이 없는 사분면은 굳이 깊게 탐색할 필요가 없다. 그 사분면의 크기(칸 수)만큼 방문 횟수를 정답에 한 번에 더해버리고 '스킵'한다.
2. 찾고자 하는 칸이 있는 사분면만 다시 4등분으로 쪼개서 재귀적으로 파고든다.
3. 크기가 1인 칸에 도달할 때까지 반복한다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

# N: 2^N x 2^N 크기의 보드, r: 행, c: 열
n, r, c = map(int, input().split())
ans = 0

# x, y: 현재 탐색 중인 공간의 가장 왼쪽 위 좌표 / size: 현재 공간의 한 변의 길이
def solve(x, y, size):
    global ans
    
    # 1. 기저 조건: 크기가 1x1인 칸에 도달했고, 거기가 목표 지점이라면 탐색 종료
    if x == r and y == c:
        print(ans)
        sys.exit() # 정답을 찾았으므로 프로그램 완전 종료
        
    # 2. 정복: 현재 탐색 중인 구역 안에 (r, c)가 포함되는지 확인
    # 포함된다면 이 구역을 다시 4등분하여 탐색
    if x <= r < x + size and y <= c < y + size:
        half = size // 2
        solve(x, y, half)                 # 1사분면 (왼쪽 위)
        solve(x, y + half, half)          # 2사분면 (오른쪽 위)
        solve(x + half, y, half)          # 3사분면 (왼쪽 아래)
        solve(x + half, y + half, half)   # 4사분면 (오른쪽 아래)
        
    # 3. 포함되지 않는다면: 이 구역은 전부 스킵하고, 구역의 넓이만큼 방문 횟수를 더함
    else:
        ans += size * size

# 시작 좌표 (0, 0), 초기 보드 크기 2^N
solve(0, 0, 2 ** n)

```
