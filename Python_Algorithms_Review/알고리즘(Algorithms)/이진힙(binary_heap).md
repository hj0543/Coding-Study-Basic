이진 힙(Binary Heap)은 앞서 배운 **'완전 이진 트리'**를 뼈대로 만들어진 특수 자료구조로, 수많은 데이터 중에서 **최댓값이나 최솟값을 빛의 속도로 뽑아내기 위해** 고안되었다.

일반적인 배열이나 리스트에서 최솟값을 찾으려면 데이터를 전부 뒤져야 해서 $O(N)$의 시간이 걸리지만, 이진 힙을 사용하면 단 **$O(1)$** 만에 최솟값을 확인하고, 데이터를 빼거나 새로 넣을 때도 트리의 높이만큼만 움직이는 **$O(\log N)$** 만에 트리의 균형을 다시 맞출 수 있다.

### 💡 핵심 규칙 2가지

이진 힙은 목적에 따라 두 가지로 나뉘며, 다음의 '느슨한 정렬' 규칙을 반드시 따른다.

1. **최소 힙 (Min-Heap):** 부모 노드의 값은 항상 자식 노드의 값보다 **작거나 같다**. 따라서 트리의 맨 꼭대기(루트)에는 항상 전체 데이터 중 **최솟값**이 자리하게 된다.
2. **최대 힙 (Max-Heap):** 부모 노드의 값은 항상 자식 노드의 값보다 **크거나 같다**. 꼭대기에는 항상 **최댓값**이 자리한다.

*(주의: 형제 노드끼리는 왼쪽이 크든 오른쪽이 크든 전혀 상관하지 않는다. 오직 위아래(부모-자식)의 대소 관계만 철저하게 지킨다.)*

### 💻 코딩 테스트 실전 구현 (`heapq` 모듈)

코딩 테스트에서 이진 힙을 1차원 배열로 직접 구현할 일은 없다. 파이썬이 제공하는 강력한 내장 라이브러리인 **`heapq`**를 사용하면 된다.
**파이썬의 `heapq`는 기본적으로 '최소 힙(Min-Heap)'으로만 동작한다.**

```python
import heapq

# 힙으로 사용할 빈 리스트 생성
pq = []

# 1. 데이터 넣기 (Push) -> O(log N)
heapq.heappush(pq, 5)
heapq.heappush(pq, 1)
heapq.heappush(pq, 3)

# 가장 작은 값이 자동으로 0번 인덱스(루트)에 위치함
print(pq[0]) # 결과: 1

# 2. 데이터 빼기 (Pop) -> O(log N)
# 가장 작은 값을 빼내고, 남은 원소들로 다시 최소 힙을 구성함
print(heapq.heappop(pq)) # 결과: 1 반환 (남은 힙: [3, 5])

# 3. 기존 리스트를 한 번에 힙으로 만들기 (Heapify) -> O(N)
arr = [7, 2, 4, 3, 1]
heapq.heapify(arr)
print(arr) # 결과: [1, 2, 4, 3, 7] (규칙에 맞게 재배치됨)

```

#### 🔥 초특급 실전 팁: 파이썬에서 '최대 힙' 만들기

파이썬은 최소 힙만 지원하므로, 최댓값을 뽑고 싶다면 **데이터를 넣을 때 부호를 반대로(`-`) 뒤집어서 넣고, 뺄 때 다시 부호를 뒤집는 꼼수**를 사용해야 한다.

```python
max_heap = []
items = [10, 30, 20]

# 넣을 때 마이너스(-)를 붙여서 넣음
for item in items:
    heapq.heappush(max_heap, -item)

# 뺄 때 다시 마이너스(-)를 붙여서 원래 양수 값으로 복구함
# 가장 큰 값이었던 30이 가장 작은 값(-30)이 되어 루트에 있었으므로 가장 먼저 나옴!
print(-heapq.heappop(max_heap)) # 결과: 30

```

### 📂 어디에 넣으면 좋을까?

* **분류:** **[자료구조 - 트리 / 우선순위 큐]**
* **사유:** 완전 이진 트리라는 '구조'를 바탕으로 만들어졌으며, 들어온 순서와 상관없이 우선순위(대소 관계)가 높은 데이터부터 먼저 나가는 **'우선순위 큐(Priority Queue)'**를 구현하는 가장 완벽한 방법이기 때문이다.
