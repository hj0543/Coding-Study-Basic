LIS(Longest Increasing Subsequence, 최장 증가 부분 수열)는 이름 그대로 어떤 수열이 주어졌을 때, **그 수열의 부분 수열 중 원소가 오름차순으로 유지되면서 가장 긴 수열**을 찾는 알고리즘이다.

예를 들어, 수열 `[10, 20, 10, 30, 20, 50]`이 있다면 LIS는 `[10, 20, 30, 50]`이 되며 그 길이는 4다. 이 문제는 구현 방식에 따라 시간 복잡도가 크게 달라지므로, 문제에서 주어지는 데이터의 양에 맞춰 적절한 알고리즘을 선택하는 것이 중요하다.

### 💡 알고리즘 동작 과정

LIS를 구하는 방식은 크게 두 가지가 있다.

#### 1. 다이나믹 프로그래밍 (DP) 방식

* 모든 위치 `i`에 대해, 그 이전에 있는 원소 `j`들을 확인한다.
* `arr[j] < arr[i]`라면, `dp[i] = max(dp[i], dp[j] + 1)`을 수행한다.
* 가장 직관적이지만 속도가 상대적으로 느리다.

#### 2. 이진 탐색 (Binary Search) 방식

* `LIS`라는 별도의 리스트를 관리하며, 수열의 원소를 하나씩 확인한다.
* 현재 원소가 `LIS`의 마지막 원소보다 크다면 뒤에 추가한다.
* 크지 않다면, `LIS` 내에서 현재 원소가 들어갈 적절한 위치를 **이진 탐색(`bisect_left`)**으로 찾아 그 자리를 현재 원소로 교체(Update)한다.
* 길이를 구하는 데 최적화된 방식이다.

---

### 💻 LIS 기본 코드 (Python - 이진 탐색 방식)

데이터의 개수가 많을 때 사용하는 가장 효율적인 코드다.

```python
from bisect import bisect_left

arr = [10, 20, 10, 30, 20, 50]
# LIS의 길이를 구하기 위한 빈 리스트 생성
lis = []

for x in arr:
    # 1. lis가 비어있거나 현재 원소가 lis의 마지막 값보다 크면 추가
    if not lis or x > lis[-1]:
        lis.append(x)
    # 2. 그렇지 않다면 이진 탐색으로 들어갈 위치를 찾아 교체
    else:
        idx = bisect_left(lis, x)
        lis[idx] = x

print(len(lis)) # 결과: 4

```

### ⏱️ 시간 복잡도

* **DP 방식:** **$O(N^2)$** (노드 개수 $N$이 5,000 이하일 때 적합)
* **이진 탐색 방식:** **$O(N \log N)$** (노드 개수 $N$이 100,000 이상인 대규모 데이터에 적합)

### 📝 예시 문제: 가장 긴 증가하는 부분 수열 (백준 11053번)

수열 $A$가 주어졌을 때, 가장 긴 증가하는 부분 수열을 구하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에 수열 $A$의 크기 $N$ ($1 \le N \le 1,000$)이 주어진다. 둘째 줄에는 수열 $A$를 이루고 있는 $A_i$가 주어진다.
* **출력:** 첫째 줄에 수열 $A$의 가장 긴 증가하는 부분 수열의 길이를 출력한다.

---

### 💡 풀이 접근법

이 문제는 $N$이 1,000으로 매우 작다. 따라서 $O(N^2)$인 DP 방식으로도 충분히 풀 수 있지만, 실전에서는 더 큰 $N$이 주어질 확률이 높으므로 $O(N \log N)$ 방식인 이진 탐색으로 구현하는 연습을 해두는 것이 좋다.

* **주의:** 이진 탐색으로 만든 `lis` 리스트는 최종 결과인 'LIS 수열' 그 자체와는 구성 원소가 다를 수 있다. (길이만 정확하게 보장한다.) 만약 LIS의 **실제 원소들**까지 출력해야 한다면, 각 원소가 들어갔던 인덱스를 기록하는 추가 작업이 필요하다.

---

### 💻 풀이 코드 (Python)

```python
import sys
from bisect import bisect_left
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

# 최적화된 LIS 길이를 구하기 위한 배열
lis = []

for x in arr:
    # 현재 숫자가 LIS의 마지막 숫자보다 크면 뒤에 붙임
    if not lis or x > lis[-1]:
        lis.append(x)
    else:
        # 그렇지 않으면 들어갈 자리를 이진 탐색으로 찾아 덮어씌움
        idx = bisect_left(lis, x)
        lis[idx] = x

# LIS 배열의 최종 길이를 출력
print(len(lis))

```
