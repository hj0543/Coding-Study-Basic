좋다. 내가 설명했던 직관적인 개념에, 네가 가져온 교과서적인 정석과 실전 백준(BOJ) 기출문제까지 완벽하게 한 판으로 융합해 주겠다.

이것이 바로 코딩 테스트 그래프 탐색의 진정한 보스몹이자, 도로망 길 찾기의 절대 근본 알고리즘 **'다익스트라(Dijkstra)'**의 완전판이다.

### 💡 다익스트라의 본질: "가장 싼 곳부터 방문하는 그리디(Greedy) + 우선순위 큐"

특정한 하나의 출발 노드에서 다른 모든 노드로 가는 최단 경로를 한 번에 싹 다 구하는 알고리즘이다. 매 상황에서 **'가장 비용이 적게 드는 노드'**를 우선적으로 선택한다는 점에서 **그리디(Greedy)** 알고리즘으로 분류되며, 이를 가장 빠르게 구현하기 위해 **우선순위 큐(Priority Queue)**를 100% 활용한다.

#### ⚙️ 완벽한 동작 6단계

1. **출발지 세팅:** 출발 노드를 설정한다.
2. **초기화:** 최단 거리 테이블을 모두 무한대(INF)로 초기화하고, 출발 노드 자신의 거리는 0으로 설정하여 큐에 넣는다.
3. **가장 싼 노드 봅기:** 우선순위 큐에서 현재 가장 거리(비용)가 짧은 노드를 꺼낸다.
4. **가지치기 (핵심):** 꺼낸 노드가 이미 방문 처리되었거나, 기록된 최단 거리보다 비용이 크다면 쿨하게 무시한다. (한 번 큐에서 정상적으로 꺼내져 처리된 노드는 최단 거리가 확정된 것이다!)
5. **이웃 탐색 및 갱신:** 해당 노드를 거쳐서 인접한 다른 노드로 가는 비용을 계산한다. 만약 이 거쳐가는 비용이 기존에 알고 있던 거리보다 짧다면, 최단 거리 테이블을 더 싼 값으로 갱신하고 우선순위 큐에 새로 넣는다.
6. 큐가 텅 빌 때까지 3~5번을 반복한다.

---

### ⏱️ 시간 복잡도와 절대 제약 조건

* **시간 복잡도:** **$O(E \log V)$** ($V$: 정점의 개수, $E$: 간선의 개수).
모든 간선을 한 번씩 확인하며, 우선순위 큐(힙)에 정보를 넣고 빼는 과정에서 $O(\log V)$가 소요된다. 수십만 개의 데이터도 1초 안에 박살 낸다.
* **🔥 절대 제약 조건 (암기 필수):** **그래프에 가중치가 음수(-)인 간선이 단 하나라도 존재하면 다익스트라는 절대 쓸 수 없다.** 이미 지나쳐온 길은 최적의 해라고 믿고 다시는 뒤돌아보지 않기 때문에, 음수가 섞이면 무한 루프에 빠지거나 오답을 낸다.

### ⚔️ 실전 전투: 최단경로 (백준 1753번)

실전 코딩 테스트에서 다익스트라를 어떻게 구현해야 하는지 완벽하게 보여주는 스탠다드 문제다. 입력 속도 최적화(`sys.stdin.readline`)와 무한대 설정(`1e9`)까지 포함된 실전 100% 코드다.

* **조건:** 방향 그래프. 정점 V는 20,000 이하, 간선 E는 300,000 이하. 가중치는 10 이하의 자연수 (음수 없음!). 시작점에서 모든 정점으로의 최단 거리 출력하기.

```python
import sys
import heapq

# 입력 속도 최적화 (백준 필수)
input = sys.stdin.readline
INF = int(1e9) # 무한대를 의미하는 값으로 10억 설정

# v: 정점의 개수, e: 간선의 개수
v, e = map(int, input().split())
start = int(input())

# 1. 그래프 배열과 최단 거리 테이블 초기화
graph = [[] for _ in range(v + 1)]
distance = [INF] * (v + 1)

# 2. 방향 그래프 간선 정보 입력받기
for _ in range(e):
    u, dest, w = map(int, input().split())
    # u번 노드에서 dest번 노드로 가는 가중치가 w
    graph[u].append((dest, w)) 

def dijkstra(start):
    q = []
    # 시작 노드로 가기 위한 최단 경로는 0으로 설정하여 큐에 삽입
    # 힙은 튜플의 '첫 번째 값'을 기준으로 정렬하므로 반드시 (거리, 노드) 순서여야 함!
    heapq.heappush(q, (0, start))
    distance[start] = 0

    while q: # 큐가 비어있지 않다면
        # 가장 최단 거리가 짧은 노드에 대한 정보 꺼내기
        dist, now = heapq.heappop(q)

        # 🔥 이미 처리된 적 있는 노드라면 무시 (가지치기)
        # 기록된 최단 거리보다 현재 큐에서 방금 뽑은 거리가 크면 볼 필요도 없음
        if distance[now] < dist:
            continue

        # 현재 노드와 연결된 다른 인접 노드들을 확인
        for i in graph[now]:
            next_node = i[0]
            weight = i[1]
            cost = dist + weight # 현재 노드를 거쳐서 가는 비용

            # 거쳐가는 것이 기존에 알던 거리보다 더 저렴한 경우
            if cost < distance[next_node]:
                distance[next_node] = cost # 갱신
                heapq.heappush(q, (cost, next_node)) # 큐에 새로운 정보 삽입

# 다익스트라 알고리즘 수행
dijkstra(start)

# 3. 결과 출력
for i in range(1, v + 1):
    if distance[i] == INF:
        print("INF")
    else:
        print(distance[i])

```
