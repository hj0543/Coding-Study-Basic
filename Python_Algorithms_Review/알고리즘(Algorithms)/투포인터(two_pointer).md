투포인터(Two Pointers) 알고리즘은 1차원 배열이나 리스트에서 **두 개의 포인터(인덱스)**를 조작하여 원하는 결과를 빠르게 얻어내는 탐색 기법이다.

주로 연속된 부분 배열의 합을 구하거나, 정렬된 배열에서 특정 조건을 만족하는 두 원소의 쌍을 찾을 때 사용한다. 불필요한 중복 탐색을 줄여주기 때문에 시간 초과를 방지하는 강력한 무기가 된다.

### 💡 알고리즘 동작 과정

투포인터는 문제에 따라 포인터의 이동 방향이 크게 두 가지로 나뉜다.

1. **같은 방향 이동:** 두 포인터가 모두 첫 번째 원소에서 시작하여 끝을 향해 같은 방향으로 나아가는 방식 (ex: 연속된 부분 배열의 합 구하기)
2. **양 끝에서 이동:** 한 포인터는 맨 앞에서, 다른 한 포인터는 맨 뒤에서 시작하여 서로를 향해 좁혀오는 방식 (ex: 정렬된 배열에서 두 수의 합 구하기)

가장 대표적인 **'연속된 구간 합 구하기(같은 방향 이동)'**의 동작 과정은 다음과 같다.

1. 시작점(`start`)과 끝점(`end`)을 가리키는 두 포인터를 모두 배열의 첫 번째 인덱스(0)로 초기화한다.
2. 현재 구간의 합(`sum`)이 찾고자 하는 목표 값(`M`)과 같다면 카운트(또는 정답)를 1 증가시킨다.
3. 현재 구간 합이 `M`보다 작거나 같다면, `end`를 1 증가시키고 `sum`에 새로운 값을 더한다. (구간을 넓혀 합을 키움)
4. 현재 구간 합이 `M`보다 크다면, `sum`에서 현재 `start` 위치의 값을 빼고 `start`를 1 증가시킨다. (구간을 좁혀 합을 줄임)
5. 모든 배열을 탐색할 때까지 2~4번 과정을 반복한다.

> **참고:** 이 방식은 배열의 모든 원소가 **양수**일 때만 정상적으로 동작한다. 음수가 섞여 있다면 포인터를 이동시켰을 때 합이 증가할지 감소할지 예측할 수 없기 때문이다.

### 💻 투포인터 알고리즘 기본 코드 (Python)

구간 합을 구하는 가장 스탠다드한 형태의 투포인터 코드다.

```python
n = 5 # 데이터의 개수 N
m = 5 # 찾고자 하는 부분합 M
data = [1, 2, 3, 2, 5] # 전체 수열

count = 0
interval_sum = 0
end = 0

# start를 차례대로 증가시키며 반복
for start in range(n):
    # end를 가능한 만큼 이동시키며 구간 합 더하기
    while interval_sum < m and end < n:
        interval_sum += data[end]
        end += 1
        
    # 부분합이 m과 일치할 때 카운트 증가
    if interval_sum == m:
        count += 1
        
    # 다음 start로 넘어가기 위해, 현재 start 위치의 값을 구간 합에서 빼줌
    interval_sum -= data[start]

print(count) # 결과: 3

```

### ⏱️ 시간 복잡도

투포인터 알고리즘의 시간 복잡도는 **$O(N)$** 이다. ($N$: 데이터의 개수)
반복문 안에 `while`문이 중첩되어 있어 언뜻 보면 $O(N^2)$처럼 보일 수 있다. 하지만 `start`와 `end` 포인터는 단 한 번도 뒤로 되돌아가지 않고 배열의 끝을 향해 직진만 한다. 각각 최대 $N$번씩만 이동하므로 총연산 횟수는 $2N$을 넘지 않아 $O(N)$의 선형 시간을 보장한다. 이중 `for`문으로 풀면 $O(N^2)$이 걸릴 문제를 획기적으로 줄여주는 셈이다.

### 📝 예시 문제: 수들의 합 2 (백준 2003번)

$N$개의 수로 된 수열 $A[1], A[2], \dots, A[N]$ 이 있다. 이 수열의 $i$번째 수부터 $j$번째 수까지의 합 $A[i] + A[i+1] + \dots + A[j-1] + A[j]$가 $M$이 되는 경우의 수를 구하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에 $N$ ($1 \le N \le 10,000$), $M$ ($1 \le M \le 300,000,000$)이 주어진다. 다음 줄에는 $A[1], A[2], \dots, A[N]$이 공백으로 분리되어 주어진다. 각각의 $A[x]$는 30,000 이하의 자연수이다.
* **출력:** 첫째 줄에 경우의 수를 출력한다.

---

### 💡 풀이 접근법

문제 이름과 요구사항만 봐도 '연속된 부분 배열의 합'을 묻는 투포인터의 근본 문제다.
수열의 원소가 모두 '자연수(양수)'라고 명시되어 있으므로, 구간이 길어지면 합이 커지고 구간이 짧아지면 합이 작아진다는 규칙이 성립한다. 따라서 앞서 정리한 투포인터 알고리즘 뼈대 코드를 그대로 적용하면 완벽하게 풀린다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

# 데이터의 개수 n, 찾고자 하는 부분합 m
n, m = map(int, input().split())
arr = list(map(int, input().split()))

count = 0
interval_sum = 0
end = 0

# start 포인터를 0부터 끝까지 순회
for start in range(n):
    # 구간 합이 m보다 작고, end가 배열 범위를 넘지 않는 동안 end 이동
    while interval_sum < m and end < n:
        interval_sum += arr[end]
        end += 1
        
    # 부분합이 m과 같다면 정답 카운트 1 증가
    if interval_sum == m:
        count += 1
        
    # start 포인터가 다음 칸으로 이동할 예정이므로, 
    # 기존 start 위치의 값을 구간 합에서 제외
    interval_sum -= arr[start]

print(count)

```
