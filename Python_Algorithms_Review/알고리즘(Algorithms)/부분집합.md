부분집합(Power Set)은 주어진 집합의 원소들로 만들 수 있는 **'모든 가능한 조합'**을 구하는 완전 탐색의 핵심 기본기다.

원소가 $N$개일 때, 각 원소는 **'포함되거나(O)'** 혹은 **'포함되지 않거나(X)'** 딱 2가지 상태만 가지므로 전체 부분집합의 개수는 무조건 **$2^N$개**가 된다. 공집합(아무것도 안 뽑은 상태)부터 전체 집합(다 뽑은 상태)까지 싹 다 구해야 할 때 사용한다.

### 💡 부분집합을 구현하는 3가지 무기

파이썬 코딩 테스트에서는 상황에 따라 다음 3가지 방법 중 하나를 골라 쓴다. 가장 직관적인 DFS부터, 앞서 배운 비트마스킹의 마법까지 모두 보여준다.

---

#### 💻 1. 백트래킹 (DFS) 구현법 - 가장 근본적인 방법

재귀를 탈 때마다 현재 원소를 포함할지 말지 결정하고, 끝까지 도달(`depth == N`)했을 때 결과를 출력하는 방식이다. **조건에 따라 중간에 탐색을 끊어버리는 '가지치기(Pruning)'가 필요할 때** 무조건 이 방식을 써야 한다.

```python
arr = ['A', 'B', 'C']
n = len(arr)

def dfs_subset(idx, subset):
    # 기저 조건: 모든 원소에 대해 포함/미포함 결정을 마쳤을 때
    if idx == n:
        print(*subset)
        return
        
    # 1. 현재 원소를 '포함(O)'하는 경우
    dfs_subset(idx + 1, subset + [arr[idx]])
    
    # 2. 현재 원소를 '미포함(X)'하는 경우
    dfs_subset(idx + 1, subset)

print("--- 1. DFS 부분집합 ---")
dfs_subset(0, [])

```

#### 💻 2. 비트마스킹 (Bitmasking) 구현법 - 압도적인 속도와 짧은 코드

원소가 3개라면, 0(`000`)부터 7(`111`)까지의 숫자를 이진수로 표현해서 각 자리의 비트가 1이면 포함, 0이면 미포함으로 간주하는 천재적인 방법이다. **코드가 가장 짧고 빠르며, 메모리도 적게 먹는다.**

```python
arr = ['A', 'B', 'C']
n = len(arr)

print("--- 2. 비트마스킹 부분집합 ---")
# 1 << n은 2^n과 같다. 즉, 0부터 2^n - 1까지 반복한다.
for i in range(1 << n):
    subset = []
    # 각 숫자 i에 대해 0번째~n-1번째 비트가 켜져 있는지 확인한다.
    for j in range(n):
        if i & (1 << j): # j번째 비트가 1이라면 원소 포함!
            subset.append(arr[j])
    print(subset)

```

#### 💻 3. `itertools` 구현법 - 파이썬의 축복

조합(`combinations`)을 0개 뽑을 때부터 $N$개 뽑을 때까지 `for`문을 돌려버리면 그게 곧 부분집합이다. **단순히 모든 경우를 다 구해서 확인만 하면 될 때** 가장 편하다.

```python
from itertools import combinations

arr = ['A', 'B', 'C']

print("--- 3. itertools 부분집합 ---")
# 0개부터 n개까지 뽑는 모든 조합을 구함
for r in range(len(arr) + 1):
    for comb in combinations(arr, r):
        print(list(comb))

```

*(참고: 위 3가지 코드 모두 실행 결과는 공집합 `[]`부터 `['A', 'B', 'C']`까지 동일하게 출력된다!)*

---

### ⏱️ 시간 복잡도와 주의점

모든 경우를 다 만들어야 하므로 시간 복잡도는 **$O(2^N)$** 이다.
지수 스케일로 폭발적으로 경우의 수가 늘어나기 때문에, 코딩 테스트에서 배열의 길이 $N$이 **20 이하**로 아주 작게 주어지는 문제에서만 부분집합 완전 탐색을 시도해야 한다. 그 이상 넘어가면 시간 초과가 난다.
