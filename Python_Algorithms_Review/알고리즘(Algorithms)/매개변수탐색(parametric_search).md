매개 변수 탐색(Parametric Search)은 **최적화 문제(특정 조건을 만족하는 최댓값이나 최솟값을 구하는 문제)를 결정 문제('예' 혹은 '아니오'로 답하는 문제)로 바꾸어 해결하는 기법**이다.

이분 탐색(Binary Search)을 응용한 가장 대표적인 형태이며, 코딩 테스트에서 "가장 긴 길이", "최소 시간", "최대 간격" 등을 구하라는 문제가 나왔을 때 데이터의 범위가 억 단위로 무지막지하게 크다면 십중팔구 이 매개 변수 탐색 문제다.

### 💡 알고리즘 동작 과정

정답이 될 수 있는 **'값의 범위'**를 이분 탐색으로 좁혀가며, 중간값(`mid`)이 조건에 부합하는지 계속 테스트하는 방식으로 동작한다.

1. 정답이 될 수 있는 가능한 최솟값을 `start`, 최댓값을 `end`로 설정한다.
2. `start`와 `end`의 중간값인 `mid`를 계산한다. 이 `mid`가 **'우리가 찾고자 하는 임시 정답(매개 변수)'**이 된다.
3. 임시 정답 `mid`를 조건 판별 함수에 넣어 **"이 값이 조건을 만족하는가?"(결정 문제)**를 확인한다.
4. **조건을 만족한다면(YES):** 일단 이 `mid` 값을 정답 변수에 기록해 둔다. 그리고 "이것보다 더 좋은(더 크거나 더 작은) 최적의 해가 있는지" 알아보기 위해 탐색 범위를 좁힌다. (예: 최댓값을 찾는 문제라면 `start = mid + 1`로 올려서 더 큰 값을 테스트해 본다.)
5. **조건을 만족하지 않는다면(NO):** 이 `mid` 값으로는 어림도 없다는 뜻이므로, 조건을 만족할 수 있는 방향으로 탐색 범위를 좁힌다. (예: 최댓값을 찾는 문제라면 `end = mid - 1`로 낮춰서 더 작은 값을 테스트해 본다.)
6. `start`가 `end`보다 커질 때까지 2~5번 과정을 반복한다.

### 💻 매개 변수 탐색 기본 코드 (Python)

어떤 문제든 매개 변수 탐색은 아래의 뼈대에서 거의 벗어나지 않는다. `check()` 함수를 어떻게 구현하느냐가 문제 풀이의 핵심이다.

```python
# 조건을 만족하는지 확인하는 결정 함수 (Yes/No 반환)
def check(mid):
    # mid 값을 기준으로 문제의 조건을 테스트하는 로직
    # 만족하면 True, 아니면 False 반환
    pass

# 탐색할 값의 최소/최대 범위 설정
start = 1
end = 1000000000
result = 0 # 최적의 값을 저장할 변수

# 이분 탐색 진행
while start <= end:
    mid = (start + end) // 2
    
    # mid 값이 조건을 만족한다면
    if check(mid):
        result = mid       # 일단 정답으로 기록
        start = mid + 1    # 더 큰 값도 가능한지 탐색 (최댓값을 구하는 경우)
        # 만약 최솟값을 구하는 문제라면 end = mid - 1 로 설정하여 더 작은 값을 탐색해야 함
    
    # mid 값이 조건을 만족하지 못한다면
    else:
        end = mid - 1      # 조건을 만족하도록 값을 낮춤 (최댓값을 구하는 경우)
        # 만약 최솟값을 구하는 문제라면 start = mid + 1 로 설정해야 함

print(result)

```

### ⏱️ 시간 복잡도

탐색 범위를 한 번 확인할 때마다 범위가 절반으로 줄어드므로 이분 탐색 자체는 $O(\log X)$가 걸린다. ($X$: 탐색 범위의 크기)
이때 매번 `check()` 함수를 실행하여 데이터 $N$개를 확인해야 하므로, 총시간 복잡도는 **$O(N \log X)$**가 된다. 탐색 범위가 10억 단위라도 $\log X$는 30 언저리이므로, 데이터 $N$번만 순회하면 넉넉하게 시간 안에 통과할 수 있다.

### 📝 예시 문제: 랜선 자르기 (백준 1654번)

집에서 시간을 보내던 오영식은 박성원의 부름을 받고 급히 달려왔다. 박성원이 캠프 때 쓸 N개의 랜선을 만들어야 하는데 너무 바빠서 영식이에게 도움을 청했다.
이미 오영식은 자체적으로 K개의 랜선을 가지고 있다. 그러나 K개의 랜선은 길이가 제각각이다. 박성원은 랜선을 모두 N개의 같은 길이의 랜선으로 만들고 싶었기 때문에 K개의 랜선을 잘라서 만들어야 한다. (중략)
만들 수 있는 최대 랜선의 길이를 구하는 프로그램을 작성하시오.

* **입력:** 첫째 줄에는 오영식이 이미 가지고 있는 랜선의 개수 K, 그리고 필요한 랜선의 개수 N이 입력된다. ($1 \le K \le 10,000$, $1 \le N \le 1,000,000$, 항상 $K \le N$) 그 후 K줄에 걸쳐 이미 가지고 있는 각 랜선의 길이가 센티미터 단위의 정수로 입력된다. 랜선의 길이는 $2^{31}-1$보다 작거나 같은 자연수이다.
* **출력:** 첫째 줄에 N개를 만들 수 있는 랜선의 최대 길이를 센티미터 단위의 정수로 출력한다.

---

### 💡 풀이 접근법

랜선의 최대 길이가 무려 21억(2^31-1)이다. 1부터 21억까지 1씩 증가시키며 랜선이 몇 개 나오는지 확인한다면 백 퍼센트 시간 초과가 난다.

1. **최적화 문제:** "N개 이상의 랜선을 만들 수 있는 **최대 길이**는 얼마인가?"
2. **결정 문제:** "랜선을 **`mid` 길이**로 잘랐을 때, 나오는 랜선의 개수가 **N개 이상인가(Yes/No)**?"

이분 탐색의 범위를 랜선 길이(1 ~ 가진 랜선 중 최댓값)로 설정하고, 중간 길이(`mid`)로 잘랐을 때 나오는 개수를 센다. 개수가 N개 이상이면 "길이를 더 늘려도 되겠다" 판단하여 `start`를 올리고, N개 미만이면 "길이가 너무 길어서 개수가 부족하다" 판단하여 `end`를 낮추면 된다.

---

### 💻 풀이 코드 (Python)

```python
import sys
input = sys.stdin.readline

k, n = map(int, input().split())
lines = [int(input()) for _ in range(k)]

# 정답(랜선의 길이)이 될 수 있는 최소 길이와 최대 길이
start = 1
end = max(lines)

result = 0

while start <= end:
    mid = (start + end) // 2 # 임시 랜선 길이
    
    count = 0
    # 현재 길이(mid)로 잘랐을 때 나오는 랜선의 총개수 계산
    for line in lines:
        count += line // mid
        
    # 랜선이 목표치(n) 이상 만들어졌다면 (조건 만족)
    if count >= n:
        result = mid       # 현재 길이를 정답으로 우선 저장
        start = mid + 1    # '최대 길이'를 찾기 위해 더 길게 잘라봄
    # 랜선이 목표치(n)보다 적게 만들어졌다면 (조건 불만족)
    else:
        end = mid - 1      # 랜선이 너무 길어서 개수가 부족한 것이므로 더 짧게 잘라봄

print(result)

```
