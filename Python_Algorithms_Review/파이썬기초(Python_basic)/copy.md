코딩 테스트에서 2차원 배열(격자, 보드판 등)을 다루는 시뮬레이션 문제를 풀 때, 원본 배열을 훼손하지 않고 상태를 저장해 두려면 이 둘의 차이를 뼈저리게 알고 있어야 한다. 단순히 `=` 기호로 복사했다가는 원본 데이터까지 다 망가져서 디버깅 지옥에 빠지기 십상이다.

### 💡 파이썬의 복사 메커니즘 3단계

파이썬에서 변수에 데이터를 복사하는 방식은 크게 3가지로 나뉜다. 가장 헷갈리기 쉬운 **2차원 리스트(리스트 안에 리스트가 있는 경우)**를 기준으로 설명하겠다.

#### 1. 단순 할당 (Assignment, `=`)

복사가 아니라 **'이름표만 하나 더 붙이는 것'**이다. 완전히 똑같은 메모리 주소를 가리키기 때문에, 한쪽을 수정하면 다른 쪽도 무조건 같이 바뀐다.

#### 2. 얕은 복사 (Shallow Copy, `copy.copy()` 또는 슬라이싱 `[:]`)

**가장 바깥쪽의 껍데기(리스트)만 새로 만들고, 그 안에 있는 알맹이(내부 리스트)들은 여전히 원본의 주소를 묶어두는 방식**이다.

* 1차원 리스트에서는 완벽하게 복사되지만, 2차원 리스트에서는 내부 리스트를 수정하면 원본도 같이 바뀌어버리는 치명적인 함정이 있다.

#### 3. 깊은 복사 (Deep Copy, `copy.deepcopy()`)

껍데기뿐만 아니라 **그 안에 들어있는 모든 내부 객체들까지 재귀적으로 싹 다 새롭게 복사**해서, 원본과 아예 남남인 독립적인 객체를 만들어 내는 방식이다.

---

### 💻 얕은 복사 vs 깊은 복사 완벽 비교 코드 (Python)

```python
import copy

# ------------------------------------------------
print("1. 단순 할당 (Assignment)")
a = [[1, 2], [3, 4]]
b = a               # a의 메모리 주소 자체를 복사

b[0][0] = 99        # b를 수정했는데
print(a)            # 결과: [[99, 2], [3, 4]] (a도 같이 바뀌어버림!)

# ------------------------------------------------
print("\n2. 얕은 복사 (Shallow Copy)")
a = [[1, 2], [3, 4]]
# b = a[:] 와 동일함
b = copy.copy(a)    # 겉 껍데기만 복사

# 겉 껍데기에 요소를 추가하는 건 원본에 영향이 없음
b.append([5, 6])
print(a)            # 결과: [[1, 2], [3, 4]] (a는 그대로)

# 하지만 내부 리스트의 값을 수정하면?
b[0][0] = 99
print(a)            # 결과: [[99, 2], [3, 4]] (내부 리스트 주소를 공유하므로 a도 바뀜 😱)

# ------------------------------------------------
print("\n3. 깊은 복사 (Deep Copy)")
a = [[1, 2], [3, 4]]
b = copy.deepcopy(a) # 내부 알맹이까지 완벽하게 새로운 객체로 생성

b[0][0] = 99        # b의 내부 리스트를 수정
print(a)            # 결과: [[1, 2], [3, 4]] (완벽히 방어! a는 원본 유지 🛡️)
print(b)            # 결과: [[99, 2], [3, 4]]

```

---

### ⏱️ 시간 복잡도와 주의할 점

* `copy.deepcopy()`는 내부를 싹 다 뒤져서 새롭게 만들어내기 때문에 속도가 **매우 느리다.** 코딩 테스트에서 `for`문을 돌 때마다 2차원 배열을 `deepcopy` 하면 $O(N^2)$ 이상의 시간이 계속 추가되어 시간 초과(TLE)를 받을 확률이 매우 높아진다.
* **꿀팁:** 파이썬 리스트 컴프리헨션을 쓰면 `deepcopy` 모듈을 쓰는 것보다 훨씬 빠르게 2차원 배열을 깊은 복사할 수 있다!

```python
a = [[1, 2], [3, 4]]
# deepcopy보다 훨씬 빠른 2차원 배열 복사 꼼수
b = [row[:] for row in a] 

```
