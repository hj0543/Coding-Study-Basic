프로그래밍에서 수십, 수백만 번의 반복 작업을 단 몇 줄로 끝내주는 핵심 문법이 바로 **`for`**와 **`while`** 반복문이다.

코딩 테스트에서는 데이터의 개수나 탐색 조건에 따라 이 둘을 적절히 골라 써야 시간 초과(TLE)나 무한 루프의 늪에 빠지지 않는다. 쉽게 말해 `for`문은 **"정해진 횟수나 데이터만큼 돌릴 때"**, `while`문은 **"특정 조건을 만족할 때까지 돌릴 때"** 사용한다.

---

## 💡 for 문 (정해진 구간 순회)

`for`문은 리스트, 튜플, 문자열 같은 '반복 가능한(Iterable) 데이터'의 처음부터 끝까지 순서대로 하나씩 꺼내보거나, `range()` 함수를 이용해 **정해진 횟수만큼 정확하게 반복**할 때 사용한다.

#### 💻 for 문 핵심 코드 (Python)

```python
# 1. range()를 이용한 단순 횟수 반복
# range(시작, 끝, 간격) -> 끝 번호는 포함되지 않음에 주의!
for i in range(1, 6):
    print(i, end=' ')
# 결과: 1 2 3 4 5

print()

# 2. 리스트의 원소를 직접 순회 (코딩 테스트에서 가장 많이 씀)
arr = [10, 20, 30]
for num in arr:
    print(num, end=' ')
# 결과: 10 20 30

print()

# 3. 인덱스와 값을 동시에 뽑아내는 enumerate() (🔥실전 꿀팁)
# 데이터의 '위치(인덱스)'도 알아야 하고 '값'도 알아야 할 때 무조건 쓴다.
words = ['A', 'B', 'C']
for idx, val in enumerate(words):
    print(f"인덱스 {idx}: {val}")
# 결과:
# 인덱스 0: A
# 인덱스 1: B
# 인덱스 2: C

```

## 💡 while 문 (조건 기반 반복)

`while`문은 반복 횟수가 명확하지 않고, **"이 조건이 참(True)인 동안에는 계속 돌아라!"**라고 명령할 때 사용한다.
앞서 배운 BFS(너비 우선 탐색)에서 `while queue:`(큐가 빌 때까지 돌아라)처럼 조건이 동적으로 변할 때 아주 강력한 힘을 발휘한다.

단, 종료 조건(`break`나 조건식의 `False` 변화)을 제대로 명시하지 않으면 영원히 끝나지 않는 **무한 루프**에 빠지므로 주의해야 한다.

#### 💻 while 문 핵심 코드 (Python)

```python
# 1. 기본 while 문 (조건이 참일 때까지 반복)
count = 5
while count > 0:
    print(count, end=' ')
    count -= 1  # 이 조건 변화가 없으면 무한 루프에 빠진다!
# 결과: 5 4 3 2 1

print()

# 2. 무한 루프와 break, continue (🔥탐색 알고리즘 필수)
target = 7
num = 1

while True: # 일단 무한히 반복
    if num == target:
        print(f"찾았다! {num}")
        break  # target을 찾으면 즉시 반복문을 완전히 탈출한다.
        
    if num % 2 == 0:
        num += 1
        continue # 짝수라면 아래 코드를 무시하고 다음 반복(조건 체크)으로 건너뛴다.
        
    num += 1

```

### ⚔️ for vs while 한눈에 비교

| 구분 | `for` 문 | `while` 문 |
| --- | --- | --- |
| **핵심 목적** | **'정해진 범위'**나 **'데이터 묶음'**을 순회할 때 | **'특정 조건'**이 만족되는 동안 계속 반복할 때 |
| **장점** | 코드가 간결하고 무한 루프에 빠질 위험이 거의 없다. | 반복 횟수를 미리 알 수 없는 복잡한 상황에 유연하게 대처할 수 있다. |
| **코테 실전 활용** | 배열 탐색, 완전 탐색(N중 루프), DP 테이블 채우기 | **BFS (큐가 빌 때까지)**, **이진 탐색 (start <= end 일 때까지)**, 투 포인터 |

---

반복문(`for`, `while`)의 핸들과 브레이크 역할을 하는 **`break`**와 **`continue`**까지 같이 묶어서 완벽하게 정리해 줄게!

단순히 100번, 1000번 반복하는 건 누구나 짤 수 있지만, 코딩 테스트에서는 "특정 조건을 만족하면 즉시 멈춰라!" 혹은 "쓸데없는 데이터는 건너뛰어라!" 같은 조건이 무조건 붙어. 이때 이 두 가지 제어문이 없으면 코드가 `if`문으로 끝없이 깊어지는 지옥(Indent Hell)을 맛보게 돼.

---

## 💡 핵심 개념: 반복문의 흐름 통제하기

* **`break` (탈출):** 자신을 감싸고 있는 **가장 가까운 반복문을 완전히 부수고(탈출하고)** 다음 코드로 넘어간다. "목적 달성했으니 이제 그만해!"라고 할 때 쓴다.
* **`continue` (건너뛰기):** 반복문을 탈출하지는 않지만, **이번 턴(회차)의 남은 코드를 무시하고 곧바로 다음 턴으로 점프**한다. "이 데이터는 볼 필요 없으니 다음 걸로 넘어가자!"라고 할 때 쓴다.

### 💻 break와 continue의 완벽한 차이 (Python)

단순한 1부터 10까지의 반복문에서 이 두 녀석이 어떻게 흐름을 비틀어버리는지 직관적으로 보여줄게.

#### 1. `break` (반복문 파괴)

```python
print("--- break 테스트 ---")
for i in range(1, 11):
    if i == 5:
        print("5를 찾았다! 반복문 종료.")
        break  # i가 5가 되는 순간, for문 자체가 완전히 끝남
    
    print(i, end=' ')
    
# 결과: 1 2 3 4 5를 찾았다! 반복문 종료.
# (6부터 10까지는 아예 실행되지 않음)

```

#### 2. `continue` (이번 턴만 스킵)

```python
print("--- continue 테스트 ---")
for i in range(1, 11):
    if i % 2 == 0:
        continue # 짝수라면? 아래 코드를 쿨하게 무시하고 다음 숫자(홀수)로 넘어감!
        
    print(i, end=' ')
    
# 결과: 1 3 5 7 9
# (반복문은 끝까지 돌지만, 짝수일 때는 print()가 실행되지 않고 스킵됨)

```

### 🔥 코딩 테스트 실전 압축: 무한 루프와 가지치기

실제 알고리즘 문제에서 `while`, `for`와 결합되어 쓰이는 가장 대표적인 두 가지 패턴이야. 이 패턴은 무조건 손에 익혀둬야 해!

#### 패턴 1: `while True:` + `break` (조건이 복잡할 때)

종료 조건이 여러 개이거나 반복문을 시작하기 전에는 알 수 없을 때, 일단 무한 루프를 돌려놓고 내부에서 `break`로 탈출하는 방식을 엄청나게 많이 써.

```python
hp = 100
attack_damage = 15
turn = 0

# 몬스터의 체력이 0 이하가 될 때까지 무한 반복
while True:
    turn += 1
    hp -= attack_damage
    print(f"{turn}턴: 공격! 남은 체력 {hp}")
    
    # 종료 조건 (탈출구)
    if hp <= 0:
        print("몬스터 처치 완료!")
        break 

```

#### 패턴 2: `continue`를 이용한 코드 깊이 줄이기 (가지치기)

`continue`를 쓰면 불필요한 `if-else` 중첩을 없애서 코드를 훨씬 깔끔하게(가독성 좋게) 만들 수 있어. 탐색 알고리즘에서 **'조건에 안 맞는 길은 빨리 포기(Pruning)'**할 때 필수야.

```python
# 안 좋은 예 (if문이 깊어짐)
for data in [1, -5, 3, -2, 7]:
    if data > 0: # 양수일 때만 처리
        # 여기서 복잡한 로직이 시작되면 코드가 오른쪽으로 계속 밀려남
        print(f"처리 중: {data}")

# 좋은 예 (continue로 예외를 먼저 쳐냄)
for data in [1, -5, 3, -2, 7]:
    if data <= 0: 
        continue # 음수면 가차 없이 버리고 다음 데이터로!
        
    # 코드가 밀려나지 않고 깔끔하게 유지됨
    print(f"처리 중: {data}")

```

### 💻 플래그(Flag) 변수
플래그(Flag) 변수는 이름 그대로 반복문 안에서 **'특정 조건이 달성되었는지 깃발을 들어 표시하는'** 상태 기록용 변수다.

주로 `True`나 `False` 값을 가지는 불리언(Boolean) 형태로 사용되며, 반복문의 흐름을 제어하거나 탐색이 끝난 후 특정 사건이 발생했었는지 판단할 때 필수적으로 쓰인다. 체육대회 청기 백기 게임에서 "조건에 맞으면 청기 올려!"라고 하는 것과 완벽하게 똑같은 원리다.

코딩 테스트에서 플래그 변수를 꺼내 들어야 하는 순간은 크게 두 가지다.

#### 1. 이중(다중) 반복문을 한 번에 탈출할 때 (🔥매우 중요)

앞서 배운 `break` 문은 **자신을 감싸고 있는 가장 가까운 반복문 딱 하나만 부수고 탈출**한다.
따라서 2차원 배열을 탐색하는 이중 `for`문에서 정답을 찾았을 때, 안쪽 `for`문에서 `break`를 걸어봤자 바깥쪽 `for`문은 멈추지 않고 계속 돌아간다. 이때 바깥쪽 반복문까지 완전히 멈추게 하려면 플래그 변수가 반드시 필요하다.

```python
board = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
target = 5

# 깃발을 아직 들지 않은 상태(False)로 초기화
is_found = False 

for i in range(3):
    for j in range(3):
        if board[i][j] == target:
            print(f"찾았다! 위치: ({i}, {j})")
            is_found = True # 1. 타겟을 찾으면 깃발을 번쩍 든다!
            break           # 2. 안쪽 반복문 탈출
            
    # 바깥쪽 반복문의 끝에서 깃발 상태를 확인한다.
    if is_found:            # 3. 깃발이 올라가 있다면?
        break               # 4. 바깥쪽 반복문도 마저 탈출한다!

```

#### 2. 반복문 종료 후 '상태'를 확인해야 할 때

수만 개의 데이터를 다 확인했는데도 **"단 한 번이라도 조건을 만족한 적이 있는지"** 알아야 할 때 사용한다. 예를 들어 소수(Prime Number)를 판별할 때 쓴다.

```python
n = 11
is_prime = True # 일단 소수라고 깃발을 들어둔다.

for i in range(2, n):
    if n % i == 0:
        is_prime = False # 나누어 떨어지면 소수가 아니므로 깃발을 내린다.
        break

# 반복문이 끝난 후 깃발의 최종 상태를 보고 판단한다.
if is_prime:
    print(f"{n}은(는) 소수입니다.")
else:
    print(f"{n}은(는) 소수가 아닙니다.")

```

### 🍯 코딩 테스트 실전 꿀팁: 파이썬의 `for-else` 문법

파이썬에는 플래그 변수를 쓰지 않고도 "반복문이 중간에 `break`로 끊기지 않고 끝까지 무사히 다 돌았는가?"를 판별해 주는 기가 막힌 **`for-else`** 문법이 존재한다.

```python
n = 11

for i in range(2, n):
    if n % i == 0:
        print(f"{n}은(는) 소수가 아닙니다.")
        break
else:
    # 이 else문은 for문이 break를 만나지 않고 끝까지 정상적으로 다 돌았을 때만 실행된다!
    print(f"{n}은(는) 소수입니다.")

```

> 플래그 변수 `is_prime`을 만들고 검사하는 과정을 이 `else` 구문 하나로 완벽하게 압축할 수 있다.
