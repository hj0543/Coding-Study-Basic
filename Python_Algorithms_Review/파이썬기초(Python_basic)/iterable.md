이터러블(Iterable)은 단어 뜻 그대로 **'반복 가능한(Iterate + able)'** 객체를 의미한다.

파이썬에서 `for`문이나 리스트 컴프리헨션처럼, 내부의 원소들을 한 번에 하나씩 차례대로 꺼내어 쓸 수 있는 모든 데이터 타입이 바로 이터러블이다. 데이터가 줄줄이 사탕처럼 엮여 있어서 하나씩 당겨서 빼먹을 수 있는 주머니라고 생각하면 쉽다.

---

### 💡 1. 무엇이 이터러블인가?

파이썬에서 우리가 흔히 쓰는 데이터를 묶어두는 자료형은 사실상 전부 이터러블이다.

* **순서가 있는 이터러블 (시퀀스):** `list`(리스트), `tuple`(튜플), `str`(문자열)
* **순서가 없는 이터러블:** `dict`(딕셔너리), `set`(집합)
* **기타:** `range()` 함수가 반환하는 객체, `map()`이나 `zip()`의 결과물, 파일을 읽어온 객체 등

### 💻 2. 코드로 보는 이터러블의 특성 (Python)

이터러블 객체는 단순히 `for`문을 돌릴 수 있다는 것 외에도, 코딩 테스트에서 자주 쓰는 파이썬의 핵심 문법들과 완벽하게 연동된다.

#### ① `for`문과 리스트 컴프리헨션의 재료가 된다

```python
# 문자열(str)도 이터러블이므로 원소를 하나씩 꺼낼 수 있다.
text = "ABC"
for char in text:
    print(char, end=' ') 
# 결과: A B C 

# 딕셔너리(dict)를 for문에 넣으면 기본적으로 'Key'를 하나씩 꺼내준다.
my_dict = {'a': 1, 'b': 2}
keys = [k for k in my_dict]
print(keys) # 결과: ['a', 'b']

```

#### ② 언패킹(`*`) 연산자로 포장을 풀 수 있다

앞서 패킹/언패킹에서 배웠던 별표(`*`) 기호는 **오직 이터러블 객체 앞에만** 붙일 수 있다.

```python
numbers = (1, 2, 3) # 튜플 (이터러블)
print(*numbers)     # 결과: 1 2 3

```

#### ③ 내장 함수의 인수로 들어간다

`list()`, `tuple()`, `set()` 같은 형변환 함수나, `sum()`, `max()`, `min()`, `map()`, `zip()` 같은 파이썬의 강력한 내장 함수들은 **반드시 이터러블을 인수로 요구**한다.

```python
arr = [10, 20, 30] # 리스트 (이터러블)

print(sum(arr)) # 결과: 60
print(max(arr)) # 결과: 30

```

### 🍯 코딩 테스트 실전 팁: 이터레이터(Iterator)와의 차이

가끔 에러 메시지에 'Iterator'라는 말이 나올 때가 있다.

* **이터러블(Iterable):** 반복 '할 수 있는' 데이터의 집합 (예: 리스트 자체)
* **이터레이터(Iterator):** 이터러블에서 값을 차례대로 '꺼내주는 기계' (예: `iter(리스트)`의 결과물, 또는 `map` 객체)

`map()`이나 `zip()` 함수를 실행하면 리스트가 아니라 요상한 객체 주소(`<map object at ...>`)가 나오는 것을 본 적이 있을 것이다. 이들은 데이터를 한 번에 메모리에 다 올려두지 않고, **"다음 거 내놔!"(`next()`)**라고 할 때만 하나씩 값을 뱉어내는 '이터레이터'이기 때문이다. 그래서 결과를 눈으로 보거나 배열처럼 인덱스로 접근하려면 `list(map(...))`처럼 다시 형변환을 해주는 것이다.

### 📂 어디에 넣으면 좋을까?

* **분류:** **[파이썬 기초 - 핵심 개념]** 또는 **[파이썬 기초 - 자료형]**
* **사유:** `for`문, 언패킹, 내장 함수(`map`, `zip`, `sum`) 등 파이썬의 편리한 문법들이 전부 이 '이터러블'이라는 공통된 규격을 바탕으로 돌아가기 때문이다. 이 개념을 알면 "여기에 리스트 대신 문자열이나 집합을 넣어도 똑같이 동작하겠구나"라는 응용력이 생긴다.
