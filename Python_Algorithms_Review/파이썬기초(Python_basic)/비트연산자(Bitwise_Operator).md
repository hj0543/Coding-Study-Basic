비트 연산자는 데이터를 이진수(0과 1) 비트 단위로 직접 조작하는 연산자다.   
CPU가 가장 좋아하는 연산이기 때문에 속도가 극단적으로 빠르며, 코딩 테스트에서는 배열 대신 정수 하나로 여러 상태를 관리하는 **'비트마스킹(Bitmasking)'** 기법이나 수학적 최적화를 위해 핵심적으로 사용한다.   

### 💡 비트 연산자의 종류와 동작 원리

| 연산자 | 기호 | 설명 |
| --- | --- | --- |
| **AND** | `&` | 두 비트가 **모두 1**일 때만 1을 반환한다. |
| **OR** | `|` | 두 비트 중 **하나라도 1**이면 1을 반환한다. |
| **XOR** | `^` | 두 비트가 **서로 다를 때**만 1을 반환한다. |
| **NOT** | `~` | 비트를 **반전**시킨다. (0은 1로, 1은 0으로) |
| **Left Shift** | `<<` | 비트를 왼쪽으로 밀어낸다. ($A \times 2^B$의 효과) |
| **Right Shift** | `>>` | 비트를 오른쪽으로 밀어낸다. ($A \div 2^B$의 효과) |

---

### 💻 비트 연산 기본 예제 (Python)

```python
a = 10  # 이진수: 1010
b = 12  # 이진수: 1100

# 1. 기본 논리 연산
print(a & b)  # 결과: 8  (1000)
print(a | b)  # 결과: 14 (1110)
print(a ^ b)  # 결과: 6  (0110)

# 2. NOT 연산 (파이썬은 2의 보수를 사용하므로 -(a + 1)이 됨)
print(~a)     # 결과: -11

# 3. 시프트(Shift) 연산 - 코딩테스트 최적화 단골
print(a << 1) # 결과: 20 (10을 2배 한 것과 같음)
print(a >> 1) # 결과: 5  (10을 2로 나눈 몫과 같음)

```

### 🔥 코딩 테스트 실전 압축: 비트마스킹 (Bitmasking)

코딩 테스트에서 비트 연산이 나오는 이유는 십중팔구 이 **비트마스킹** 때문이다. `visited` 배열을 만들지 않고, **정수 1개만으로 최대 64개의 방문 상태나 집합을 표현**할 수 있다. 속도도 빠르고 메모리도 거의 쓰지 않는다.

예를 들어, 0번, 2번, 3번 아이템을 가지고 있는 상태를 이진수 `1101` (십진수 13)로 표현할 수 있다.

```python
# 현재 상태: 13 (이진수 1101)
state = 13

# 1. 원소 추가 (1번 아이템 획득하기) -> OR 연산 사용
state |= (1 << 1)  # 1101 | 0010 = 1111 (15)

# 2. 원소 확인 (2번 아이템이 있는지?) -> AND 연산 사용
if state & (1 << 2):
    print("2번 아이템을 보유하고 있다.")

# 3. 원소 삭제 (3번 아이템 버리기) -> AND와 NOT 연산 조합
state &= ~(1 << 3) # 1111 & 0111 = 0111 (7)

# 4. 원소 토글 (0번 아이템이 있으면 버리고, 없으면 획득) -> XOR 연산 사용
state ^= (1 << 0)  # 0111 ^ 0001 = 0110 (6)

```

---

비트 연산은 처음엔 직관적이지 않지만, 한 번 익혀두면 상태 공간 트리를 탐색할 때 메모리와 실행 시간을 획기적으로 줄여주는 치트키가 된다. 앞서 배웠던 펜윅 트리(Fenwick Tree)의 `i & -i` 연산도 이 비트의 마법을 이용해 가장 마지막 1의 위치를 찾아내는 최적화 기법이다.
