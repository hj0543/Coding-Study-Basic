파이썬에서 데이터를 다룰 때 가장 뼈대가 되는 개념이자, 코딩 테스트에서 원인 모를 버그(참조 오류)를 만들어내는 주범이 바로 **뮤터블(Mutable, 가변)**과 **이뮤터블(Immutable, 불변)**의 차이다.

이름 그대로 생성된 후에 **'값을 수정할 수 있는가(Mutable)'** 아니면 **'절대 수정할 수 없는가(Immutable)'**를 뜻한다.

---

### 💡 1. 자료형의 완벽한 분류

파이썬의 모든 자료형은 이 두 가지 중 하나로 명확하게 나뉜다.

| 구분 | 의미 | 해당되는 파이썬 자료형 |
| --- | --- | --- |
| **Immutable (불변)** | 한 번 만들어지면 **절대** 값을 바꿀 수 없다. | `int`(정수), `float`(실수), `str`(문자열), **`tuple`(튜플)**, `bool`(참/거짓) |
| **Mutable (가변)** | 생성된 후에도 내부의 값을 마음대로 바꿀 수 있다. | **`list`(리스트)**, **`dict`(딕셔너리)**, **`set`(집합)** |

---

### 💻 2. 코드를 통한 직관적 이해 (Python)

#### ❌ Immutable (불변)의 특징: "수정 불가, 새로 만들어야 함"

문자열은 불변 객체다. 일부만 슬쩍 바꾸려고 하면 파이썬이 에러를 뱉어버린다.

```python
text = "hello"
# text[0] = "H"  # 🚨 TypeError 발생! (문자열은 불변이니까 수정할 수 없음)

# 값을 바꾸고 싶다면? 아예 새로운 문자열을 만들어서 덮어씌워야 한다.
text = "H" + text[1:] 
print(text) # 결과: Hello

```

#### ⭕ Mutable (가변)의 특징과 치명적인 함정: "참조(주소) 공유"

코딩 테스트에서 가장 많이 당하는 함정이다. 리스트는 가변 객체이기 때문에, 변수에 할당할 때 값을 복사하는 게 아니라 **'메모리 주소'**를 공유해 버린다.

```python
arr1 = [1, 2, 3]
arr2 = arr1       # arr1의 '주소'를 arr2에게 줌 (얕은 복사)

arr2[0] = 99      # arr2만 수정했다고 생각하겠지만...

print("arr1:", arr1) # 결과: [99, 2, 3] 
print("arr2:", arr2) # 결과: [99, 2, 3]
# 🔥 arr1까지 같이 바뀌어버리는 대참사가 발생한다!

```

> **해결책:** 값만 똑같이 복사해서 완전히 남남인 리스트를 만들려면 슬라이싱(`arr2 = arr1[:]`)을 쓰거나 `arr2 = arr1.copy()`를 사용해야 한다. (이것이 얕은 복사/깊은 복사 개념의 출발점이다.)

### 🔥 3. 코딩 테스트 실전 압축: "딕셔너리의 Key"

코딩 테스트에서 해시(Hash)를 쓸 때 아주 중요한 절대 규칙이 있다.
**"딕셔너리(`dict`)의 Key나 집합(`set`)의 원소로는 반드시 Immutable(불변) 객체만 들어갈 수 있다."**

값이 중간에 변해버리면 해시 테이블에서 데이터를 찾을 수 없게 되기 때문이다.

```python
my_dict = {}

# 1. 튜플(Immutable)은 Key로 쓸 수 있다! (좌표 저장할 때 엄청 많이 씀)
my_dict[(0, 1)] = "Start" 

# 2. 리스트(Mutable)는 Key로 쓸 수 없다!
# my_dict[[0, 1]] = "Start" # 🚨 TypeError: unhashable type: 'list'

```

그래서 2차원 배열의 특정 좌표 `[r, c]`를 딕셔너리에 저장하거나 방문 처리(`set`)할 때는, 반드시 튜플인 `(r, c)` 형태로 변환해서 넣어야 한다.

### 📂 어디에 넣으면 좋을까?

* **분류:** **[파이썬 기초 - 메모리와 객체]** 또는 **[파이썬 기초 - 자료형]**
* **사유:** 알고리즘이나 자료구조를 배우기 전, 파이썬이라는 언어가 메모리를 어떻게 다루는지 이해하기 위한 가장 밑바닥의 필수 교양이기 때문이다. 이 개념을 모르면 DFS/BFS에서 2차원 배열을 다룰 때 원인 모를 오답 지옥에 빠지게 된다.
