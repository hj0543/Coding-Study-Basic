## 1. 트리(Tree)의 개념과 특징

지금까지 배운 배열, 스택, 큐가 데이터가 한 줄로 늘어선 선형 구조(1:1 관계)였다면, 트리는 데이터가 나뭇가지처럼 뻗어나가는 **비선형 구조(1:N 관계)**다.

* **계층형 자료구조**: 상위 원소(부모)에서 하위 원소(자식)로 내려가면서 점차 확장되는 트리 모양의 구조를 가진다.
* **재귀적 정의 (가장 중요★)**: 트리 안에는 또 다른 작은 트리(서브 트리, Subtree)들이 무한히 포함되어 있다.
* 이 "재귀적 구조" 때문에, 트리를 탐색할 때는 필연적으로 **재귀 함수(DFS)**를 사용하게 된다. (부모를 처리하는 로직이 자식을 처리하는 로직과 완전히 동일하기 때문)

## 2. 반드시 알아야 할 트리의 핵심 용어 (Terminology)

알고리즘 문제의 지문은 아래 용어들을 섞어서 출제된다. 헷갈리기 쉬운 개념(차수 vs 높이)을 확실히 잡아야 한다.

### ① 가족 관계 관련 용어

* **루트 노드 (Root Node)**: 트리의 시작점이자 최상위 노드. (부모가 없는 유일한 노드)
* **단말 노드 / 리프 노드 (Leaf Node)**: 자식이 하나도 없는 노드. 트리의 맨 끝단.
* *실전 팁: 재귀 탐색을 할 때, 현재 노드가 리프 노드이면 탐색을 멈추는 **종료 조건(Base Case)**이 된다.*


* **형제 노드 (Sibling Node)**: 같은 부모를 둔 자식 노드들.
* **조상 노드 (Ancestor)**: 특정 노드에서 간선을 따라 루트 노드까지 거슬러 올라가는 경로에 있는 모든 노드들. (예: 내 부모, 조부모, 증조부모...)
* **자손 노드 (Descendant)**: 특정 노드를 루트로 하는 서브 트리에 속한 모든 하위 노드들.

### ② 숫자와 관련된 용어 (차수 vs 높이)

* **노드의 차수 (Degree)**: 특정 노드에 **직접 연결된 자식 노드의 개수**.
* **트리의 차수**: 트리 내에 있는 모든 노드의 차수 중 **가장 큰 값**. (트리가 얼마나 양옆으로 넓게 퍼지는가를 의미)
* **노드의 높이 (Height / Level)**: 루트 노드에서 현재 노드까지 도달하기 위해 거쳐야 하는 **간선(Edge)의 수**. (루트를 0으로 볼지 1로 볼지는 문제마다 다르므로 지문 확인 필수)
* **트리의 높이 / 깊이 (Depth)**: 트리 내의 노드들이 가진 높이 중 **가장 큰 값**. (트리가 위아래로 얼마나 깊은가를 의미)

## 3. 예제로 보는 트리 노드별 속성 분석

수업 노트에 등장한 트리(루트 노드 A)를 표로 분해해 보면, 각 용어의 의미가 명확해진다.

| 노드 | 차수 (Degree) | 노드 종류 | 조상 노드 (위로) | 자손 노드 (아래로) |
| --- | --- | --- | --- | --- |
| **A** | **3** | **루트 노드** | 없음 | B, C, D, E, F, G, H, I, J, K |
| **B** | 2 | 내부 노드 | A | E, F, K |
| **C** | 1 | 내부 노드 | A | G |
| **D** | **3** | 내부 노드 | A | H, I, J |
| **E** | **0** | **단말(리프) 노드** | B, A | 없음 |
| **F** | 1 | 내부 노드 | B, A | K |
| **G** | **0** | **단말(리프) 노드** | C, A | 없음 |
| **H** | **0** | **단말(리프) 노드** | D, A | 없음 |
| **I** | **0** | **단말(리프) 노드** | D, A | 없음 |
| **J** | **0** | **단말(리프) 노드** | D, A | 없음 |
| **K** | **0** | **단말(리프) 노드** | F, B, A | 없음 |

### 💡 표에서 얻을 수 있는 통찰 (Insight)

1. **단말(리프) 노드의 차수는 무조건 0이다.** (자식이 없기 때문)
2. 노드 A와 노드 D의 차수가 3으로 가장 크다. 따라서 **이 트리의 차수는 3**이다.
3. K의 조상 노드를 구하려면 위로 거슬러 올라가면 된다. ($K \rightarrow F \rightarrow B \rightarrow A$)
* *알고리즘 응용: 두 노드의 공통 조상 중 가장 가까운 것을 찾는 **LCA(최소 공통 조상)** 알고리즘이 A형 심화 문제로 자주 등장한다.*

## 4. 실전! 트리를 코드로 표현하는 방법 (미리 보기)

트리는 그래프의 일종이므로, 앞서 배운 **인접 리스트 (Adjacency List)**를 똑같이 사용하여 컴퓨터 메모리에 저장한다.

```python
# 노드의 개수가 N개일 때
N = 11
tree = [[] for _ in range(N + 1)]

# A(1)의 자식이 B(2), C(3), D(4) 라면
tree[1].append(2)
tree[1].append(3)
tree[1].append(4)

# 특정 노드의 '차수(Degree)'를 알고 싶다면?
# 단순히 그 노드의 인접 리스트 길이를 구하면 끝이다!
degree_of_A = len(tree[1]) # 3

```

✅ **마무리 점검**
비선형 자료구조인 트리의 큰 그림과 필수 용어들을 세세하게 뜯어보았다.
트리는 이 자체로 출제되기보다는, **이진 트리(Binary Tree)**라는 특수한 형태로 가공되어 **트리 순회(전위/중위/후위)**나 **힙(Heap), 이진 탐색 트리(BST)** 알고리즘으로 발전하게 된다.
