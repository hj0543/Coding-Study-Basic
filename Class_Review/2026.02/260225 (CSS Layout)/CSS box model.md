
# 📝 [TIL] CSS - display 속성 (Block vs Inline) 완벽 정리

> **"요소를 큼직한 벽돌(Block)로 쌓을 것인가, 물 흐르듯(Inline) 배치할 것인가?"**

## 1. Normal Flow (일반적인 흐름)

CSS로 특별한 위치 지정(`position: absolute` 등)을 하지 않았을 때, 웹 브라우저가 HTML 요소들을 화면에 배치하는 **기본적이고 자연스러운 배치 규칙**을 말합니다. 이 Normal Flow를 결정짓는 가장 핵심적인 속성이 바로 `display`입니다.

---

## 2. Block 타입 (독립된 덩어리)

화면의 **가로 전체(한 줄)를 혼자 독차지하는** 이기적인(?) 요소입니다.

* **특징**:
* 항상 새로운 줄(행)에서 시작합니다.
* 요소의 내용(Content)이 짧아도, 남은 가로 공간을 모두 차지합니다.


* **크기 및 여백 조절**: `width`, `height`, `margin`, `padding` 속성이 **모두 정상적으로 완벽하게 적용**됩니다.
* **대표적인 태그**: `<h1>`~`<h6>`, `<p>`, `<div>`, `<ul>`, `<li>`

### 💡 핵심 태그: `<div>` (Division)

* 그 자체로는 아무런 시각적 의미나 변화가 없는 '투명한 상자'입니다.
* 여러 HTML 요소를 하나의 그룹으로 묶어 레이아웃(헤더, 사이드바, 푸터 등)을 짜고 CSS 스타일을 한 번에 적용하기 위해 가장 많이 쓰이는 뼈대 요소입니다.

---

## 3. Inline 타입 (흐름에 맡기는 요소)

문장 안의 텍스트(단어)처럼, **내용물의 크기만큼만 공간을 차지**하며 자연스럽게 흘러가는 요소입니다.

* **특징**:
* 줄 바꿈이 일어나지 않고, 다른 인라인 요소들과 한 줄에 나란히 배치됩니다.


* **크기 및 여백 조절 (🚨 가장 많이 하는 실수!)**:
* **`width`와 `height`를 아무리 주어도 무시됩니다.** (크기 지정 불가)
* **수직 방향(상하)**: `padding`, `margin`, `border`를 줄 수는 있지만, 다른 요소를 밀어내지 못하고 텍스트 위아래로 겹쳐버립니다. (레이아웃에 영향을 주지 않음)
* **수평 방향(좌우)**: `padding`, `margin`, `border`가 정상적으로 적용되어 다른 요소를 양옆으로 밀어냅니다.


* **대표적인 태그**: `<a>`, `<img>`, `<span>`, `<strong>`

### 💡 핵심 태그: `<span>`

* `<div>`의 인라인 버전입니다. 자체적인 시각적 변화는 없습니다.
* 블록 요소처럼 줄 바꿈을 일으키지 않으므로, 긴 문장 속에서 **특정 단어나 구문에만 색상, 폰트 등의 스타일을 콕 집어 적용할 때** 매우 유용합니다.

---

## 4. Inline-block (하이브리드 타입)

"Block처럼 크기도 내 마음대로 조절하고 싶은데, Inline처럼 한 줄에 나란히 배치하고 싶어!" 할 때 사용하는 만능 속성입니다.

```css
.box {
  display: inline-block;
}

```

* **Inline의 장점**: 요소들이 줄 바꿈 없이 한 줄에 나란히 배치됩니다.
* **Block의 장점**: `width`, `height`, 상하좌우 `margin`과 `padding`이 **모두 정상적으로 팍팍 적용되어 다른 요소를 밀어냅니다.**
* **활용**: 내비게이션 바의 메뉴 버튼들이나, 바둑판식으로 나열되는 카드 UI를 만들 때 자주 사용됩니다.

---

## 5. None (숨김 처리)

요소를 화면에서 완전히 지워버립니다.

```css
.hidden-box {
  display: none;
}

```

* **특징**: 단순히 눈에 안 보이는 것을 넘어, **화면에서 차지하던 공간조차 완전히 사라집니다.**
* **비교**: 공간은 그대로 남겨둔 채 투명하게만 만들고 싶다면 `visibility: hidden;`이나 `opacity: 0;`을 사용해야 합니다.

---

### 📊 display 속성 한눈에 요약

| 속성 | 줄 바꿈 | `width` / `height` 적용 | 상하 `margin` / `padding` (요소 밀어내기) |
| --- | --- | --- | --- |
| **block** | O (항상 새 줄) | O | O |
| **inline** | X (한 줄에 나란히) | **X (불가)** | **X (적용은 되나 밀어내지 못함)** |
| **inline-block** | X (한 줄에 나란히) | O | O |
| **none** | - | - | 화면에서 공간 자체를 삭제 |

> **오늘의 교훈**: "a 태그나 span 태그의 크기가 안 변한다면? `display: block`이나 `inline-block`으로 바꿔주자!"
