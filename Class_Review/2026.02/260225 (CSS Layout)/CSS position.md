
# 📝 [TIL] CSS - Position & z-index 완벽 정리

> **"요소를 화면의 원하는 곳에 찰떡같이 띄우고, 쌓고, 고정하는 마법의 속성"**

## 1. CSS Layout vs Position의 차이

CSS에서 요소를 배치하는 방식은 크게 두 가지 관점으로 나뉩니다.

* **CSS Layout (`display` 속성 중심)**
* 블록(상하), 인라인(좌우), Flex, Grid 등을 이용해 전체적인 페이지의 **뼈대와 흐름(Normal Flow)**을 구성합니다.


* **CSS Position (`position` 속성 중심)**
* 요소를 정상적인 흐름(Normal Flow)에서 떼어내어 **다른 위치로 강제 이동**시키거나, 공중에 띄워 다른 요소 위에 겹치게 만드는 등 특별한 배치를 담당합니다.



---

## 2. Position 핵심 유형 5가지 ★★★

네 가지 방향 속성(`top`, `bottom`, `left`, `right`)을 이용해 요소를 이동시킵니다. 각 속성이 **"무엇을 기준으로 이동하는가?"**를 이해하는 것이 핵심입니다.

### ① static (기본값)

* **특징**: HTML 문서의 기본 흐름을 얌전히 따릅니다.
* **포인트**: 방향 속성(`top`, `left` 등)과 `z-index`를 주어도 **완벽하게 무시**됩니다. 그냥 "작성된 순서대로 배치한다"는 의미입니다.

### ② relative (상대 위치)

* **특징**: **"자기 자신이 원래 있어야 할 자리"**를 기준으로 이동합니다.
* **포인트**: 요소를 화면에서 이동시켜도, 원래 있던 공간(빈자리)은 유령처럼 그대로 남아 있어 다른 요소의 레이아웃을 망치지 않습니다.
* **주요 용도**: 자기 자신을 이동시키기보단, **`absolute` 자식의 '기준점' 역할을 하기 위해** 부모 요소에게 부여하는 경우가 99%입니다.

### ③ absolute (절대 위치)

* **특징**: 문서의 일반적인 흐름에서 완전히 빠져나와 **공중 부양**합니다. (기존 자리가 사라짐)
* **기준점**: `static`이 아닌(즉, `relative`, `absolute`, `fixed` 중 하나인) **가장 가까운 조상 요소**를 기준으로 위치가 결정됩니다.
* **포인트**: 만약 조상 중에 기준이 될 요소가 하나도 없다면, 브라우저 화면 전체(`<html>`, 뷰포트)를 기준으로 삼아버리니 주의해야 합니다.

```css
.parent { 
  position: relative; /* 자식의 기준점이 됨 (울타리 역할) */
}
.child {
  position: absolute;
  right: 0;
  bottom: 0; /* 부모 영역의 오른쪽 하단에 완벽하게 밀착! */
}

```

### ④ fixed (고정 위치)

* **특징**: 브라우저의 **보이는 화면(Viewport)**을 기준으로 위치가 완전히 고정됩니다.
* **포인트**: 스크롤을 아무리 내려도 화면의 그 자리에 못 박힌 듯 가만히 있습니다. (예: 상단 네비게이션 바, '맨 위로 이동' 화살표 버튼)

### ⑤ sticky (끈적한 위치)

* **특징**: 평소에는 `static`이나 `relative`처럼 흐름을 따르다가, 스크롤을 내려 **임계점(지정한 `top` 위치 등)**에 도달하는 순간 `fixed`처럼 화면에 찰싹 달라붙습니다.
* **포인트**: 무한정 따라오는 것이 아니라, **부모 요소의 영역 안에서만 작동**합니다. 부모 영역이 끝나면 같이 화면 밖으로 밀려 올라갑니다. (예: 표의 헤더, 사이드바 목차)

## 3. Position 속성 한눈에 요약표

| 속성 | 기준점 | 기존 공간 차지 여부 | 주요 특징 |
| --- | --- | --- | --- |
| **static** | 문서 흐름 (기본) | 차지함 | `top`, `left`, `z-index` 안 먹힘 |
| **relative** | 본인 원래 위치 | **차지함** (이동해도 빈자리 남음) | `absolute`의 기준점(부모) 역할 |
| **absolute** | 위치 지정된 부모 | 차지 안 함 (공중 부양) | 부모에게 `relative` 주기 필수! |
| **fixed** | 뷰포트 (브라우저 화면) | 차지 안 함 (공중 부양) | 스크롤을 무시하고 화면에 고정 |
| **sticky** | 부모 요소 내부 | 차지함 | 스크롤 임계점부터 화면에 고정 |

## 4. z-index (쌓임 순서)

화면은 2D지만, `position`을 사용해 요소를 겹치게 만들면 화면 앞으로 튀어나오는 **Z축(깊이)**의 개념이 생깁니다. 이를 제어하는 속성입니다.

### 🔹 z-index의 절대 규칙

1. **적용 조건**: `position` 값이 `static`이 아닌 요소(`relative`, `absolute`, `fixed`, `sticky`)에만 적용됩니다.
2. **순서 결정**: 정수 값을 사용하며, **값이 클수록 화면 앞쪽(사용자와 가까운 쪽)에 겹쳐서 쌓입니다.** 값이 같으면 HTML 문서의 작성 순서상 나중에 작성된 것이 위에 올라옵니다.
3. **부모 종속성 (Stacking Context, 가장 중요★)**:
* `z-index`는 그룹별(부모별)로 경쟁합니다.
* 부모 A의 `z-index`가 `1`이고, 부모 B의 `z-index`가 `2`라면?
* **부모 A의 자식이 `z-index: 9999;`를 가져도, 부모 B(`z-index: 2`)를 절대 이기고 위로 올라갈 수 없습니다.** (부모의 서열이 자식의 서열을 결정함)
