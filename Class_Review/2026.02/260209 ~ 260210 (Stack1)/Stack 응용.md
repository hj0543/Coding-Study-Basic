# 📝 [TIL] 알고리즘 - 스택(Stack)의 응용

> **"스택은 상태를 기억하고 되돌아가는(Backtracking) 모든 로직의 근간이다."**

## 1. 괄호 검사 (Parenthesis Matching)

코딩 테스트나 IM 등급에서 스택의 개념을 묻기 위해 가장 자주 출제되는 기초 응용이다.

* **조건**:
1. 왼쪽 괄호의 개수와 오른쪽 괄호의 개수가 같아야 한다.
2. 같은 괄호에서 왼쪽 괄호는 오른쪽 괄호보다 먼저 나와야 한다.
3. 괄호 사이에는 포함 관계만 존재한다. (교차 금지)


* **검사 로직**:
문자열을 순회하면서 다음 규칙을 따른다.
1. **여는 괄호** `(`, `{`, `[` 를 만나면 스택에 **Push** 한다.
2. **닫는 괄호** `)`, `}`, `]` 를 만나면 스택에서 **Pop** 하여 짝이 맞는지 비교한다.


* **실패(오류) 조건 3가지**:
1. 닫는 괄호를 만났는데 스택이 비어있는 경우. (여는 괄호 부족)
2. Pop한 여는 괄호와 현재 닫는 괄호의 종류가 다른 경우. (짝 불일치)
3. 문자열을 끝까지 읽었는데 스택에 괄호가 남아있는 경우. (닫는 괄호 부족)



```python
# 괄호 검사 핵심 코드 패턴
def check_matching(text):
    stack = []
    pairs = {')': '(', '}': '{', ']': '['}
    
    for char in text:
        if char in '({[': # 여는 괄호
            stack.append(char)
        elif char in ')}]': # 닫는 괄호
            if not stack:
                return False # 에러 1: 스택이 비어있음
            top = stack.pop()
            if pairs[char] != top:
                return False # 에러 2: 짝이 맞지 않음
                
    if stack:
        return False # 에러 3: 여는 괄호가 남음
    return True # 완벽한 수식

```
