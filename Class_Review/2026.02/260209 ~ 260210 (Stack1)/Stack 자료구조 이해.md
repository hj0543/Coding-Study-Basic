
# 📝 [TIL] 알고리즘 - 스택(Stack) 자료구조 이해

> **"가장 나중에 들어간 것이 가장 먼저 나온다. (후입선출)"**

## 1. 스택(Stack)의 기본 개념

* **정의**: 물건을 쌓아 올리듯 자료를 쌓아 올린 형태의 선형 자료구조이다.
* **핵심 특징 (LIFO)**: **Last-In-First-Out (후입선출)**. 즉, 마지막에 삽입된 자료가 가장 먼저 삭제된다.
* **비유**: 프링글스 감자칩 통, 식당의 쟁반 쌓기, 브라우저의 '뒤로 가기' 버튼.

---

## 2. 스택의 주요 연산 (Operations)

스택을 제어하기 위해서는 다음 4가지 연산이 필수적이다.

1. **`push(item)`**: 스택의 맨 위(Top)에 새로운 자료를 **삽입**한다.
2. **`pop()`**: 스택의 맨 위(Top)에 있는 자료를 **삭제하고 반환**한다. (꺼내기)
3. **`isEmpty()`**: 스택이 비어있는지 **확인**한다. (비어있으면 True, 아니면 False)
4. **`peek()` / `top()**`: 스택의 맨 위(Top)에 있는 자료를 **반환만** 한다. (삭제하지 않고 확인만 함)

---

## 3. 파이썬에서의 스택 구현

파이썬은 내장 **리스트(List)**가 스택의 역할을 완벽하게 수행하므로 별도의 라이브러리가 필요 없다. 하지만 알고리즘적 사고를 위해 **Top 포인터를 이용한 정석적인 구현**도 알아두어야 한다.

### ① 파이썬 리스트를 활용한 구현 (실전용)

가장 많이 쓰이고 파이썬다운(Pythonic) 방법이다.

```python
stack = []

# 1. Push (삽입)
stack.append(1)
stack.append(2)
stack.append(3) # stack: [1, 2, 3]

# 2. Pop (삭제 및 반환)
top_item = stack.pop() # 3 반환, stack: [1, 2]

# 3. Peek (Top 확인)
# 인덱스 에러 방지를 위해 스택이 비어있지 않은지 먼저 확인하는 것이 좋다.
if stack: 
    print(stack[-1]) # 2 출력 (삭제 안 됨)

# 4. isEmpty (공백 확인)
if not stack:
    print("스택이 비어있습니다.")

```

### ② 배열과 Top 변수를 이용한 구현 (이론 파악용)

C나 Java처럼 크기가 고정된 배열을 사용할 때의 구현 방식이다. (동작 원리 이해에 필수적이다)

```python
SIZE = 10
stack = [0] * SIZE
top = -1 # 스택이 비어있음을 의미하는 초기값

def push(item):
    global top
    if top == SIZE - 1:
        print("Stack Overflow!") # 스택이 꽉 참
        return
    top += 1
    stack[top] = item

def pop():
    global top
    if top == -1:
        print("Stack Underflow!") # 스택이 빔
        return None
    
    item = stack[top]
    top -= 1
    return item

```

## 4. 스택의 주요 응용 분야

스택은 단순히 데이터를 넣고 빼는 것을 넘어, 컴퓨터 시스템 전반에서 핵심적인 역할을 한다.

### ① 함수 호출 관리 (Function Call Stack)

* 프로그램에서 함수가 호출될 때마다 메모리의 **스택 영역**에 함수의 정보(지역 변수, 복귀 주소 등)가 Push된다.
* 함수 실행이 끝나면 Pop되어 **이전 함수(호출했던 곳)로 돌아간다.**
* **재귀 함수(Recursion)**가 깊어질수록 스택에 계속 데이터가 쌓이므로, 종료 조건이 없으면 `Stack Overflow` 에러가 발생한다.

### ② 괄호 검사 (Parenthesis Matching)

코딩 테스트 단골 문제다. 여는 괄호와 닫는 괄호의 짝이 맞는지 검사한다.

* **로직**:
1. 수식을 읽으면서 **여는 괄호(`(`, `{`, `[`)**를 만나면 스택에 **Push** 한다.
2. **닫는 괄호(`)`, `}`, `]`)**를 만나면 스택에서 **Pop** 하여 짝이 맞는지 확인한다.
3. 수식을 다 읽었는데 스택에 괄호가 남아있거나, Pop할 괄호가 없으면 에러(짝이 안 맞음)이다.



### ③ 실행 취소 (Undo / Redo)

* 워드 프로세서의 `Ctrl + Z` 기능.
* 사용자가 한 행동을 순서대로 스택에 Push해두고, 취소를 누르면 Pop하여 가장 최근 행동부터 되돌린다.

---

> **💡 오늘의 결론**
> 스택은 **"가장 최근의 상태를 기억하고, 필요할 때 되돌아가는"** 용도로 사용된다.
> 이 특성 때문에 **DFS(깊이 우선 탐색)**에서 갈림길을 만났을 때, 현재 위치를 스택에 저장(Push)해두고 끝까지 갔다가 막히면 다시 꺼내어(Pop) 돌아오는 백트래킹 로직의 핵심 기반이 된다.
