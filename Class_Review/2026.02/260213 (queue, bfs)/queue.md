> **"먼저 들어온 데이터가 먼저 나가는 줄 서기. (선입선출)"**

## 1. 큐(Queue)의 기본 개념

* **핵심 특징 (FIFO)**: **First-In-First-Out (선입선출)**. 가장 먼저 삽입된 원소가 가장 먼저 삭제된다.
* **주요 포인터**:
* **`front`**: 데이터가 꺼내지는(삭제되는) 쪽의 인덱스 (마지막으로 삭제된 위치).
* **`rear`**: 데이터가 들어가는(삽입되는) 쪽의 인덱스 (마지막으로 저장된 위치).



### 🔹 큐의 핵심 연산

* `enqueue(item)`: 큐의 뒤쪽(`rear` 다음)에 원소를 삽입.
* `dequeue()`: 큐의 앞쪽(`front`)에서 원소를 삭제하고 반환.
* `is_empty()` / `is_full()`: 공백 상태 / 포화 상태 확인.
* `qpeek()`: 삭제하지 않고 맨 앞(`front + 1`)의 원소 확인만 하기.

## 2. 선형 큐 (Linear Queue)

1차원 배열을 사용하여 데이터를 일렬로 저장하는 가장 기본적인 큐.

* **초기 상태**: `front = rear = -1`
* **공백 상태**: `front == rear`
* **삽입 (`enqueue`)**: `rear`를 1 증가시키고 그 자리에 저장.
* **삭제 (`dequeue`)**: `front`를 1 증가시키고 그 자리의 값을 반환.

### 🚨 선형 큐의 치명적 문제점: "잘못된 포화 상태 인식"

* 삽입과 삭제를 반복하다 보면, `front`와 `rear`가 계속 오른쪽으로만 이동한다.
* 결국 배열의 앞부분이 텅 비어있음에도 불구하고, `rear == n-1` (마지막 인덱스)에 도달하면 **큐가 꽉 찼다고 잘못 인식(False Full)**하여 더 이상 삽입을 하지 못한다.
* 매번 원소를 앞으로 당기는 방법은 원소 이동 시간이 너무 오래 걸려 비효율적이다.

## 3. 원형 큐 (Circular Queue)

선형 큐의 공간 낭비 문제를 해결하기 위해, **배열의 처음과 끝이 논리적으로 연결되어 있다고 가정**하는 큐.

* **초기 상태**: `front = rear = 0`
* **핵심 원리**: 나머지 연산자 (`%`, `mod`)
* 인덱스가 배열의 끝(`n-1`)에 도달하면 다시 처음(`0`)으로 돌아오게 만든다.


* **상태 구분을 위한 약속**: 공백과 포화 상태를 쉽게 구분하기 위해 **`front`가 가리키는 자리는 항상 빈자리로 둔다.**

### 🔹 선형 큐 vs 원형 큐 연산 비교

| 구분 | 삽입 위치 (`rear`) | 삭제 위치 (`front`) | 포화 상태 검사 (`is_full`) |
| --- | --- | --- | --- |
| **선형 큐** | `rear += 1` | `front += 1` | `rear == n-1` |
| **원형 큐** | `rear = (rear + 1) % n` | `front = (front + 1) % n` | `(rear + 1) % n == front` |

## 4. 파이썬 실전 큐 구현: 덱 (deque) ★★★

실제 코딩테스트(특히 A형의 BFS)에서는 배열로 원형 큐를 직접 구현하지 않는다. 파이썬의 내장 라이브러리인 **`collections.deque`**를 사용하는 것이 정석이다.

### 🚨 주의: 리스트의 `pop(0)`을 쓰면 안 되는 이유

```python
q = [1, 2, 3]
print(q.pop(0)) # 1 출력

```

파이썬 리스트에서 `pop(0)`을 하면, 맨 앞의 요소를 뺀 뒤 **나머지 모든 요소를 한 칸씩 앞으로 당겨야 하므로 $O(N)$의 시간**이 걸린다. 데이터가 10만 개면 시간 초과가 발생한다.

### 💡 정답: `deque` (Double-Ended Queue)

양쪽 끝에서 삽입과 삭제가 모두 $O(1)$(상수 시간)에 이루어지는 매우 빠른 자료구조다. (내부적으로 연결 리스트(Linked Queue)로 구현되어 있음)

```python
from collections import deque

# 1. 큐 생성
q = deque()

# 2. 삽입 (enqueue)
q.append(1)
q.append(2)
q.append(3) # q: deque([1, 2, 3])

# 3. 삭제 (dequeue)
# 반드시 popleft()를 써야 왼쪽(front)에서 빠진다!
first_item = q.popleft() # 1 반환, q: deque([2, 3])

# 4. 공백 검사
if not q:
    print("큐가 비어있습니다.")

```

## 5. 우선순위 큐 (Priority Queue)

들어온 순서와 상관없이, **우선순위가 가장 높은 데이터가 먼저 나가는** 큐.

* **문제점**: 일반적인 1차원 배열로 구현하면 삽입/삭제 시 원소를 재배치해야 하므로 $O(N)$의 시간이 걸려 비효율적이다.
* **해결책**: 트리 구조인 **힙(Heap)**을 사용하여 $O(\log N)$만에 삽입/삭제를 처리한다. (트리 단원에서 자세히 다룰 예정)

✅ **마무리 점검**
스택(DFS)과 큐(BFS)의 자료구조적 기초가 모두 완성되었다.
특히 **`from collections import deque`와 `popleft()`**는 A형 BFS 문제를 풀 때 숨 쉬듯 써야 하는 코드다.
