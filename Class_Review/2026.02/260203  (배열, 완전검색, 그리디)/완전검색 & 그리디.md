# 📝 완전 검색과 그리디 (Exhaustive Search & Greedy)

문제 해결의 가장 기본이 되는 '모든 경우 다 해보기'와 '당장 좋은 것 선택하기' 기법이다.

## 1. 완전 검색 (Exhaustive Search)

문제의 해법으로 생각할 수 있는 **모든 경우의 수를 나열**해보고 확인하는 기법이다.

* **Brute-force** 또는 **Generate-and-Test**라고도 불린다.
* **특징**: 모든 경우를 다 검사하므로 해를 찾을 확률은 100%다.
* **단점**: 경우의 수가 많으면 시간이 너무 오래 걸린다.
* **활용**: 입력 데이터의 크기(N)가 작을 때 유용하다. 알고리즘 설계 시 가장 먼저 시도해봐야 하는 접근법이다.

---

## 2. 순열 (Permutation)

완전 검색을 하기 위해 **순서를 고려하여 나열**하는 방법이다.

* **정의**: 서로 다른 `n`개 중 `r`개를 택하여 일렬로 나열하는 것 ().
* **팩토리얼 ()**: `n`개의 요소를 모두 나열하는 경우의 수. 이 10을 넘어가면 시간 복잡도가 폭발적으로 증가하므로 완전 검색으로 풀기 어렵다.

### 순열 생성 예시 (반복문)

`{1, 2, 3}`을 포함하는 모든 순열을 생성하는 단순한 방법이다.

```python
for i1 in range(1, 4):
    for i2 in range(1, 4):
        if i2 != i1:
            for i3 in range(1, 4):
                if i3 != i1 and i3 != i2:
                    print(i1, i2, i3)

```

## 3. 탐욕 알고리즘 (Greedy Algorithm)

최적해를 구하기 위해 사용되는 근시안적인 방법이다.

* **핵심**: 여러 경우 중 하나를 결정해야 할 때, **그 순간에 최적이라고 생각되는 것**을 선택해 나가는 방식이다.
* **한계**: 각 단계의 최적 선택이 전체의 최적해라는 보장은 없다. (검증이 필요함)

### 동작 과정

1. **해 선택**: 현재 상태에서 부분 문제의 최적해를 구한 뒤, 부분해 집합(Solution Set)에 추가한다.
2. **실행 가능성 검사**: 새로운 부분해 집합이 문제의 제약 조건을 위반하지 않는지 검사한다.
3. **해 검사**: 새로운 부분해 집합이 전체 문제의 해가 되는지 확인한다.

## 4. 실전 예제: Baby-gin Game

0~9 사이의 숫자 카드 6장을 뽑았을 때, **run**(연속된 3숫자)과 **triplet**(동일한 3숫자)으로만 구성되어 있는지 확인하는 게임이다.

### ① 완전 검색 접근법

1. 6장의 카드로 만들 수 있는 **모든 순열**을 생성한다.
2. 앞의 3자리와 뒤의 3자리가 각각 `run`이나 `triplet`인지 검사한다.
3. 하나라도 만족하는 경우가 있으면 Baby-gin이다.

### ② 그리디 접근법 (Counts 배열 활용)

순열을 만들지 않고, **숫자의 개수(빈도수)**만 세어서 즉시 판단한다.

1. `counts` 배열에 각 숫자(0~9)의 개수를 저장한다.
2. **Triplet 확인**: `counts[i] >= 3`이면 데이터를 3개 삭제하고 `triplet` 조사 성공.
3. **Run 확인**: `counts[i], counts[i+1], counts[i+2] >= 1`이면 각각 1개씩 삭제하고 `run` 조사 성공.

```python
num = 444345 # 예시 입력
c = [0] * 12 # 인덱스 에러 방지를 위해 여유 공간 확보 (0~9 + 여유 2칸)

# 1. 개수 카운팅
for i in range(6):
    c[num % 10] += 1
    num //= 10

i = 0
tri = run = 0

# 2. 그리디 탐색 (0번부터 9번까지 순회)
while i < 10:
    # Triplet 검사 (우선순위는 상관없음)
    if c[i] >= 3:
        c[i] -= 3
        tri += 1
        continue # 카드가 더 남았을 수 있으므로 i 증가 없이 다시 검사
    
    # Run 검사
    if c[i] >= 1 and c[i+1] >= 1 and c[i+2] >= 1:
        c[i] -= 1
        c[i+1] -= 1
        c[i+2] -= 1
        run += 1
        continue
        
    i += 1

if run + tri == 2:
    print("Baby Gin")
else:
    print("Lose")

```
