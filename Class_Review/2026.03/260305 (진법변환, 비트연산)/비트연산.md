# 비트연산
* 비트 연산은 컴퓨터의 가장 깊은 곳(Low-level)에서 2진수의 각 자리(비트, Bit)를 직접 조작하는 연산이다. 
* 일반적인 덧셈이나 곱셈보다 컴퓨터가 훨씬 빠르게 처리할 수 있어서, 알고리즘의 속도를 극강으로 끌어올리거나 메모리를 효율적으로 쓸 때(비트 마스킹) 자주 사용된다.

---

## 1. 비트 연산자 한눈에 보기

| 연산자 | 이름 | 설명 | 파이썬 기호 |
| --- | --- | --- | --- |
| **AND** | 논리 곱 | 두 비트가 **모두 1일 때만** 1 | `&` |
| **OR** | 논리 합 | 두 비트 중 **하나라도 1이면** 1 | `|` |
| **XOR** | 배타적 논리 합 | 두 비트가 **서로 다를 때만** 1 | `^` |
| **NOT** | 논리 부정 | 0은 1로, 1은 0으로 **뒤집기** | `~` |
| **Left Shift** | 왼쪽 시프트 | 비트를 왼쪽으로 밀기 (곱하기 $2^n$) | `<<` |
| **Right Shift** | 오른쪽 시프트 | 비트를 오른쪽으로 밀기 (나누기 $2^n$) | `>>` |

---

## 2. 기본 비트 연산 (AND, OR, XOR)

계산을 눈으로 확인하기 위해, 앞서 배운 `bin()` 함수를 활용하여 10진수 10과 12를 2진수로 바꿔서 연산해본다.

* `10` = `1010` (2진수)
* `12` = `1100` (2진수)

```python
a = 10  # 1010
b = 12  # 1100

# 1. AND 연산 (&) : 둘 다 1이어야 1
#   1010
# & 1100
# ------
#   1000  (10진수로 8)
print(a & b)  # 출력: 8

# 2. OR 연산 (|) : 하나라도 1이면 1
#   1010
# | 1100
# ------
#   1110  (10진수로 14)
print(a | b)  # 출력: 14

# 3. XOR 연산 (^) : 서로 달라야 1 (같으면 0)
#   1010
# ^ 1100
# ------
#   0110  (10진수로 6)
print(a ^ b)  # 출력: 6

```

---

## 3. 비트 반전 (NOT 연산)

NOT 연산(`~`)은 0을 1로, 1을 0으로 모조리 뒤집는다. 그런데 파이썬에서 양수에 `~`를 씌우면 음수가 나온다. (컴퓨터가 음수를 표현하는 방식인 '2의 보수' 때문이다.)

* **공식처럼 외워두면 편하다:** `~x = -(x + 1)`

```python
a = 10
# ~10 = -(10 + 1) = -11
print(~a)  # 출력: -11

```

---

## 4. 시프트(Shift) 연산 : 비트 밀어내기

시프트 연산은 2진수의 모든 비트를 지정한 칸 수만큼 왼쪽(`<<`)이나 오른쪽(`>>`)으로 밀어버린다. 이는 **2를 곱하거나 나누는 것과 완벽히 똑같은 효과**를 낸다.

* **Left Shift (`<< n`)**: 왼쪽으로 밀고 빈자리는 0으로 채운다. $\rightarrow$ **값에 $2^n$ 을 곱한 결과**가 된다.
* **Right Shift (`>> n`)**: 오른쪽으로 밉니다. 밀려난 비트는 버려진다. $\rightarrow$ **값을 $2^n$ 으로 나눈 몫**이 된다.

```python
a = 10  # 2진수: 1010

# 왼쪽으로 1칸 밀기 (10 * 2^1)
# 1010 -> 10100 (20)
print(a << 1)  # 출력: 20

# 왼쪽으로 2칸 밀기 (10 * 2^2)
# 1010 -> 101000 (40)
print(a << 2)  # 출력: 40

# 오른쪽으로 1칸 밀기 (10 // 2^1)
# 1010 -> 0101 (5)
print(a >> 1)  # 출력: 5

```

## 💡 실무/알고리즘 꿀팁: 비트 연산은 언제 쓸까?

단순히 계산을 넘어, 코딩 테스트나 실무에서는 이런 마법 같은 일에 쓰인다.

1. **홀수/짝수 판별 (가장 빠른 방법)**: `x & 1`
* `10 & 1 == 0` (짝수), `11 & 1 == 1` (홀수). `% 2` 보다 컴퓨터 입장에서 훨씬 빠른 연산이다.


2. **XOR의 마법 (짝 없는 숫자 찾기)**: `x ^ x == 0`
* 같은 숫자를 두 번 XOR하면 0이 된다. 배열에서 `[2, 3, 2, 4, 3]` 처럼 다 두 번씩 나오는데 하나만 한 번 나온다면, 모든 요소를 쭈욱 `^` 연산해보자. 짝이 맞는 애들은 다 0이 되어 사라지고, 정답인 `4`만 남는다.


3. **상태값 저장 (비트 마스킹)**: 여러 개의 True/False(불리언) 값을 변수 하나에 2진수 자리수별로 저장해서 메모리를 극단적으로 아낄 때 사용한다.

## 1. 컴퓨터의 음수 표현 방법과 '2의 보수'

컴퓨터는 0과 1밖에 모르기 때문에 사람처럼 숫자 앞에 마이너스 기호(`-`)를 붙일 수 없다. 그래서 **가장 왼쪽의 비트(MSB: Most Significant Bit)를 '부호'를 나타내는 데 사용**하기로 약속했다.

* `0` 이면 양수 (+)
* `1` 이면 음수 (-)

### 왜 '2의 보수'를 사용할까?

초기에는 그냥 부호 비트만 `1`로 바꾸는 방식(부호-크기 방식)이나, 0과 1을 모두 뒤집기만 하는 방식(1의 보수)을 썼다. 하지만 이 방식들은 치명적인 문제가 있었다.

1. **`+0`과 `-0`이 따로 존재하게 됨** (메모리 낭비 및 혼란)
2. **양수와 음수의 덧셈 연산이 한 번에 안 됨** (회로가 복잡해짐)

이 문제를 완벽하게 해결한 천재적인 방법이 바로 **'2의 보수(Two's Complement)'**이다. 2의 보수를 사용하면 컴퓨터는 뺄셈 회로를 따로 만들 필요 없이, 그냥 덧셈 회로만으로 뺄셈을 처리할 수 있게 된다.

### 🎯 2의 보수 구하는 방법 (공식: 다 뒤집고 + 1)

어떤 양수를 음수(2의 보수)로 만드는 과정은 딱 두 단계이다.

1. **1의 보수 만들기**: 모든 비트의 0과 1을 서로 뒤집는다. (파이썬의 `~` 연산)
2. **1 더하기**: 위 결과의 가장 마지막 자리(LSB)에 1을 더해준다.

**[예시: 8비트 환경에서 5를 -5로 만들기]**

* **+5**: `0000 0101`
* **다 뒤집기**: `1111 1010` (1의 보수)
* **1 더하기**: `1111 1011` (완성! 이것이 컴퓨터가 인식하는 **-5**)

*(검증: `0000 0101`(+5)과 `1111 1011`(-5)을 더해보면 `1 0000 0000`이 되는데, 8비트 환경이므로 맨 앞의 1은 버려지고 완벽하게 `0000 0000`(0)이 된다!)*

## 2. SWEA 10726 (이진수 표현) 풀이

방금 배운 비트 연산의 지식을 폭발시킬 수 있는 아주 좋은 문제다.

**문제 요약:**

> 정수 $N$과 $M$이 주어질 때, $M$을 2진수로 표현했을 때 **마지막 $N$개의 비트가 모두 `1`로 켜져 있는지** 판별하여 'ON' 또는 'OFF'를 출력하라.

### 💡 풀이 접근법 2가지

이 문제는 앞서 우리가 배운 '정석 알고리즘(나머지 활용)'과 '비트 연산(마스킹)' 두 가지로 모두 풀 수 있다.

#### 방법 1: 나눗셈과 나머지 활용 (직관적)

$M$을 2로 나누면서 나오는 나머지가 계속 1인지 $N$번 확인한다. 중간에 한 번이라도 나머지가 0이 나오면 (비트가 0이라는 뜻이므로) 'OFF'다.

#### 방법 2: 비트 마스킹 활용 (빠른 방법) 🔥

$N$개의 비트가 모두 1인 '마스크(Mask)'를 하나 만든다.
예를 들어 $N=4$ 라면, 마지막 4자리가 `1111`인 숫자를 만드는 것이다.

* 파이썬의 시프트 연산을 활용하면 `(1 << N) - 1` 이 바로 $N$개의 비트가 1인 숫자가 된다. ($2^4 - 1 = 15 \rightarrow 1111_{(2)}$)
* 그리고 입력받은 $M$과 이 마스크를 **AND 연산(`&`)** 했을 때, 그 결과가 마스크와 똑같이 나온다면 마지막 $N$자리가 모두 1이었다는 뜻이 된다!

---

### 💻 파이썬 정답 코드 (비트 마스킹 방식)

```python
# SWEA 10726: 이진수 표현

T = int(input()) # 테스트 케이스 개수

for tc in range(1, T + 1):
    N, M = map(int, input().split())
    
    # 1. N개의 비트가 모두 1인 마스크(mask) 만들기
    # 예: N=3 이면 (1 << 3) - 1 -> 8 - 1 = 7 (이진수로 111)
    mask = (1 << N) - 1
    
    # 2. M과 마스크를 AND(&) 연산하여 마지막 N자리를 추출
    # 그 추출한 값이 마스크(모두 1인 상태)와 완전히 같다면 ON!
    if (M & mask) == mask:
        result = "ON"
    else:
        result = "OFF"
        
    # 결과 출력
    print(f"#{tc} {result}")

```

### 코드 핵심 포인트 (왜 마스크를 쓸까?)

만약 $N=4$, $M=47$ (`101111`) 이라면:

* `mask` = $(1 \ll 4) - 1 = 16 - 1 = 15$ (`001111`)
* `M & mask` = `101111` & `001111` = `001111` (마스크 값과 동일) $\rightarrow$ **ON**

만약 $M=30$ (`011110`) 이라면:

* `M & mask` = `011110` & `001111` = `001110` (마지막 자리가 0이라서 마스크 값과 다름) $\rightarrow$ **OFF**

이렇게 비트 연산을 쓰면 `for`문이나 `while`문을 돌면서 하나하나 검사할 필요 없이, 단 한 줄의 `&` 연산으로 끝내버릴 수 있다. 이것이 알고리즘에서 비트 마스킹이 강력한 이유다.
