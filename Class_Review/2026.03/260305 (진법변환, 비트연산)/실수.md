## 1. 실수의 깔끔한 출력과 반올림 (f-string 문법)

실무나 코딩 테스트에서 소수점이 길게 늘어지는 실수를 화면에 출력할 때, 원하는 자리수까지만 반올림해서 보여줘야 하는 경우가 많다. 파이썬의 **f-string**을 사용하면 아주 직관적이고 강력하게 소수점을 제어할 수 있다.

* **문법**: `f"{변수명:.Nf}"` (N은 남기고 싶은 소수점 아래 자리수)
* 이 방식은 단순히 문자열을 자르는 것이 아니라, **지정한 자리수 아래에서 자동으로 반올림**을 수행한 뒤 문자열로 만들어 준다.

```python
pi = 3.14159265

# 1. 기본 실수 출력
print(f"기본 출력: {pi}")         # 출력: 기본 출력: 3.14159265

# 2. 소수점 아래 2자리까지만 출력 (셋째 자리에서 반올림)
print(f"소수점 2자리: {pi:.2f}")   # 출력: 소수점 2자리: 3.14

# 3. 소수점 아래 4자리까지만 출력
print(f"소수점 4자리: {pi:.4f}")   # 출력: 소수점 4자리: 3.1416

# 4. 정수처럼 보이게 소수점 없애기 (첫째 자리에서 반올림)
print(f"소수점 0자리: {pi:.0f}")   # 출력: 소수점 0자리: 3

```

## 2. 소수점이 있는 10진수 ↔️ 2진수 변환 원리

정수의 진법 변환은 '나눗셈'을 썼지만, **소수점 아래의 진법 변환은 '곱셈'**을 사용한다.

### 🎯 10진수 소수 ➡️ 2진수 소수 (계속 곱하기)

소수 부분에 2를 곱해서 나온 결과의 **정수 부분**을 차례대로 취한다. 소수 부분이 0이 될 때까지 반복한다.

**[예시: 10진수 0.625를 2진수로]**

1. $0.625 \times 2 = \mathbf{1}.25$  (정수부 **1** 추출, 소수부 0.25 남음)
2. $0.25 \times 2 = \mathbf{0}.5$    (정수부 **0** 추출, 소수부 0.5 남음)
3. $0.5 \times 2 = \mathbf{1}.0$     (정수부 **1** 추출, 소수부 0.0 ➡️ 종료!)

* **결과**: 추출한 정수를 위에서부터 순서대로 적으면 `0.101`이 된다.

### 🎯 2진수 소수 ➡️ 10진수 소수 (음수 가중치)

소수점 아래로 내려갈수록 가중치가 $2^{-1}, 2^{-2}, 2^{-3} \dots$ 순으로 작아진다. (즉, $0.5, 0.25, 0.125 \dots$)

**[예시: 2진수 0.101을 10진수로]**

* $1 \times 2^{-1} + 0 \times 2^{-2} + 1 \times 2^{-3}$
* $= 0.5 + 0 + 0.125 = \mathbf{0.625}$

## 3. 컴퓨터의 실수 저장 방식: 부동소수점 (Floating Point)

컴퓨터는 메모리가 한정되어 있기 때문에, 엄청나게 큰 실수나 미세하게 작은 실수를 고정된 비트 안에 우겨넣어야 한다. 그래서 소수점의 위치를 둥둥 떠다니게(Float) 만들어서 저장하는 **부동소수점(IEEE 754 표준)** 방식을 사용한다.

64비트(Double precision) 환경에서 실수는 크게 3파트로 쪼개어 저장된다.

1. **부호(Sign) 1비트**: 양수(0)인지 음수(1)인지
2. **지수(Exponent) 11비트**: 소수점이 어디에 위치하는지
3. **가수(Mantissa) 52비트**: 실제 유효한 숫자 데이터들

이 방식 덕분에 적은 메모리로 넓은 범위의 숫자를 표현할 수 있지만, 치명적인 **'정밀도의 한계'**를 가지게 된다.

## 4. 치명적인 함정, 덧셈의 오류 (`0.1 + 0.2 != 0.3`)

이 '정밀도의 한계' 때문에 프로그래밍 역사상 가장 유명한 오류인 덧셈의 오류가 발생한다.

```python
print(0.1 + 0.2 == 0.3)  # 출력: False
print(0.1 + 0.2)         # 출력: 0.30000000000000004

```

### 💡 왜 이런 일이 발생할까?

앞서 배운 '10진수 소수 ➡️ 2진수 소수' 변환법으로 10진수 `0.1`을 변환해 보면 그 이유를 알 수 있다.

* $0.1 \times 2 = 0.2$ (정수 **0**)
* $0.2 \times 2 = 0.4$ (정수 **0**)
* $0.4 \times 2 = 0.8$ (정수 **0**)
* $0.8 \times 2 = 1.6$ (정수 **1**)
* $0.6 \times 2 = 1.2$ (정수 **1**)
* $0.2 \times 2 = 0.4$ (정수 **0**) ➡️ *여기서부터 무한 반복 발생!*

즉, 10진수 `0.1`은 2진수로 바꾸면 `0.0001100110011...` 처럼 영원히 끝나지 않는 **무한 소수**가 된다.
컴퓨터는 부동소수점의 가수부(52비트)까지만 이 숫자를 저장하고 **나머지 꼬리는 싹둑 잘라버린다(반올림).** 이렇게 미세하게 잘려나가 근사치가 된 `0.1`과 `0.2`를 더하니, 당연히 완벽한 `0.3`이 나오지 않고 `0.30000000000000004` 같은 찌꺼기가 생기는 것이다.

### 🛡️ 해결책 (안전한 실수 비교)

이런 덧셈의 오류를 피하기 위해, 두 실수가 같은지 비교할 때는 `==` 대신 파이썬 내장 라이브러리인 `math.isclose()`를 사용해야 한다. (미세한 찌꺼기 오차를 무시하고 비교해 줌)

```python
import math
print(math.isclose(0.1 + 0.2, 0.3))  # 출력: True

```
