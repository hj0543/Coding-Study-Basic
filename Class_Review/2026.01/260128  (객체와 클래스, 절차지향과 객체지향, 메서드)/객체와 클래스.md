
# 📝 클래스와 객체 (Class & Object)

파이썬은 **모든 것이 객체(Object)**로 이루어진 객체 지향 언어이다. 이를 이해하기 위해 '설계도'와 '제품'의 비유를 가장 많이 사용한다.

## 1. 개념 정의

### ① 클래스 (Class) = 설계도 (Blueprint)

* 객체를 찍어내기 위한 **틀**이다.
* 관련된 **데이터(속성)**와 **기능(메서드)**을 하나로 묶어 정의한 것이다.
* **표기법**: `PascalCase`를 사용한다. (예: `MyClass`, `Person`)

### ② 객체 (Object) = 사물 (Thing)

* 클래스에 의해 정의된 속성과 기능을 가진 실체이다. (추상적인 개념)

### ③ 인스턴스 (Instance) = 제품 (Product)

* 클래스를 통해 **메모리에 실제로 생성된 객체**를 부르는 말이다.
* **관계**: "아이유는 객체다" (O), "아이유는 가수(Singer) 클래스의 **인스턴스**다" (O)

```python
# 클래스 정의 (설계도 제작)
class Singer:
    pass

# 인스턴스 생성 (제품 생산)
iu = Singer() 
bts = Singer()

# 타입 확인
print(type(iu)) # <class '__main__.Singer'>

```

## 2. 클래스의 구성 요소

### ① 생성자 메서드 (`__init__`)

* 인스턴스가 **생성될 때 자동으로 호출**되는 특별한 메서드이다.
* 주로 인스턴스의 **초기값(속성)을 설정**하는 데 사용한다.

### ② `self` (자기 자신)

* 인스턴스 **자기 자신**을 가리키는 변수다.
* 메서드 정의 시 **첫 번째 인자**로 반드시 `self`를 적어야 한다. (호출할 때는 파이썬이 알아서 넣어주므로 생략한다.)

```python
class Person:
    # 생성자: 태어날 때 이름과 나이를 부여함
    def __init__(self, name, age):
        self.name = name  # 인스턴스 변수 생성
        self.age = age

    # 메서드: 행동 정의
    def introduce(self):
        # self.name을 통해 내 이름에 접근
        print(f"저는 {self.name}입니다.")

p1 = Person('Alice', 25) # __init__ 호출됨
p1.introduce()           # self에 p1이 자동으로 전달됨

```

## 3. 변수의 종류 (핵심)

변수가 어디에 선언되었느냐에 따라 **공유 여부**가 달라진다.

### ① 인스턴스 변수 (Instance Variable)

* **`self.변수명`** 으로 정의한다. (`__init__` 내부 등)
* **각 인스턴스마다 독립적인 값**을 가진다. (내 돈은 내 돈, 네 돈은 네 돈)

### ② 클래스 변수 (Class Variable)

* **클래스 바로 아래**에 정의한다.
* **모든 인스턴스가 공유**하는 값이다. (공용 통장)

```python
class Circle:
    pi = 3.14  # [클래스 변수] 모든 원이 공유하는 값

    def __init__(self, radius):
        self.radius = radius  # [인스턴스 변수] 각 원마다 다른 반지름

c1 = Circle(1)
c2 = Circle(5)

print(c1.radius) # 1
print(c2.radius) # 5
print(c1.pi)     # 3.14 (공유 값)

```

## 4. 이름 공간(Namespace)과 탐색 순서

`instance.attribute`로 접근했을 때, 파이썬은 다음 순서로 값을 찾는다.

> **1. 인스턴스(Instance) 내부 → 2. 클래스(Class) 내부**

이 순서 때문에 **클래스 변수와 같은 이름의 인스턴스 변수를 만들면 가려지는(Shadowing)** 현상이 발생한다.

### 코드 흐름 분석

```python
class Circle:
    pi = 3.14  # 클래스 변수

    def __init__(self, r):
        self.r = r # 인스턴스 변수

c1 = Circle(1)
c2 = Circle(2)

# 1. 값 조회 (Read)
print(c1.pi) # 3.14 (c1에 pi가 없어서 클래스 변수 pi 참조)

# 2. 값 할당 (Write) - 주의!
c1.pi = 100 
# 클래스 변수 pi를 바꾸는 게 아니라, 
# c1이라는 인스턴스 내부에 'pi'라는 새로운 인스턴스 변수를 만든 것이다!

# 3. 결과 확인
print(c1.pi)       # 100 (본인의 인스턴스 변수 pi)
print(Circle.pi)   # 3.14 (클래스 변수는 그대로임)
print(c2.pi)       # 3.14 (c2는 인스턴스 변수 pi가 없어서 여전히 클래스 변수 참조)

```

## 💡 기본 데이터 타입도 사실 클래스다

파이썬의 `int`, `str`, `list` 등도 모두 클래스이며, 우리가 사용하는 값들은 그 클래스의 인스턴스이다.

```python
a = 10       # int 클래스의 인스턴스
b = "hello"  # str 클래스의 인스턴스

# "hello".upper()는
# str 클래스에 정의된 upper 메서드를 "hello"라는 인스턴스가 호출한 것이다.

```
