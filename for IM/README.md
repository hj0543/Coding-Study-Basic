삼성 SW 역량테스트 **IM (Intermediate) 등급**은 'Advanced(A형)'로 넘어가기 위한 관문이다. A형이 BFS/DFS 같은 알고리즘을 요구한다면, **IM은 "복잡한 알고리즘 없이, 반복문(for/while)과 조건문(if)만으로 문제를 정확하게 구현할 수 있는가?"**를 본다.    

난이도는 대략 **백준 브론즈 1 ~ 실버 3** 정도이며, **자료구조나 테크닉보다는 '꼼꼼함'**이 생명이다.    

IM 등급 취득을 위해 반드시 정복해야 할 **4가지 핵심 유형**과 **추천 문제**를 정리

# 🎯 IM 대비 필수 체크리스트

## 1. 2차원 배열과 델타 탐색 (가장 중요)

IM 문제의 60% 이상은 **격자(Grid) 위에서의 시뮬레이션**이다.

* **필수 역량**:
* `dx`, `dy`를 이용한 4방향 탐색.
* 범위 체크 (`0 <= nx < N`).
* 배열 돌리기 (회전), 배열 일부 자르기.
* **빙고, 오목, 지뢰찾기** 같은 로직 구현.



## 2. 단순 구현 & 시뮬레이션 (시키는 대로 하기)

복잡한 생각보다는 문제에 적힌 순서대로 상태를 변화시키는 유형이다.

* **필수 역량**:
* 스위치 켜고 끄기 (특정 배수만 끄기, 구간 뒤집기 등).
* 줄 세우기 (삽입/삭제).
* 색종이 붙이기 (2차원 배열 영역 채우기).



## 3. 완전 검색 (Brute Force)

이 작기 때문에(보통 100 이하), 스마트한 방법 찾느라 고민할 필요 없다.

* **필수 역량**:
* 2중, 3중 `for`문을 돌려서 모든 경우를 다 확인하기.
* 예: 일곱 난쟁이 (9명 중 합이 100이 되는 7명 찾기).



## 4. 문자열 처리 & 카운팅

* **필수 역량**:
* 문자열 뒤집기, 특정 패턴 찾기.
* `DAT` (Direct Address Table) 활용: 리스트의 인덱스를 숫자로 보고 카운팅하기 (빈도수 세기).

# 💡 IM 시험 꿀팁 (Mindset)

1. **인덱스 에러 주의**: 문제는 보통 `1번`부터 시작하는데, 배열은 `0번`부터 시작한다.   
* 팁: 헷갈리면 배열을 `N + 1` 크기로 만들어서 `0`번 인덱스를 버리고 `1`번부터 쓸 것.


2. **시간 복잡도 무시**: IM은 보통 데이터 크기가 작습니다. 효율성보다는 **정확성**이다다. 3중 for문을 써도 되니 답만 정확하게 나오게 할 것.

3. **지문 정독**: "자신을 포함한다/안 한다", "이상/초과" 같은 조건을 놓치면 1시간 동안 디버깅해야 한다.
