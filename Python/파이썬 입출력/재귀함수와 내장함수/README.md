### 1. 재귀 함수 (Recursive Function)

> **학습 목표:** 재귀 함수의 개념과 동작 원리를 이해하고, 간단한 문제를 재귀적으로 해결할 수 있다.

재귀 함수는 함수 내부에서 **자기 자신을 다시 호출**하는 함수를 의미합니다. 복잡한 문제를 동일한 구조의 더 작은 문제로 쪼개어 해결할 때 유용합니다.

#### 핵심 구성 요소

* **종료 조건 (Base Case):** 재귀 호출을 멈추는 조건입니다. 이 조건이 없으면 함수가 무한히 호출되어 'RecursionError'가 발생합니다.
* **재귀 단계 (Recursive Step):** 문제를 더 작은 단위로 쪼개어 자기 자신을 호출하는 부분입니다.

#### 대표 예시: 팩토리얼 ()

```python
def factorial(n):
    # 종료 조건: n이 0이면 1을 반환
    if n == 0:
        return 1
    else:
        # 재귀 호출: n과 n-1의 팩토리얼을 곱한 결과를 반환
        return n * factorial(n - 1)

# 팩토리얼 계산 예시
print(factorial(5))  # 120

```

### 2. 파이썬 내장 함수 (Built-in Functions)

> **학습 목표:** 다양한 내장 함수(len, max, sum)를 활용할 수 있다.

파이썬은 별도의 `import` 없이도 즉시 사용할 수 있는 유용한 함수들을 미리 제공합니다. 이미지에서 언급된 주요 함수들은 다음과 같습니다.

| 함수명 | 설명 | 예시 |
| --- | --- | --- |
| **`len()`** | 객체의 길이나 항목의 개수를 반환합니다. | `len([1, 2, 3])` → `3` |
| **`max()`** | 인자 중 가장 큰 값을 반환합니다. | `max(1, 5, 2)` → `5` |
| **`sum()`** | 숫자로 이루어진 반복 가능한 객체의 합을 구합니다. | `sum([1, 2, 3])` → `6` |

### 3. 함수 작성 원칙 (Clean Code)

> **학습 목표:** 함수 이름 규칙과 단일 책임 원칙을 준수하여 가독성 좋은 함수를 작성할 수 있다.

* **함수 이름 규칙:** 함수의 역할이 명확히 드러나도록 동사 위주의 이름을 사용합니다.
* **단일 책임 원칙 (Single Responsibility Principle):** 하나의 함수는 반드시 **하나의 기능**만 수행하도록 작성하여 유지보수성을 높입니다.
