삼성 SW 역량테스트 **Advanced (A형) 등급**은 'Pro(B형/C형)'로 가기 위한 본격적인 알고리즘 관문이다. IM이 단순 구현이라면, **A형은 "완전 탐색(Brute Force)을 기반으로 하되, 효율적인 탐색(DFS/BFS/Backtracking)과 정교한 시뮬레이션을 구현할 수 있는가?"**를 본다.

난이도는 대략 **백준 골드 5 ~ 골드 2** 정도이며, **자료구조(Queue, Stack) 활용 능력과 시간 복잡도 최적화**가 생명이다.

A형 등급 취득을 위해 반드시 정복해야 할 **4가지 핵심 유형**과 **추천 문제**를 정리했다.

# 🎯 Advanced (A형) 대비 필수 체크리스트

## 1. 그래프 탐색 (DFS & BFS) - 기본 중의 기본

A형 문제의 80%는 그래프 탐색을 포함한다. 최단 거리는 BFS, 경로의 특징이나 끝까지 파고드는 건 DFS가 유리하다.

* **필수 역량**:
* `collections.deque`를 활용한 **BFS 구현** (큐).
* **재귀(Recursion)**를 활용한 **DFS 구현** (스택).
* **방문 처리 (`visited`)**: 중복 방문 방지 및 상태 기록.
* **Flood Fill**: 연결된 영역(섬, 단지) 찾기.



## 2. 백트래킹 (Backtracking) - 스마트한 완전 탐색

무식하게 다 찾는 건 시간 초과다. **"가망이 없으면 되돌아간다(Pruning, 가지치기)"**는 것이 핵심.

* **필수 역량**:
* **상태 트리(State Space Tree)** 그리기.
* **순열과 조합 ()**: 직접 구현하거나 라이브러리 없이 재귀로 짜기.
* **재귀 종료 조건 (Base Case)** 명확히 설정하기.
* `return` 후 상태 복구하기 (원상복구).



## 3. 시뮬레이션 (Simulation) - 2차원 배열 심화

IM의 시뮬레이션보다 훨씬 복잡한 규칙이 적용된다. (동시 이동, 회전, 중력, 폭발 등)

* **필수 역량**:
* **배열 회전**: 90도/180도 회전, 부분 회전 구현.
* **중력 작용**: 빈 공간이 생기면 위의 요소가 떨어지는 로직.
* **동시성 처리**: 모든 객체가 동시에 이동할 때, `temp` 배열을 쓰거나 상태를 저장해두고 갱신하기.
* **상어 시리즈**: 물고기 냄새, 상어 이동 우선순위 등 복합 조건 처리.



## 4. 완전 탐색 + 최적화

단순히 4중 for문을 돌리는 게 아니라, **"조합으로 벽을 3개 세우고(백트래킹) -> 바이러스를 퍼뜨려본다(BFS)"** 처럼 두 가지 알고리즘을 섞어 쓴다.

* **필수 역량**:
* 문제의  크기를 보고 시간 복잡도 예측하기 (이면 순열 가능, 이면 부분집합 가능 등).
* **Deepcopy**: 2차원 배열 상태를 보존하고 시뮬레이션 돌려보기.

# 💡 A형 시험 꿀팁 (Mindset)

1. **시간 복잡도 계산**: 문제의 을 보고 어떤 알고리즘을 쓸지 결정해야 한다.
* : 순열/백트래킹 가능.
* :  부분집합 가능.
* :  (플로이드 와샬) 가능.
* :  (BFS/DFS) 가능.


2. **재귀 깊이 주의**: 파이썬은 재귀 깊이 제한이 타이트하다. DFS 문제라면 반드시 `import sys; sys.setrecursionlimit(10**6)`을 상단에 적고 시작하자.
3. **모듈화(함수 분리)**: 코드가 길어진다. `move()`, `check()`, `rotate()` 처럼 기능별로 함수를 쪼개야 디버깅할 때 멘탈이 안 나간다.
4. **deepcopy 조심**: 2차원 배열을 복사할 때 `b = a`라고 쓰면 참조만 복사되어 원본이 망가진다. `b = [row[:] for row in a]` 또는 `copy.deepcopy`를 활용하자.
